## What Is Not Migrated

- Upstream implementation code is not migrated into TuringResearch's source tree.
- Third-party PDFs, images, datasets, and raw generated outputs are not migrated.
- Private session logs, raw user data, local paths, `.env`, API keys, and secrets are excluded.

## Why Code Is Not Migrated

TuringResearch's implementation remains self-developed. Upstream code may be referenced for comparison, but the repository should preserve a clean authorship story: upstream projects provide authorized academic workflow outputs and design references; TuringResearch implements its own local-first research operating system modules.
