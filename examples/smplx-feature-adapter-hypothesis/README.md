# TuringResearch Example: SMPL-X Feature Adapter Hypothesis

Mode: fake-service dry run.

This example demonstrates a hypothesis-to-feature-adapter flow for SMPL-X related research ideas using only public or user-owned notes.

## Expected Artifacts

- HypothesisPortfolio
- IdeaPortfolio
- DecisionReport
- StressTestReport

## Inputs

- `input/request.json` defines the adapter hypothesis prompt and source-hygiene assumptions.
- `fake_run_config.yaml` fixes dry-run mode, fake hypothesis/ideation/convergence/stress services, and no-network execution.

## Outputs

- `expected_outputs/artifact_list.json` lists the required artifacts.
- `expected_outputs/expected_summary.md` documents the expected dry-run result.

No real network access, real API key, Semantic Scholar key, Apify token, private repository content, or unauthorized implementation detail is required.
