# Research Catalog Demo

Status: fake/demo only.

This demo shows how a public research task can be routed through the
TuringResearch Research Catalog.

## Demo Task

Prepare a public-safe review plan for a research workflow without running a real
experiment.

## Catalog Route

| Step | Surface | Demo output |
| --- | --- | --- |
| 1 | Campaign Catalog | Choose `north_star` and `knowledge_acquisition`. |
| 2 | Skill SOPs | Route to review-only skills, not automatic execution. |
| 3 | Vault graph | Organize concepts and claims as review nodes. |
| 4 | Ontology SOPs | Check aliases, gaps, hierarchy, and dangling edges. |
| 5 | Stress tests | Challenge evidence, artifacts, privacy, route, and advisor claims. |
| 6 | Experiment runbook | Prepare artifact requirements for a human-run experiment. |
| 7 | Advisor pack | Package review material without final-paper claims. |
| 8 | Public release | Run privacy/security and fake/live gates before publishing. |

## Expected Boundary

The catalog recommends local review surfaces. It does not call live providers,
run experiments, execute plugins, or mark fake/demo output as observed evidence.
