# Launch README Sections

Status: planning draft.

Round: 163.

This document lists the README sections that should be visible before public
launch.

## Required Sections

1. Project title and one-line position.
2. Why star it / why it matters.
3. What it does.
4. What it does not do.
5. Public demo.
6. VGGT dogfooding case.
7. Quickstart.
8. Packages / namespaces.
9. Documentation links.
10. Architecture snapshot.
11. Roadmap and repository strategy.
12. Safety boundaries.
13. Development checks.
14. Status.

## First-Screen Requirements

The first screen must include:

- local-first Research OS;
- evidence-led research work;
- fake/demo-first posture;
- human review requirement;
- no autonomous science claim.

## Comparison Section Option

If space allows, add a short comparison with ordinary literature tools:

- ordinary tools summarize papers;
- TuringResearch connects evidence, artifacts, routes, paper scaffolds,
  dashboards, advisor packs, plugins, and gates.

## Copy Guardrails

Avoid:

- "automates research";
- "writes the paper";
- "proves VGGT success";
- "replaces advisor review";
- "benchmark-winning";
- fake user quotes;
- fake offer or internship association.
