# Showcase Thread Draft

Status: draft.

Round: 197.

1. I built TuringResearch Plus: a local-first Research OS for evidence-led
   research workflows.

2. Research state tends to scatter across papers, notes, artifacts, routes,
   dashboards, advisor feedback, and half-written claims.

3. TuringResearch Plus connects evidence ledgers, artifact audits, route DSL,
   paper review, advisor packs, dashboards, plugin safety, and privacy gates.

4. The key rule: planned work is not observed evidence. Missing evidence stays
   visible.

5. v1.0 is fake/demo-first by default. Live adapters are optional and disabled
   unless explicitly enabled.

6. It includes a public demo suite, VGGT public-safe dogfooding case, Campaign
   Catalog, MCP config polish, and split-ready local bundles.

7. The plugin system is intentionally conservative: manifests are required,
   unknown plugins are disabled, `execute_code` is not a default, and secrets
   access is forbidden.

8. The dashboard and advisor pack are review surfaces, not proof of experiment
   success.

9. The engineering focus is contracts, tests, privacy gates, safety boundaries,
   and honest product positioning.

10. It does not automate science, write final papers, replace advisors, or
    claim VGGT/SparseConv3D success.

11. If you care about local-first research tooling and evidence discipline, the
    repo is worth a look.
