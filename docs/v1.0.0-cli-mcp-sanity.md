# v1.0.0 CLI / MCP Sanity

Status: sanity guide draft.

Round: 178.

## CLI Surface

TuringResearch Plus exposes two console scripts:

| Command | Target | Purpose |
| --- | --- | --- |
| `turingresearch-plus` | `turing_research.mcp_server:main` | local smoke entrypoint |
| `turingresearch-plus-mcp` | `turing_research.mcp_server:main` | explicit MCP stdio entrypoint |

Both commands point to the same safe stdio module. Importing the module must not
start a network server.

## MCP Surface

| Field | Value |
| --- | --- |
| server name | `turingresearch-plus` |
| module | `turing_research.mcp_server` |
| entrypoint | `turing_research.mcp_server:main` |
| transport target | STDIO |
| default mode | fake/local |

## Safe Commands

```powershell
python -m turing_research.mcp_server --manifest
python -m turing_research.mcp_server --health-check
turingresearch-plus-mcp --manifest
```

These commands must stay safe for default local checks:

- no real API key required;
- no live networking required;
- no plugin execution;
- no private path read;
- no output promoted to observed research evidence.

## Config Template

`.mcp.example.json` is a no-secret template. It is not a private live config.
Do not commit a modified config containing real credentials.

## Plugin Boundary

Plugin tools remain disabled by default:

```text
TURINGRESEARCH_ENABLE_PLUGINS=0
TURINGRESEARCH_ENABLE_PLUGIN_LIVE_MODE=0
```

The MCP plugin registry may describe candidate tools, but it does not enable
third-party tools by itself.

## v1.0 Release Note

The CLI/MCP surface is a local smoke and integration surface for v1.0. Public
release still requires clean branch review and maintainer approval.
