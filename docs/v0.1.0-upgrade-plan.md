# TulingResearch Plus v0.1.0 Upgrade Plan

## From Release Candidate To v0.1.0

1. Perform final human review of public docs and release materials.
2. Confirm `VERSION`, `pyproject.toml`, `src/tuling_research/__init__.py`, and `src/tuling_research_plus/__init__.py` all report `0.1.0`.
3. Run the release validation commands:

   ```powershell
   python -m pytest
   python -m ruff check .
   python -m mypy src
   ```

4. Review `docs/release-blockers.md`.
5. Review `docs/license-review.md`.
6. Tag the release only after human approval.

## Post-v0.1.0 Upgrade Candidates

- Implement selected contract-only tools.
- Add live adapters behind explicit `live` or `manual` configuration.
- Expand PDF OCR and advanced layout parsing.
- Add richer paper drafting once `ExperimentReport` artifacts exist.
- Prepare public package publication after license and security review.
