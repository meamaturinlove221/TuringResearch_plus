# v1.5.0 Release Checklist

Status: manual checklist.

Round: 358.

Do not execute release actions automatically. This checklist is for a maintainer
reviewing whether to publish v1.5.0rc0.

## Pre-release Checks

- [ ] Review `docs/v1.5.0-release-notes.md`.
- [ ] Review `docs/github-release-draft-v1.5.0.md`.
- [ ] Review `docs/v1.5.0-full-replay-report.md`.
- [ ] Review `docs/v1.5.0-security-audit.md`.
- [ ] Review `docs/v1.5.0-privacy-audit.md`.
- [ ] Review `docs/v1.5.0-secret-scan-report.md`.
- [ ] Confirm `VERSION`, `pyproject.toml`, and package `__version__` values are
      `1.5.0rc0`.
- [ ] Run v1.5 full replay locally.
- [ ] Run v1.5 security/privacy gate locally.
- [ ] Run name integrity and public release hygiene checks.

## Manual Release Actions

- [ ] Create or select the clean release branch manually.
- [ ] Commit only reviewed v1.5 changes.
- [ ] Create tag manually if approved.
- [ ] Publish GitHub release manually if approved.
- [ ] Publish PyPI package manually if approved.
- [ ] Publish docs site manually if approved.
- [ ] Create split child repositories manually only if approved.

## Must Not Happen Automatically

- no automatic tag;
- no automatic GitHub release;
- no automatic PyPI publish;
- no automatic public docs deployment;
- no automatic child repository creation;
- no automatic external push;
- no live provider call;
- no SSH or SFTP connection;
- no remote command execution.

## Human Review Boundary

The release can proceed only after maintainer review. This checklist is not a
release action and not proof that publication happened.
