# v1.4.0 Release Notes

Status: release-candidate notes.

Round: 327.

TuringResearch Plus v1.4.0 is an original repo production parity
release-candidate line. It turns the v1.3 fake/default original parity surfaces
into production-style operator paths with CLI entry points, E2E demos,
dashboards, safety gates, and full regression coverage.

This release does not publish a SaaS product, does not execute experiments
automatically, does not enable live networking by default, does not create child
repositories automatically, and does not adopt ARIS runtime features into v1.4.

## Highlights

- Session production parity.
- Session CLI surface.
- Shell script equivalent export.
- Cross-platform archive hardening.
- Remote dry-run plan.
- Return import human confirmation.
- Scholar production parity.
- Paper content/reference/reading E2E.
- Web production parity.
- URL normalization/cache/content fixtures.
- Apify fake/live report.
- yogsoth production parity.
- Campaign/catalog/vault/ontology/stress/convergence/experiment E2E.
- Parity dashboard v2.

## What Is New Since v1.3

- Session parity now has a fake/default production workflow with CLI, script
  export, archive hardening, remote dry-run planning, human confirmation, E2E
  replay, and dashboard v2.
- Scholar parity now has production-style paper content, reference, and
  three-pass reading E2E demos.
- Web parity now has hardened URL normalization, cache manifest provenance,
  local extraction fixtures, and an Apify fake/live report with live skipped by
  default.
- yogsoth parity now has E2E workflows for campaign trace, Research Catalog,
  Vault Wiki, Ontology, Stress / Convergence, and Experiment Runbook flows.
- Original Repo Parity Dashboard v2 shows structural, runtime, production, and
  deferred status in one public-safe view.
- Security/privacy and full regression now cover v1.4 production additions.

## What Remains Review-gated

- Release branch push, tag creation, GitHub release, and PyPI publishing.
- Live provider tests and live adapter polish.
- Live SSH/SFTP beyond explicit private opt-in.
- ARIS study and any future cross-model review loops.
- Remote execution orchestration.
- Automatic experiment execution.

## Release Boundary

v1.4.0 remains a local-first Research OS release candidate. Human review is
required before publication, claims, external repository creation, live provider
usage, live SSH/SFTP usage, or public release actions.
