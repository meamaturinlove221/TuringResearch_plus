# v1.0.0 Fake / Live Mode Guide

Status: mode guide draft.

Round: 178.

## Default: Fake Mode

Fake mode is the default and release-safe path.

Expected defaults:

```text
TURINGRESEARCH_MODE=fake
TURINGRESEARCH_ENABLE_LIVE_TESTS=0
TURINGRESEARCH_ENABLE_PLUGINS=0
TURINGRESEARCH_ENABLE_PLUGIN_LIVE_MODE=0
```

Fake mode may use:

- local fixtures;
- dry-run reports;
- public demo data;
- static dashboards;
- manifest validation;
- contract tests.

Fake mode must not require:

- live API credentials;
- network access;
- private project paths;
- raw data;
- model payloads;
- unknown plugin execution.

## Optional: Live Mode

Live mode is opt-in only.

A private local config may set:

```text
TURINGRESEARCH_ENABLE_LIVE_TESTS=1
```

Live mode still requires human review:

- credentials must remain outside committed files;
- retrieved material is not automatically verified evidence;
- live provider output must not be marked observed without evidence review;
- live plugin mode remains disabled unless separately reviewed.

## Plugin Mode

Plugin tools are disabled by default. Enabling plugin metadata review is not
the same as enabling plugin execution.

Default:

```text
TURINGRESEARCH_ENABLE_PLUGINS=0
TURINGRESEARCH_ENABLE_PLUGIN_LIVE_MODE=0
```

## What Not To Do

- Do not commit live credentials.
- Do not put real API keys into `.mcp.example.json`.
- Do not treat live retrieval as final research evidence.
- Do not enable unknown third-party plugin execution.
- Do not make live tests part of default test runs.
