## Workstream Test Expectations

### v1.0 baseline stabilization

- name integrity;
- v1 release contracts;
- v1 security/privacy gate;
- branch-state documentation checks.

### Split-ready finalization

- split safety tests;
- privacy gate;
- compliance gate when case material is involved;
- no nonexistent external repository URL check.

### Docs site

- docs link check;
- public release hygiene;
- no private path / secret scan;
- no fake-result-as-observed wording.

### Local server dashboard

- unit tests for local server config;
- workflow smoke for fake public demo data;
- static fallback rendering check;
- no login/no cloud boundary checks.

### Paper writing beta

- paper model/contracts tests;
- evidence-link tests;
- claim-safety tests;
- human-review marker tests.

### More public demos / case gallery

- public demo workflow tests;
- privacy gate;
- expected output checks;
- no unsupported result claim checks.

### CI/CD release hardening

- release contract tests;
- branch hygiene docs/checks;
- no automatic publish/tag action in default tests.

## Non-testable Human Gates

The following remain human review items:

- final release approval;
- license posture;
- public child repository creation;
- final README tone;
- public screenshots/GIFs.
