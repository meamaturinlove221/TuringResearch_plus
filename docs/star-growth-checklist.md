## Before Public Launch

- [ ] README first screen explains local-first Research OS.
- [ ] One-line pitch is clear.
- [ ] 30-second pitch is clear.
- [ ] Quickstart is local and fake/demo-safe.
- [ ] Public demo is easy to find.
- [ ] Dashboard screenshot/GIF plan is ready.
- [ ] VGGT dogfooding case is clearly review-only.
- [ ] Plugin safety story is clear.
- [ ] Privacy and compliance boundaries are visible.
- [ ] Tests and contracts are summarized.
- [ ] Roadmap is honest.
- [ ] Limitations are visible.
- [ ] Architecture diagram is visible near the top-level docs.
- [ ] Demo route can be completed in 10 minutes without live services.
- [ ] Public case study says dogfooding, not experiment success.

## Star Protection

- [ ] Main repo remains the flagship.
- [ ] Split candidates link back to main repo.
- [ ] Case/examples are optional demo/case material.
- [ ] Install path remains in main repo.
- [ ] Main repo keeps complete quickstart.
- [ ] No spoke repo claims to replace the main repo.
- [ ] Spoke README files ask readers to inspect/star the flagship first.
- [ ] Main repo still contains enough demo material to be understood alone.

## Anti-overclaim Check

- [ ] No automatic research completion claim.
- [ ] No final paper generation claim.
- [ ] No VGGT success claim.
- [ ] No SparseConv3D success claim.
- [ ] No fake benchmark.
- [ ] No fake users.
- [ ] No fake internship or offer association.
- [ ] No legal compliance guarantee.

## Launch Warmup Checklist

- [ ] GitHub profile snippet reviewed.
