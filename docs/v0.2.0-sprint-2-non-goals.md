# TuringResearch Plus v0.2.0 Sprint 2 Non-Goals

Status: scope locked.

Sprint 2 deliberately stays narrow. It improves experiment semantics,
gatekeeping, failure language, method cards, and architecture drafts. It does
not expand into live execution, synchronization, heavy PDF processing, or
automatic paper writing.

## Explicit Non-Goals

- Do not run real VGGT experiments.
- Do not implement Modal live execution.
- Do not implement SSH artifact sync.
- Do not implement NAS / SMB shared artifact storage.
- Do not implement GitHub artifact sync.
- Do not implement cloud object storage sync.
- Do not implement heavy OCR.
- Do not implement MinerU or Marker as required dependencies.
- Do not implement complete paper auto-writing.
- Do not implement complex visual classification models.
- Do not enable default networking.
- Do not add live Semantic Scholar or arXiv adapters in this sprint.
- Do not implement real citation graph expansion.
- Do not claim SparseConv3D success without real evidence.
- Do not read private `D:/vggt` paths in default tests or docs examples.

## Allowed Local Work

- Contracts-first schema design.
- Local deterministic models and services.
- JSON and Markdown serialization.
- Fake/dry-run workflow examples.
- Local fixture tests.
- Mermaid architecture draft output.
- Conservative `requires-human-review`, `requires-real-paper`, and
  `not-enough-evidence` outputs.

## Network Policy

No network use is part of Sprint 2 implementation. The only exception is a
future explicitly scoped upstream scan round. Such a scan must be documented as
upstream watch work, not as Sprint 2 implementation behavior.

## Future Sync Adapters

The following remain future roadmap items:

- Handoff Bundle Export / Import.
- NAS / SMB Shared Artifact Store.
- SSH / SFTP Remote Artifact Reader.
- GitHub Artifact Sync.
- Optional cloud object storage adapter.

## Paper And Figure Boundaries

Paper-to-Method Card and Figure-to-Architecture may summarize local,
provenance-backed information. They must not fabricate paper claims, method
components, evaluation results, architecture edges, or visual conclusions.
