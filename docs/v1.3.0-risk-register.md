## Risks

| Risk | Severity | Mitigation |
| --- | --- | --- |
| Full parity is mistaken for unsafe upstream feature adoption | high | Keep fake/default demos and explicit deferred/rejected sections. |
| Session runtime parity drifts into SSH/tmux/provision | high | Keep remote execution disabled by default and require review-only fake workflow first. |
| Scholar/Web parity triggers default networking | high | Keep live provider flags opt-in and tests fake by default. |
| Campaign trace is mistaken for autonomous agent runtime | high | Make traces deterministic and non-executing. |
| Vault/Ontology demo is mistaken for final truth graph | medium | Mark graph output as review material. |
| Stress/convergence output is treated as release approval | medium | Keep human review and separate release gate. |
| Experiment runbook is mistaken for real execution | high | Use proposed-only language and no run execution. |
| ARIS slips into implementation scope | high | Keep `docs/v1.3.0-aris-still-deferred.md` and non-goals active. |
| Dirty branch hides unrelated changes | high | Require clean release branch before publication. |

## General Mitigation

- Keep tests fake/default.
- Keep live adapters opt-in.
- Keep human review required.
- Keep release actions manual.
- Keep public demos free of private data and unsupported claims.
