# v0.6 User Guide

Status: review-oriented user guide.

This guide explains the current v0.6 fake/default workflow surface. It does not
describe a hosted product, cloud service, or automatic experiment runner.

## Getting Started

Install in editable mode and run default tests:

```powershell
python -m pip install -e .[dev]
python -m pytest
```

Optional MCP smoke checks:

```powershell
python -m turing_research.mcp_server --manifest
turingresearch-plus-mcp --health-check
```

Live adapters are optional and disabled by default. Missing API keys should not
fail default tests.

## Workspace

Use a workspace when you want to organize multiple research cases.

Primary docs:

- [Multi-project Workspace](multi-project-workspace.md)

## Evidence And Artifacts

Evidence and artifact helpers organize review material and proposed imports.
They do not verify claims automatically.

Primary docs:

- [VGGT Evidence Ledger](vggt-evidence-ledger.md)
- [Artifact Auditor](artifact-auditor.md)
- [Visual Evidence Auditor](visual-evidence-auditor.md)
- [Remote Artifact Integration](remote-artifact-integration.md)

Remote artifact features are safety-gated and metadata-first. They do not
download large files or private data by default.

## Paper Workflow

Paper helpers build scaffolds, not final paper conclusions.

## VGGT Dogfooding Case

`examples/vggt-human-prior-survey/` is a research dogfooding case. It is useful
for exercising TuringResearch workflows, but it does not prove SparseConv3D
success and does not replace real VGGT experiments.
