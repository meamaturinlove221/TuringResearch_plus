# Public Docs Navigation Polish

Status: completed.

Round: 334.

The docs-site navigation has been polished around the path an external visitor
needs: what the project is, how to run it, how to extend it, what examples
exist, what the boundaries are, and where the roadmap goes next.

## Public Navigation Structure

1. Overview.
2. Quickstart.
3. Concepts.
4. Original Repo Parity.
5. Public Demo.
6. Docs / Dashboard.
7. Plugin / MCP.
8. Split Repos.
9. Security / Privacy.
10. FAQ.
11. Roadmap.

## Design Intent

- Keep the first-read path short and legible.
- Move internal implementation surfaces behind public-facing categories.
- Make production parity visible without overclaiming.
- Keep ARIS deferred and explicit.
- Preserve local-first, no-deploy, no-live-default boundaries.

## Non-goals

- No new core runtime.
- No public deployment.
- No real URL.
- No analytics.
- No private data upload.
- No automatic release publication.
