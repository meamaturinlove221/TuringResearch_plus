# v1.4.0 Roadmap

Status: adjusted roadmap.

Round: 291.

v1.4 is now scoped as **Original Repo Production Parity**, not an ARIS study
implementation cycle. Round 290 proposed ARIS study / fake prototypes, but the
latest human upstream scan shows the reference repositories are still mostly
advancing stable production surfaces around session context, Scholar/Web tool
entry points, MCP configuration, Apify/cache behavior, dotenv cleanup, and
research-engine usability. That means the better v1.4 investment is to make the
original-reference parity line more production-like before studying ARIS.

This is a plan adjustment based on a human scan summary. It is not a strict
machine diff and does not claim upstream added, modified, or deleted files.

## Default Recommendation

- v1.4 should prioritize original repo production parity.
- v1.4 should not implement ARIS features.
- v1.4 should keep fake/default paths runnable and live paths explicit opt-in.
- v1.4 should preserve the current runnable mainline.

## Locked v1.4 Scope

1. Neocortica-Session production parity.
2. Neocortica-Scholar production parity.
3. Neocortica-Web production parity.
4. yogsoth-ai research engine production parity.
5. Full original repo production replay.
6. Parity dashboard v2.
7. README / docs production polish.
8. v1.4 release prep.

## Recommended Sprint 1

Sprint 1 should start with:

1. Session production parity hardening: Git context, pod deployment planning,
   dotfile handling, shell safety, and cross-platform archive behavior.
2. Scholar production parity hardening: MCP config, paper content/reference/
   reading tool surfaces, README tool list, SKILL guide, and arxiv2md fallback
   policy.
3. Web production parity hardening: Apify templates, web fetching/content,
   cache, metadata, MCP config, and dotenv removal boundaries.
4. yogsoth research engine production replay: campaign trace, vault/ontology,
   stress/convergence, and experiment runbook continuity.
5. Dashboard and README polish after the production replay is green.

## ARIS Boundary

ARIS remains future reference only. v1.4 does not implement cross-model review,
proof-checker, meta-optimize, paper-claim-audit, automated sleep research loops,
or other ARIS runtime machinery.

## Release Boundary

v1.4 planning does not publish, tag, create a release, enable live adapters,
enable remote execution, upload data, or create child repositories
automatically.
