# TulingResearch Plus v0.1.0 Limitations

## Explicit Limitations

- Live network adapters are not enabled by default.
- OCR is not complete.
- Advanced PDF layout parsing is not complete.
- Paper draft does not fabricate experiment results.
- Real GPU experiment execution is not included.
- API keys are required only for future live mode.
- `v0.1.0` mainly demonstrates architecture, contracts, dry-run workflow, and local tools.

## Operational Limits

- Default tests and examples use fake mode or dry-run mode.
- External API integrations require future explicit configuration.
- Contract-only tools are public contract surfaces, not complete live implementations.
- TulingResearch Plus does not copy incompatible-license project code.
- Automatic PyPI publishing is not included.
- Public release still requires final human review.

## PDF Limits

- PDF Markdown Phase A supports local PDF paths and minimal text extraction.
- Heavy OCR is future work.
- Advanced layout-aware parsing is future work.
- Production-grade figure and table extraction is future work.

## Paper Pipeline Limits

The Paper Draft Gate blocks full paper drafting until required upstream artifacts exist, especially `ExperimentReport`. This prevents fabricated result text.

## Live Mode Limits

Future live mode may require API keys, provider configuration, rate-limit handling, and live/manual test markers. These are intentionally outside the default `v0.1.0` path.
