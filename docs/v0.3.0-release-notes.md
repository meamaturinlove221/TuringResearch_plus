# TuringResearch Plus v0.3.0 Release Notes

TuringResearch Plus v0.3.0 extends the v0.2 beta review workflow with
Git-based context handoff design, pod workflow packaging, refined scholar
pipelines, fake-default web retrieval, related-work positioning, skill routing,
vault graph review, ontology SOPs, and a VGGT Research Knowledge Pack.

This release remains conservative by design. Default workflows run locally in
fake, fixture, cache, or dry-run mode. Live web / Apify behavior is disabled by
default and live tests are skipped by default.

## Scope

v0.3.0 includes two integrated sprint surfaces:

- Sprint 1: context handoff, pod workflow, scholar pipeline, and web adapter
  planning.
- Sprint 2: web / Apify adapters, related-work positioning, skill routing,
  vault graph, ontology SOPs, and the VGGT knowledge pack.

## Main Use Cases

- Prepare portable context packages for remote or pod-side work without
  transferring secrets or raw data.
- Use cached/fake scholar and web inputs as review material for method cards and
  related-work positioning.
- Route tasks to repo-scoped `turingresearch-*` skills without executing them.
- Organize research concepts with a lightweight vault graph and wikilink export.
- Build a VGGT / SMPL-X review pack that separates observed evidence from
  planned work and missing artifacts.

## Release Boundary

TuringResearch Plus v0.3.0 does not run VGGT, does not run Modal, does not
publish to PyPI in this prep round, and does not generate final paper
conclusions. It does not claim SparseConv3D success.
