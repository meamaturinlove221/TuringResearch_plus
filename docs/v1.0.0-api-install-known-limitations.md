# v1.0.0 API / Install Known Limitations

Status: integration limitations recorded.

Round: 181.

This document records limitations for the v1.0 API/install surface. It exists
to keep public positioning honest.

## Limitations

- no SaaS product surface;
- no cloud user system;
- no real experiment execution;
- no automatic experiment execution;
- no automatic final paper writing;
- no default live networking;
- live adapters remain optional and explicit opt-in;
- unknown plugin execution is disabled by default;
- plugin tools disabled by default in the MCP example config;
- PDF/PPTX/live backends are optional and may skip gracefully;
- fake/demo replay is not observed research evidence;
- public quickstart outputs are demo-only;
- human review is required before any research claim;
- API freeze is a release-hardening contract, not a guarantee that all
  experimental modules are stable;
- `turing_research_plus` remains a compatibility namespace, while new domain
  namespaces are facade imports.

## Not Claims

The v1.0 API/install gate does not claim:

- production SaaS readiness;
- real VGGT experiment success;
- SparseConv3D success;
- automatic remote experiment execution;
- verified final paper conclusions;
- legal compliance approval;
- OS-level plugin sandboxing.

## Review Requirement

Before public release, a maintainer should review:

- package metadata;
- CLI/MCP examples;
- public API stability matrix;
- fake/live mode wording;
- public demo wording;
- privacy/security scan output;
- known limitations in README and docs.
