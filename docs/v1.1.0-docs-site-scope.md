# v1.1.0 Docs Site Scope

Status: scope locked.

Round: 215.

The docs site is a local-first static documentation layer over existing
repository Markdown. It does not deploy anything, does not require cloud
services, and does not introduce a large frontend framework.

## Scope

The docs site must cover:

1. introduction
2. quickstart
3. concepts
4. workspace
5. evidence
6. artifacts
7. paper
8. routes
9. dashboard
10. plugins
11. privacy
12. demos
13. case studies
14. API
15. FAQ

## Source Of Truth

The source of truth remains:

- `README.md`
- `docs/`
- `examples/public_demo/`
- `split_ready/`

The `docs-site/` directory should stay a lightweight navigation and generation
layer.

## Generation Strategy

Preferred v1.1 path:

- keep source docs in Markdown;
- use `docs-site/nav.yaml` for navigation;
- use `docs-site/pages/` as stable page stubs;
- optionally generate static HTML later from Markdown;
- keep static output local-first and reviewable.

## Non-goals

- No deployment.
- No cloud dependency.
- No login.
- No SaaS surface.
- No large frontend framework.
- No private data reads.
- No fake external links.
