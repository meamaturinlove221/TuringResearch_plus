# v1.5.0 Implementation Order

Status: scope locked.

Round: 331.

## Recommended Order

| Order | Workstream | Output | Gate |
| --- | --- | --- | --- |
| 1 | public docs deployment prep | deployment checklist, static docs bundle notes, nav checks | docs privacy gate |
| 2 | docs site build hardening | stricter docs-site build validation and broken-link checks | docs-site smoke |
| 3 | physical split execution pack | manual split pack checklist and bundle manifests | split safety gate |
| 4 | `vggt-case` repo manual pack | reviewed case repo bundle and handoff notes | case privacy gate |
| 5 | examples repo manual pack | reviewed examples repo bundle and handoff notes | examples privacy gate |
| 6 | optional live Scholar/Web/SFTP polish | opt-in docs, skipped-live tests, config validation | live skip gate |
| 7 | dashboard UX showcase | clearer local/public dashboard examples | dashboard smoke |
| 8 | v1.5 full replay | release-style fake/default replay across public externalization surfaces | full replay |
| 9 | v1.5 release prep | release notes, limitations, test summary, upgrade guide | release contract |
| 10 | v1.6 roadmap | next-scope recommendation, including ARIS study decision | roadmap review |

## Critical Path

The critical path is docs deployment prep -> docs-site hardening -> split pack
readiness -> full replay -> release prep. Optional live polish can run in
parallel only if it remains skipped by default and does not block the public
fake/default path.

## Do Not Start With ARIS

ARIS is not on the v1.5 implementation path. Starting with ARIS would blur the
release/public-externalization goal and introduce research-automation risk
before public release infrastructure is stable.
