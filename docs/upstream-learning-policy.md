# TuringResearch Plus Upstream Learning Policy

When public upstream changes appear:

1. Check whether TuringResearch Plus already has an equivalent capability.
2. If yes, compare upstream advantage, TuringResearch Plus advantage, gap,
   adoptable pattern, and whether to apply now, later, or ignore.
3. If no, generate an IdeaCard and a learning report before any implementation
   decision.
4. Only adopt architecture ideas, interface patterns, test strategies, quality
   gates, or documentation patterns.

Do not copy incompatible license code or private materials.

## Signals Worth Watching

- Vault edge audit and wiki edge audit patterns.
- Inline wiki link conventions for edge-creating skills.
- `ENTRY.md` routing tables for campaign orchestration.
- npm and npx publish metadata patterns.
- Ontology SOPs including seed concept search, source gathering, concept page
  creation, alias resolution, edge batch creation, hierarchy visualization, gap
  detection, merge candidates, confidence update, and ontology export.
