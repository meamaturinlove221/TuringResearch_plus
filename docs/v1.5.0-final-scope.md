# v1.5.0 Final Scope

Status: scope locked.

Round: 331.

v1.5 is scoped as **Public Externalization**. It follows v1.4 original repo
production parity and focuses on making the project easier to publish, review,
split, demo, and optionally exercise in private live mode.

This round locks scope only. It does not implement features, deploy docs,
create repositories, publish releases, or enable live behavior.

## Locked v1.5 Scope

1. Public docs deployment prep.
2. Docs site build hardening.
3. Physical split execution pack.
4. `vggt-case` repo manual pack.
5. Examples repo manual pack.
6. Optional live Scholar/Web/SFTP polish.
7. Dashboard UX showcase.
8. v1.5 full replay.
9. v1.5 release prep.
10. v1.6 roadmap.

## Scope Interpretation

- Public docs deployment prep means static-site readiness, navigation checks,
  deployment notes, and public-safe inputs. It does not mean automatic
  deployment.
- Physical split execution pack means manual, reviewed bundles for future child
  repos. It does not mean automatic GitHub repository creation.
- Optional live polish means better opt-in docs, config checks, and skipped-live
  tests. It does not make live network, SSH, SFTP, or providers default.
- Dashboard UX showcase means clearer local/public demo views. It does not add
  SaaS scope or live data dashboards.
- v1.5 full replay and release prep are gate work, not publication actions.

## ARIS Boundary

ARIS remains future reference only. v1.5 does not implement:

- ARIS runtime;
- cross-model review;
- proof-checker;
- meta-optimize;
- paper-claim-audit;
- automated sleep research loops.

If ARIS returns, it should be a separately scoped v1.6 study-only track by
default.

## Release Boundary

v1.5 planning does not publish, tag, create a release, create child
repositories, enable live adapters, enable remote execution, upload data, or
guarantee public attention.
