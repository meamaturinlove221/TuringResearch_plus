# v0.4.0 Sprint 1 Recommendation

Status: recommended scope.

## Recommendation

Because the current VGGT line still needs cross-machine and remote GPU handoff,
v0.4 Sprint 1 should prioritize controlled artifact return and run visibility.

Recommended Top 5:

1. GitHub Artifact Sync.
2. SSH / SFTP Remote Artifact Reader.
3. Modal Run Dashboard.
4. Advisor Pack PDF / PPTX Export.
5. Paper Digest / Three-pass Reading Expansion.

## Implementation Order

1. GitHub Artifact Sync contract and fake fixture path.
2. GitHub Artifact Sync minimal local/export-import implementation.
3. SSH / SFTP Remote Artifact Reader design and fake adapter.
4. Modal Run Dashboard data model and static report view.
5. Advisor Pack PDF / PPTX export from existing Markdown sections.
6. Paper Digest / Three-pass Reading expansion for reviewed notes.
7. v0.4 Sprint 1 integration gate.

## Rationale

- GitHub Artifact Sync is the closest practical extension of v0.3 Git-based
  context handoff and handoff bundles.
- SSH / SFTP is valuable for remote GPU and pod-style workflows, but carries
  more credential and path safety risk, so it should follow a stricter contract
  phase.
- Modal Run Dashboard should read structured run outputs; it must not run Modal.
- Advisor export is useful once evidence and run status can be presented safely.
- Paper Digest helps writing, but should not outrun artifact and evidence
  discipline.

## Alternate Writing-First Scope

If the project shifts away from remote GPU work and toward writing, the
recommended order should become:

1. Paper Digest / Three-pass Reading Expansion.
2. Related Work Positioning Refinement.
3. Paper Collision Risk Deep Review.
4. Advisor Pack PDF / PPTX Export.
5. Lightweight UI.

## Required Tests

- Fake/default sync tests with no network.
- Credential absence tests.
- Secret and raw-data exclusion tests.
- SMPL-X model file exclusion tests.
- Dashboard reads fixture outputs only.
- Advisor export does not invent evidence.
- Paper digest marks manual review requirements.
