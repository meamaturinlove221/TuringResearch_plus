# Star Growth Plan

Status: planning draft.

Round: 157.

This plan focuses public attention on the flagship repository before any future
hub-and-spoke split.

## Main Goal

Make the flagship repo easy to understand, star, and revisit.

## README First Impression

The README should immediately show:

- what the project is;
- why it matters;
- what it can demo today;
- what it refuses to overclaim;
- where to start.

## Star-Worthy Surfaces

- refined README;
- public demo screenshots;
- short demo GIF;
- static dashboard;
- VGGT case-study draft;
- plugin safety story;
- quality/replay report;
- clear limitations.

## What To Avoid

- fragmenting attention across many early repos;
- moving core code out of the flagship;
- hiding limitations below marketing text;
- presenting fake replay as benchmark success;
- publishing case-study material without privacy and claim safety checks.

## Near-Term Actions

1. Keep all public onboarding in the flagship.
2. Add screenshot and GIF plans before generating media.
3. Keep public demo and VGGT case links near the top of README.
4. Keep docs index current.
5. Keep release and security gates visible.

## Later Satellite Strategy

The first satellite candidates should be:

1. `turingresearch-vggt-case`
2. `turingresearch-examples`

Both should link back to the flagship in the first viewport and avoid
duplicating the main install story.

## Success Signals

- users can understand the project in one README pass;
- demos are inspectable without private data;
- limitations are clear;
- stars and issues concentrate on the flagship;
- external reviewers can explain the architecture and safety posture.
