# TulingResearch Plus v0.2.0 Risk Register

This risk register tracks v0.2.0 delivery risks before implementation begins.

| Risk ID | Risk | Area | Severity | Likelihood | Mitigation | Owner skill | Related backlog |
| --- | --- | --- | --- | --- | --- | --- | --- |
| R-01 | Live adapter rate limits make tests flaky. | Live Adapters | high | medium | Keep live tests skipped by default and use mocked contract tests. | `tulingresearch-qa-release` | BL-01, BL-02, BL-03, BL-04 |
| R-02 | Provider schema drift breaks live metadata parsing. | Live Adapters | high | medium | Adapter boundary, schema validation, typed adapter errors, fixture updates. | `tulingresearch-fusion-semantic-graph` | BL-01, BL-09 |
| R-03 | API keys leak through docs, tests, or logs. | Live Adapters | critical | low | `.env.example` only, no real values, secret hygiene docs, no stdout logs for stdio. | `tulingresearch-qa-release` | BL-04 |
| R-04 | PDF figure/table extraction produces misleading artifacts. | PDF Markdown | high | medium | Provenance metadata, warnings, quality report, fixture coverage. | `tulingresearch-pdf-markdown-core` | BL-05, BL-06, BL-08 |
| R-05 | Section tree changes break existing PDF Markdown consumers. | PDF Markdown | medium | medium | Preserve PDFMarkdownOutput compatibility and add regression tests. | `tulingresearch-pdf-markdown-core` | BL-07 |
| R-06 | Citation graph expansion exceeds budget. | Semantic Graph | high | medium | Depth limits, max node limits, frontier controls, BudgetGate integration. | `tulingresearch-fusion-semantic-graph` | BL-09 |
| R-07 | Survey gaps are generated without enough evidence. | Literature Survey | high | medium | Enforce EvidenceRef, abstract-only gates, full-text ratio checks. | `tulingresearch-fusion-literature-survey` | BL-10, BL-11, BL-12 |
| R-08 | Vault traversal returns unsupported or cyclic paths. | Vault | medium | medium | Hop limits, cycle guards, edge filters, evidence path tests. | `tulingresearch-fusion-wiki-vault` | BL-14 |
| R-09 | Artifact ingestion duplicates claim/evidence pages. | Vault | medium | medium | Idempotent ingestion keys and duplicate edge rejection. | `tulingresearch-fusion-wiki-vault` | BL-15 |
| R-10 | Upstream watch uses non-public or unclear sources. | Race Mode | high | medium | Public snapshot requirement, unknown-source watch-only mode, Source Hygiene Gate. | `tulingresearch-race-upstream-watch` | BL-16 |
| R-11 | Idea scoring promotes unsafe sources. | Race Mode | high | low | Source hygiene downgrade rules and priority regression tests. | `tulingresearch-race-priority-elevator` | BL-17 |
| R-12 | Feature capsule workflow creates implementation tasks too early. | Race Mode | medium | medium | Require passed hygiene, contract/test/docs/SOP skeleton, done criteria. | `tulingresearch-race-feature-capsule-factory` | BL-18 |
| R-13 | Figure captions imply unsupported conclusions. | Paper Pipeline | medium | medium | Require figure provenance and evidence-linked captions. | `tulingresearch-paper-figure-asset-pipeline` | BL-19 |
| R-14 | Method section becomes ready without figures. | Paper Pipeline | medium | low | Section status gate and missing figure tests. | `tulingresearch-paper-docflow-article-blocks` | BL-20, BL-21 |
| R-15 | Examples require API keys by accident. | QA / Examples | high | medium | Fake-mode examples remain default; live examples marked manual/live. | `tulingresearch-qa-release` | BL-22 |

## Risk Review Cadence

- Review before each v0.2.0 milestone starts.
- Update severity after implementation tests are added.
- Any critical risk must become a release blocker until mitigated.
