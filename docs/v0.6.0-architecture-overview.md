# v0.6 Architecture Overview

Status: consolidated architecture map.

v0.6 organizes TuringResearch Plus as a local research operating system layer.
It remains fake/default, review-first, and no-network by default.

## Layers

```text
Project Template
-> Workspace Registry
-> Evidence / Artifact / Route Surfaces
-> Cross-project Graph
-> Privacy Gate
-> Paper Assembly
-> Dashboard / Replay
-> Plugin / Skill / Capability Metadata
-> Quality Regression Gate
```

## Workspace Layer

The workspace layer indexes multiple research projects. It can summarize
project status and missing paths, but it is not an evidence source.

Key docs:

- [Multi-project Workspace](multi-project-workspace.md)
- [General Research Project Template](general-research-project-template.md)
- [Cross-project Evidence Graph](cross-project-evidence-graph.md)

## Evidence And Artifact Layer

Evidence and artifact layers preserve provenance and review status. Remote
artifact surfaces produce metadata and proposed imports, not verified evidence.

Key docs:

- [Artifact Auditor](artifact-auditor.md)
- [Remote Artifact Integration](remote-artifact-integration.md)
- [Privacy / Data Policy Layer](privacy-data-policy-layer.md)

## Paper Assembly Layer

Paper assembly builds section skeletons and gate reports. It blocks final paper
generation when citations, real results, or evidence are missing.

Key docs:

- [Paper Writing Scaffold](paper-writing-scaffold.md)
- [Method Section Builder](method-section-builder.md)
- [Related Work Draft Assistant](related-work-draft-assistant.md)
- [Experiment Section Builder](experiment-section-builder.md)
- [Paper Assembly Gate](paper-assembly-gate.md)

## Plugin And Skill Layer

The plugin system is manifest-only. It validates metadata and safety reports,
but it does not execute third-party code or dynamically load entrypoints.

Key docs:

- [Plugin Architecture](plugin-architecture.md)
- [MCP Plugin Registry](mcp-plugin-registry.md)
- [Tool Capability Manifest](tool-capability-manifest.md)
- [Skill Marketplace Layout](skill-marketplace-layout.md)
- [Extension Safety Gate](extension-safety-gate.md)

## Replay And Quality Layer

Replay and quality gates check fake/default outputs and release readiness
signals. They do not run real experiments.

Key docs:

- [Benchmark Replay Suite](benchmark-replay-suite.md)
- [Quality Metrics](quality-metrics.md)
- [Regression Gate](regression-gate.md)
- [v0.6 Full Replay Report](v0.6.0-full-replay-report.md)
- [v0.6 Regression Report](v0.6.0-regression-report.md)

## Non-goals

- No SaaS user system.
- No default network access.
- No automatic remote experiment execution.
- No automatic final paper writing.
- No private data upload.
- No demo result promoted to observed evidence.
