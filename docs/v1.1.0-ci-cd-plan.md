# v1.1.0 CI/CD Plan

Status: planning only.

Round: 226.

This plan defines the v1.1 CI/CD direction for TuringResearch. It does not
publish packages, create GitHub releases, create tags, run live tests, upload
secrets, or create child repositories.

## Goals

- Keep the default CI fake/demo safe.
- Make lint, unit, contract, workflow, docs, privacy, and name integrity checks
  visible before release review.
- Keep live adapters and live tests opt-in.
- Keep release draft and tag operations manual.
- Reduce dirty-branch release risk without adding complex automation.

## Current Baseline

The repository already has lightweight GitHub Actions for lint and tests:

- `.github/workflows/lint.yml`
- `.github/workflows/test.yml`

The v1.1 plan should treat those as a base, not as a full release automation
system.

## Proposed CI Jobs

| Job | Purpose | Default |
| --- | --- | --- |
| lint | `python -m ruff check .` | required |
| mypy | `python -m mypy src` | required for release branch, optional while iterating |
| unit tests | `tests/unit` | required |
| contract tests | `tests/contract` | required |
| workflow tests | `tests/workflow` fake/default only | required |
| name integrity | old-name prevention | required |
| privacy/security gate | public release safety | required |
| docs checks | docs-site build and README/doc links | required |
| live tests | live adapters | skipped by default |
| release draft | manual checklist only | manual approval |

## Release Branch Gate

Release branches should pass:

```powershell
python -m ruff check .
python -m pytest
python -m mypy src
python -m pytest tests/contract/test_name_integrity.py -q
python -m pytest tests/contract/test_public_release_hygiene.py -q
python -m pytest tests/workflow/test_public_demo_privacy_gate.py -q
```

Additional focused gates can be added for v1.1:

```powershell
python -m pytest tests/workflow/test_v1_1_paper_demo_integration.py -q
python -m pytest tests/workflow/test_v1_1_docs_dashboard_integration.py -q
python -m pytest tests/workflow/test_v1_1_split_ready_bundles.py -q
```

## Live Test Policy

Live tests remain skipped unless all are true:

- the test is explicitly marked live;
- secrets are provided by the operator outside the repository;
- `TURINGRESEARCH_ENABLE_LIVE_TESTS=1` is set;
- the run is not a public release default gate.

## Release Automation Boundary

Allowed in v1.1:

- generate release checklist text;
- verify version files;
- verify changelog/release notes presence;
- verify release draft exists;
- create a manual release readiness report.

Not allowed in v1.1:

- automatic PyPI publish;
- automatic GitHub release creation;
- automatic tag creation;
- automatic child repository creation;
- upload of secrets or private artifacts.

## Human Approval Required

Before any tag or release:

1. Review README.
2. Review release notes.
3. Review public demo and case gallery.
4. Review privacy/security scan results.
5. Review known limitations.
6. Confirm release branch.
7. Manually approve tag/release operation.
