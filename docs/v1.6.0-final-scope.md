# v1.6.0 Final Scope

Status: scope locked.

Round: 361.

v1.6 is scoped as **Public Release Execution Pack**. It follows the v1.5 final
handoff and focuses on making the existing work genuinely ready for public
manual release, installation, demonstration, and review. It is not a new
feature-expansion release.

This round locks scope only. It does not deploy docs, create repositories,
publish releases, build distribution artifacts, run live providers, execute
remote commands, or implement ARIS.

## Locked v1.6 Scope

1. Docs deployment ready.
2. GitHub Pages-ready workflow.
3. Split repo manual execution pack.
4. Optional live smoke ready.
5. Package / install readiness.
6. Release artifact build.
7. Public launch checklist.
8. Screenshot / demo asset pack.
9. v1.6 full regression.
10. v1.7 roadmap.

## Scope Interpretation

- Docs deployment ready means final static output checks, URL insertion policy,
  rollback notes, and post-deployment verification. It does not mean automatic
  deployment.
- GitHub Pages-ready workflow means a reviewed workflow plan or template for a
  maintainer to approve. It does not mean publishing a site in this round.
- Split repo manual execution pack means final human-ready commands,
  checklists, manifests, and README/link checks. It does not mean automatic
  GitHub repository creation or external push.
- Optional live smoke ready means private opt-in smoke check documentation,
  skipped tests, credential policy, and review gates. It does not make
  networking, SSH/SFTP, or providers default.
- Package / install readiness means local install, package metadata, import
  surface, CLI smoke, and release contract checks.
- Release artifact build means building or documenting reproducible local
  artifacts only after safety checks. It does not mean PyPI publication.
- Public launch checklist means manual release and post-release verification
  steps, not an automated launch.
- Screenshot / demo asset pack means public-safe visual/demo assets. It does
  not include private data, raw data, or research-result claims.
- v1.6 full regression means release-path tests and safety gates, not new
  runtime capability.
- v1.7 roadmap means planning the next slice after v1.6, not starting it.

## Release Posture

v1.6 should make TuringResearch easier to install, inspect, present, and
manually publish. Every real public action remains maintainer-controlled.

## ARIS Boundary

ARIS remains future reference only. v1.6 does not implement:

- ARIS runtime;
- cross-model review;
- proof-checker;
- meta-optimize;
- paper-claim-audit;
- automated research loops;
- automatic evidence promotion from ARIS-like reports.

If ARIS returns, it should be a separately scoped study-only track with its own
safety gate and disabled-by-default behavior.
