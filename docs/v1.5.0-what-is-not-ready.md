# v1.5.0 What Is Not Ready

Status: release-candidate boundary inventory.

Round: 360.

This file lists what v1.5 deliberately does not claim as ready. These items
require separate human approval, future scope locks, or later implementation.

## Not Published or Deployed

Not ready as completed public actions:

- public docs deployment;
- real public docs URL;
- analytics configuration;
- hosted dashboard service;
- external static hosting release.

v1.5 only provides a local docs deployment dry-run and static/local-first
showcase pages.

## Not Split Into External Repositories

Not ready as completed repository actions:

- automatic child repository creation;
- external remote push;
- real child-repo URL;
- first child-repo release;
- child-repo issue template setup.

v1.5 only provides manual packs and human review checklists.

## Not Live by Default

Not ready as default behavior:

- live Scholar provider calls;
- live Web / Apify calls;
- live SFTP transfer;
- live SSH connection;
- remote command execution;
- remote delete;
- live output promoted to observed evidence.

v1.5 only documents optional live behavior and keeps live tests skipped by
default.

## Not a Research Result Claim

Not ready as evidence claims:

- no experiment success claim;
- no automatic experiment execution;
- no remote run result claim;
- no fake/demo output treated as verified evidence;
- no paper citation verification from fake data;
- no sparse-convolution experiment success claim without reviewed evidence.

Human review and real evidence remain required before any research claim.

## Not ARIS

Not implemented in v1.5:

- ARIS runtime;
- cross-model review;
- proof-checker;
- meta-optimize;
- paper-claim-audit;
- automated research loop;
- autonomous paper-writing or resubmission machinery.

ARIS remains future-study only.

## Not a Clean Release Branch

The current working tree is historically dirty. v1.5 is not ready for release
publication until a human creates or checks out a clean release branch, commits
only reviewed changes, and reruns release checks.
