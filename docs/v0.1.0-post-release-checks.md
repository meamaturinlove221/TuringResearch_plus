# TulingResearch Plus v0.1.0 Post-Release Checks

Run these checks after a maintainer publishes the release. They are not publishing commands.

## Clean Clone Install

```powershell
git clone <repo-url> TulingResearch_plus-smoke
cd TulingResearch_plus-smoke
python -m venv .venv-smoke
.\.venv-smoke\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
python -m pytest
```

## Import Package

```powershell
python -c "import tuling_research, tuling_research.pdf, tuling_research_plus"
python -c "import tuling_research_plus.artifacts, tuling_research_plus.campaign, tuling_research_plus.race, tuling_research_plus.paper"
```

## MCP Smoke

```powershell
python -m tuling_research.mcp_server --manifest
python -m tuling_research.mcp_server --health-check
```

Expected:

- Server name is `tulingresearch-plus`.
- No server starts on import.
- JSON output is explicit for manifest and health check.

## Run Fake Examples

```powershell
python -m pytest tests/workflow
```

Expected:

- VGGT human-prior survey example passes.
- SMPL-X feature-adapter hypothesis example passes.
- Citation graph demo passes.
- PDF-to-Markdown demo passes.
- No real API key or live network access is required.

Confirm the public repository does not include:

- `.env` with real values.
- API keys.
- Private papers.
- Restricted datasets.
- `private_data/`
- `secrets/`
- `*.pem`
- `*.key`

## Post-Release Decision

If any post-release check fails, open a release-blocker issue and consider a `v0.1.1` patch release. Do not silently change release artifacts without recording the fix.
