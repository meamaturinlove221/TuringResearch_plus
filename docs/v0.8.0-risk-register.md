# v0.8.0 Risk Register

Status: planning draft.

Round: 149.

| Risk | Severity | Affected candidate | Mitigation |
| --- | --- | --- | --- |
| Local dashboard exposes private project files | Critical | Local server dashboard | Default to explicit project roots, privacy scan dashboard inputs, and keep static export fallback. |
| Local dashboard is mistaken for SaaS | Medium | Local server dashboard | Document local-only scope, no account system, no hosted service, and no default external bind. |
| Paper beta writes unsupported conclusions | Critical | Research paper writing beta | Keep claim, evidence, citation, and result gates; block final abstract/results/conclusion without human approval. |
| Fake/demo evidence is promoted to observed | Critical | Paper beta, case studies, demos | Require status labels and regression tests for planned/demo/not-enough-evidence. |
| Public plugin registry implies safe execution | Critical | Public plugin registry draft | Registry metadata only; disabled-by-default; compatibility and safety reports required. |
| OS sandbox research creates false confidence | High | OS-level plugin sandbox research | Separate research notes, policy model, and runtime enforcement; do not execute unknown code. |
| Plugin sandbox bypass through shell/network/secrets | Critical | OS-level plugin sandbox research | Deny shell/secrets by default, require explicit network/live flags, and test permission gates. |
| Case study leaks private data or advisor feedback | Critical | More case studies | Use privacy scan, redaction report, compliance review, and human approval before publication. |
| Compliance expansion is interpreted as legal advice | High | Dataset/license compliance expansion | Prominent disclaimers, missing-license flags, and legal/human review markers. |
| Export polish introduces heavy mandatory dependencies | Medium | Real PPTX/PDF polish | Keep optional backends, graceful skip, and source bundle fallback. |
| MCP marketplace polish destabilizes tool surface | High | MCP marketplace polish | Keep manifest/metadata first, compatibility tests, and no automatic install. |
| Multi-project analytics transfers evidence across projects | High | Multi-project analytics | Analyze patterns only; never apply one project evidence to another project. |
| Collaboration workflow uploads private data | Critical | Collaboration workflow | Local-first review queues only until explicit privacy and sync policy exists. |

## Release Blockers

- Any real secret, token, or API key committed.
- Any raw dataset, private model payload, or large private artifact committed.
- Any private local project path in public examples.
- Any unknown third-party plugin enabled by default.
- Any plugin execution without an approved runtime sandbox gate.
- Any paper conclusion generated without human review.
- Any fake/demo result marked as observed.
- Any case-study success claim without evidence support.
- Any license/compliance statement presented as legal advice.
- Any prior project naming in v0.8 files.

## Watch Items

- Browser and local server security defaults.
- Optional PDF/PPTX backend dependency licenses.
- Plugin metadata trust and signing strategy.
- Public registry governance and review status.
- Case-study redaction quality across new project types.
- Paper beta copyright and citation-safety boundaries.
