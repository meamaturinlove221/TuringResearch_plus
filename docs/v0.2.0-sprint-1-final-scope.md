# TulingResearch Plus v0.2.0 Sprint 1 Final Scope

Round 36 freezes the planning inputs for v0.2.0 Sprint 1 after assimilating the
VGGT dogfooding branch into `dev/mainline`. This scope is planning-only until a
later implementation round starts.

## Sprint Goal

Build the local evidence infrastructure needed for VGGT / SMPL-X dogfooding
without overclaiming experiment status, promotion readiness, or advisor
acceptance.

## Selected Items

| Rank | Item | Sprint status | Primary owner skill | Scope |
| ---: | --- | --- | --- | --- |
| 1 | Artifact Auditor | planned | `tulingresearch-race-architecture-box-builder` / `tulingresearch-fusion-wiki-vault` | Index local bundles, zip/json/csv/png/npz artifacts, sidecars, board inventories, checksums, and missing files. |
| 2 | Visual Evidence Auditor | planned | `tulingresearch-paper-figure-asset-pipeline` | Classify proxy heatmaps, mask/delta boards, full pointcloud proxies, and true region pointcloud closeups without claiming advisor approval. |
| 3 | VGGT/SMPL-X Evidence Ledger | planned | `tulingresearch-cache-and-ledger` | Define status-safe evidence rows for V770/V129/V260/V900/V930/V999/V120/V121 and preserve observed versus local-observed boundaries. |
| 4 | Advisor Pack Builder | planned | `tulingresearch-paper-writing-pipeline` | Build advisor-readable packs from evidence ledger and auditor outputs, with no promotion wording and no fabricated results. |
| 5 | PDF Phase B Figure/Table Extraction | planned | `tulingresearch-pdf-markdown-core` | Extract figure/table metadata from local fixture PDFs and align outputs with paper figure registry provenance. |

## Required Boundaries

- No real VGGT experiment execution.
- No writes into VGGT project directories.
- No reading of private VGGT local paths unless the user provides local opt-in
  config in the working environment.
- No fabricated VGGT, Modal, SparseConv3D, or advisor results.
- No live network dependency in default tests.
- No public API change unless contracts, docs, and tests are updated first.

## Evidence Status Rules

| Evidence label | Meaning in Sprint 1 |
| --- | --- |
| `observed` | User-provided or document-observed engineering context. Not enough for local confirmation. |
| `local-observed` | Confirmed by committed local scan summary/index files. |
| `requires-human-review` | Needs manual review or missing local evidence before being treated as local evidence. |
| `review-ready-proxy` | A possible future label for assembled proxy evidence. It is not promotion. |
| `missing` | Required file or artifact is absent. |
