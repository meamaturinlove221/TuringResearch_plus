# Internship Portfolio Positioning

Status: planning policy.

Round: 151.

TuringResearch can serve as a strong internship and portfolio project if the
repository strategy keeps the story coherent: one flagship research OS with
clear safety gates, local-first design, public demos, and evidence-aware
workflows.

## Portfolio Message

TuringResearch demonstrates:

- product architecture for a local-first research OS;
- Python package and MCP distribution design;
- test-driven workflow modules;
- privacy, compliance, and release-gate thinking;
- plugin architecture and sandbox policy design;
- dashboard/export/documentation polish;
- research case-study discipline without result overclaiming.

## Why Monorepo Helps Short Term

- Reviewers can see the whole system in one place.
- The README can tell a complete story.
- Tests and docs show sustained engineering discipline.
- Public demos and case studies show product thinking beyond code.
- Stars and issues concentrate on one visible project.

## When Multirepo Helps Later

Multirepo can help when each satellite becomes a polished portfolio artifact:

- a case-study repository with a clean narrative;
- an examples repository with demo-safe walkthroughs;
- a plugin repository with registry and compatibility reports;
- a dashboard repository with a focused local UI story.

## What To Show Publicly

- The flagship repo README and quickstart.
- Public demo expansion.
- VGGT dogfooding case summary with redaction and claim safety.
- Dashboard/export examples.
- Plugin safety and compatibility reports.
- Test and release gate summaries.
- Roadmap honesty and known limitations.

## What Not To Show

- Private local paths.
- Raw datasets.
- Licensed/private model payloads.
- Advisor-private feedback.
- Fake results marked as real.
- Unsupported experiment success claims.
- Unreviewed third-party plugin execution.

## Recommended Public Narrative

Lead with:

> TuringResearch is a local-first research OS for organizing evidence, artifacts,
> papers, dashboards, advisor communication, plugins, and release gates.

Then show:

1. public demo;
2. workspace and dashboard;
3. paper/review workflow;
4. plugin safety;
5. case-study discipline;
6. maintenance and release gates.

## Split Timing For Portfolio

Short term:

- keep the flagship repo as the main portfolio artifact;
- improve docs and demo clarity;
- avoid fragmenting stars and review attention.

Long term:

- split case studies or examples only when they are polished enough to be
  standalone artifacts;
- keep every satellite linked back to the flagship.
