# TuringResearch Plus v0.2.0 Sprint 1 Retrospective

## What Worked

- Contracts-first boundaries kept missing evidence visible.
- The evidence ledger prevented planned, observed, local-observed, hard-blocked,
  and not-enough-evidence states from collapsing into one bucket.
- Advisor Pack Builder produced a useful human-readable summary without
  fabricating results.
- PDF Phase B stayed lightweight and testable with generated fixture PDFs.

## What Did Not Fully Land

- Visual Evidence Auditor remains a dry-run artifact surface, not a full
  classifier.
- The committed VGGT example still lacks `local_scan_evidence_ledger.json` and
  `local_scan_visual_inventory.md`.
- Upstream learning inputs requested for this planning round were not present in
  the checkout.

## Process Lessons

- Keep VGGT-local path access explicit and machine-private.
- Continue writing reports before implementation when evidence is weak.
- Prefer small deterministic workflow tests over broad live tests.
- Make every missing artifact a first-class output.

## Carry Forward

- Promote reusable hard gates into a shared library.
- Make experiment route descriptions explicit enough to test.
- Add a failure taxonomy so advisor packs and evidence ledgers use the same
  failure vocabulary.
- Convert real papers into method cards only when provenance is available.
