# TuringResearch Plus v0.2.0 Sprint 2 Recommendation

## Recommendation

Sprint 2 should focus on experiment semantics rather than live adapters. The
recommended top five are:

1. Hard Gate Library
2. Experiment Route DSL
3. Failure Taxonomy Engine
4. Paper-to-Method Card
5. Figure-to-Architecture

## Implementation Order

1. Hard Gate Library
2. Experiment Route DSL
3. Failure Taxonomy Engine
4. Paper-to-Method Card
5. Figure-to-Architecture

## Why This Order

- Hard Gate Library centralizes the gates already appearing in evidence,
  visual, advisor, paper, and PDF workflows.
- Experiment Route DSL gives VGGT runs a structured plan before any Modal or
  SparseConv3D ingestion happens.
- Failure Taxonomy Engine gives consistent language for blockers, regressions,
  route failures, and not-ready claims.
- Paper-to-Method Card can use PDF Phase B outputs without claiming paper
  results.
- Figure-to-Architecture builds on PDF figures and method cards, but should
  wait until cards have stable provenance.

## Required Contracts

- `contracts/hard_gates.yaml`
- `contracts/experiment_routes.yaml`
- `contracts/failure_taxonomy.yaml`
- `contracts/method_cards.yaml`
- `contracts/figure_architecture.yaml`

## Required Tests

- Gate evaluation with pass/block/requires-human-review.
- Route DSL parsing and JSON/Markdown serialization.
- Failure taxonomy normalization and severity mapping.
- Method card generation from local paper metadata and PDF Phase B assets.
- Figure-to-Architecture dry-run with no fabricated diagram claims.
- Regression tests that keep SparseConv3D success unclaimed without evidence.

## Risks And Mitigations

| Risk | Mitigation |
| --- | --- |
| Route DSL becomes too broad. | Keep the first version VGGT-focused and fixture-tested. |
| Gates duplicate existing checks. | Extract only repeated gates from Sprint 1 reports and tests. |
| Failure taxonomy becomes subjective. | Use explicit severity/status mappings and require examples. |
| Method cards overstate paper claims. | Require provenance and `requires-real-paper` when inputs are incomplete. |
| Figure-to-Architecture invents diagrams. | Treat output as draft mapping only unless source figures and method cards support it. |

## Deferred

- Live Semantic Scholar adapter.
- Real citation graph expansion.
- NAS/SSH/GitHub sync.
- Large UI work.
- OCR heavy pipeline.
