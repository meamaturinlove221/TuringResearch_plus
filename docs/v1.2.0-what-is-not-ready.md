# v1.2.0 What Is Not Ready

Status: limitation summary.

Round: 258.

## Not Ready For v1.2

- ARIS runtime implementation.
- Full ARIS merge.
- Cross-model review production runtime.
- Proof-checker production runtime.
- Meta-optimize runtime.
- Automatic paper writing.
- Automatic experiment execution.
- Remote execution orchestration.
- SSH, tmux, or provision workflow.
- MinerU or heavy PDF fallback.
- Default live networking.
- OS-level plugin sandbox.
- Public plugin marketplace.
- Automatic child repository creation.
- Automatic GitHub release or PyPI publish.

## Not Ready Without Human Review

- Release branch push.
- Tag creation.
- GitHub release.
- Public claims about research outcomes.
- Live provider runs.
- Public screenshots or GIFs.
- Any v1.3 ARIS prototype.

## Why

These items either require stronger safety design, live credentials, legal or
privacy review, dependency review, or a separate future scope lock. Keeping them
out of v1.2 protects the runnable mainline.
