# v1.6.0 Implementation Order

Status: scope locked order.

Round: 361.

This order keeps v1.6 focused on public release readiness. It intentionally
places real public actions behind review gates.

## Recommended Order

1. Docs deployment readiness.
   - Final scan of `docs-site/dist/`.
   - URL insertion policy.
   - GitHub Pages workflow draft.
   - Rollback and post-deployment verification plan.

2. Package / install readiness.
   - Local install smoke.
   - Package metadata review.
   - CLI entrypoint smoke.
   - Public import surface and release contract checks.

3. Release artifact build.
   - Local artifact build plan.
   - Artifact manifest and checksums.
   - No publish by default.
   - Manual PyPI decision deferred to maintainer approval.

4. Split repo manual execution pack.
   - Final review of `split_manual/`.
   - Child-repo README backlink checks.
   - Manual command checklist.
   - No automatic repository creation.

5. Optional live smoke ready.
   - Scholar/Web/SFTP private smoke policy.
   - Env-only credential checks.
   - Live tests skipped by default.
   - No remote command execution.

6. Screenshot / demo asset pack.
   - Public-safe static screenshots or demo assets.
   - No private paths, raw data, credentials, or observed-result claims.
   - Clear fake/default and local-first labels.

7. Public launch checklist.
   - Manual GitHub release checklist.
   - Manual docs deployment checklist.
   - Manual split repo checklist.
   - Post-release verification checklist.

8. v1.6 full regression.
   - v1.5 baseline replay.
   - v1.6 release readiness gates.
   - Security/privacy gate.
   - Name integrity and public hygiene.

9. v1.7 roadmap.
   - Decide whether to proceed with real docs deployment, physical split repo
     creation, optional live smoke expansion, dashboard polish, or plugin
     polish.
   - Keep ARIS study separate.

## Ordering Rationale

Install/readiness and docs deployment checks should happen before any real
public action. Split repo execution and optional live smoke depend on the same
release safety posture, so they stay behind manual approval. ARIS remains out
of this order.
