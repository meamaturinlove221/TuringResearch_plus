## Integrated Capability Surface

### Paper Intelligence

The fake Semantic Scholar adapter feeds Citation Graph Expansion, which can be
used with Paper-to-Method Card and Paper Collision Risk Detector. The output
supports JSON and Markdown. Results are not human-verified by default and must
not be treated as final related-work conclusions.

Verified path:

`FakeSemanticScholarAdapter -> CitationGraph -> PaperMethodCard -> CollisionRiskReport -> SafeClaims`

### Experiment Intelligence

The route DSL, hard gate library, run ingestor, failure taxonomy, and evidence
ledger proposed updates form a conservative planning-and-review chain.

Verified path:

`ExperimentRouteSpec -> HardGateValidationReport -> RunIngestReport -> FailureAttributionReport -> EvidenceLedger proposed updates`

Route compilation is planning only. Hard gate validation failures are plan or
package validation results, not proof of a real experiment failure.

### Artifact Handoff

Artifact audit and handoff import/export are connected through local review
artifacts. Handoff import validates manifest and sha256 and produces proposed
updates only.

Verified path:

`ArtifactAuditReport -> HandoffBundle -> HandoffImportReport -> proposed evidence updates`

Raw datasets, secrets, cache folders, and SMPL-X body model files are excluded.

### Advisor Intelligence

Advisor Pack inputs can reference evidence ledger rows, artifact audit reports,
visual dry-run status, failure taxonomy reports, collision risk reports, and
handoff import reports. Round 61 does not regenerate a full Advisor Pack.

### VGGT Route

The VGGT dogfooding plan and Modal SparseConv route pack are connected to run
ingest fixtures, failure taxonomy, and next-action generation. The current
state remains review-oriented and does not claim SparseConv3D success.
