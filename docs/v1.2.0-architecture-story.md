# v1.2.0 Architecture Story

Status: interview narrative.

Round: 253.

## Short Version

TuringResearch is built as a local-first Research OS, not a single prompt or an
autonomous agent runtime. The architecture keeps research state explicit:
workspaces, evidence, artifacts, routes, papers, dashboards, plugins, privacy
checks, and release gates are separate reviewable surfaces.

v1.2 strengthened that architecture by aligning stable ideas from the original
reference repositories while keeping risky automation outside the mainline.

## Architecture Layers

| Layer | Interview explanation |
| --- | --- |
| Workspace | Keeps multi-project research context organized locally. |
| Evidence and artifacts | Separates claims from files and required proof. |
| Route DSL and gates | Makes experiment plans, blockers, and safety checks explicit. |
| Paper and advisor surfaces | Builds review material without final-paper automation. |
| Dashboard and docs | Makes the project inspectable for GitHub and interviews. |
| Plugin and MCP policy | Keeps optional tools disabled or fake/default unless explicitly enabled. |
| Reference parity | Pulls stable ideas into tested local modules, not copied upstream code. |

## v1.2 Architectural Upgrade

v1.2 added parity in five practical areas:

- context handoff and structured return safety;
- scholar/web usage and MCP config clarity;
- skill SOP routing;
- campaign, vault, ontology, stress-test, and runbook organization;
- public demo and replay surfaces that show the system without private data.

These are boring in the right way: they make the system easier to run, explain,
test, and audit.

## How It Still Runs

The v1.2 architecture avoids fragile runtime expansion:

- fake/demo paths remain default;
- live adapters remain optional;
- no unknown remote execution is introduced;
- no agent runtime replaces the master orchestrator;
- no demo output becomes a verified live claim;
- tests cover the public demo, parity dashboard, research catalog, and full
  fake replay.

## How To Say It In An Interview

"The key architecture choice was to make research workflows inspectable before
making them autonomous. v1.2 is mostly parity and gates: I aligned stable
reference ideas, wrote contracts and tests around them, and deferred the more
ambitious ARIS loops until the base system had a stronger safety and regression
story."
