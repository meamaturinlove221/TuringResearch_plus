# v1.1.0 Release Notes

Status: release-candidate notes.

Round: 229.

TuringResearch Plus v1.1.0 is a post-v1.0 stabilization release-candidate line.
It keeps the project local-first and fake/demo-first while improving the public
entry path, split-ready bundles, docs/dashboard surfaces, paper writing beta,
demo coverage, and CI checks.

This release does not publish a SaaS product, does not execute experiments
automatically, does not create child repositories automatically, and does not
turn demo output into observed research evidence.

## Highlights

- Post-v1.0 baseline stabilization and public entry cleanup.
- Split-ready case/examples bundles prepared for human repository creation.
- Local-first docs site skeleton and static docs-site builder.
- Read-only localhost dashboard.
- Read-only Dashboard Data API with JSON export.
- Paper writing beta for review-only draft packages.
- More public demo cases across robotics, medical imaging, software tooling,
  and multimodal evaluation planning.
- Case Study Gallery for browsing public-safe cases.
- CI/CD release automation plan.
- GitHub Actions hardening for CI, docs checks, and privacy gate.

## What Is New Since v1.0

- Better public README/docs entry points.
- Better issue and feedback templates.
- Better split-ready package preparation and cross-link policy.
- Better docs/dashboard integration.
- Better demo diversity beyond the VGGT case.
- Better release regression gate coverage.

## What Remains Review-gated

- Physical split repository creation.
- Local server dashboard usage beyond localhost read-only demo mode.
- Paper writing beta outputs.
- Live adapters and live tests.
- Plugin execution and third-party plugin proposals.
- Release branch push, tag creation, GitHub release, and PyPI publishing.

## Release Boundary

v1.1.0 remains a local-first Research OS release candidate. Human review is
required before publication, claims, external repository creation, or public
release actions.
