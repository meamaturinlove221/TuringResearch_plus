## Freeze Notes

- No entire experimental module is promoted to stable in Round 176.
- Stability is assigned to reviewed surfaces, not to every helper in a module.
- `turing_research_plus` remains the compatibility namespace.
- New namespace packages are facade exports, not independent repositories.

## Promotion Rules

An API surface may move toward stable only if:

- docs exist;
- tests exist;
- contract fields are explicit;
- fake/live boundary is clear;
- privacy and safety boundaries are documented;
- import compatibility is covered;
- no internal helper is required by public users.

## Demotion Rules

An API surface must remain experimental or internal if:

- it depends on case-specific fixtures;
- it depends on optional live providers;
- it executes or loads unknown code;
- it has unclear ownership;
- it lacks contract tests;
- it may leak private data or unsupported claims.
