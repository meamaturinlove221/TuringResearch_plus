the baseline for v1.1 planning and execution. It does not implement features,

## Carryover Risks

- no verified `release/v1.0.0-rc` branch;
- current local branch is `pre-rename-round38-checkpoint`;
- working tree remains historically dirty;
- public launch still requires human README/release/demo review;
- split repositories are local bundles only, not published child repos;
- security/privacy gates are pattern-based and need final human review;
- fake/live boundary may need more onboarding clarity after real feedback;
- public screenshots/GIFs still need to be prepared from public-safe surfaces.

## v1.1 Priority Adjustments

The v1.1 roadmap remains directionally correct, but Round 204 makes one
ordering constraint explicit:

1. Stabilize the post-v1 branch/release baseline before creating public child
   repos.
2. Keep `turingresearch-vggt-case` as the first split candidate after human
   approval.
3. Start public docs site planning in parallel with release-baseline cleanup.
4. Keep local server dashboard and paper writing beta behind safety scope docs.
5. Do not begin broad plugin ecosystem work until release and review capacity
   are cleaner.

## No-go Blockers

These block treating v1.0 as a published baseline:

- release branch not verified;
- release branch not pushed;
- maintainer release approval missing;
- final README/release/public-demo review missing;
- final license posture review missing;
- final tag/GitHub release decision missing.
