# v0.5.0 Beta Risk Register

Status: planning.

Round: 100.

| Risk | Level | Mitigation |
| --- | --- | --- |
| Binary export makes draft look final | high | Preserve limitations, source refs, and requires-human-review markers in every export. |
| Public demo accidentally includes private data | high | Run secret/path/model-file scans before every release-prep step. |
| Dashboard appears like live experiment UI | medium | Keep static/review-only labels visible. |
| MCP install docs drift from pyproject entrypoints | medium | Add contract tests for CLI and MCP examples. |
| Plugin architecture expands scope too early | high | Defer until public hardening and install flow are stable. |
| Real VGGT replay gets treated as new result | high | Require new artifacts and keep replay labels explicit. |
| Live adapters become default by accident | medium | Keep live tests skipped by default and env-gated. |
| Documentation overclaims paper conclusions | medium | Keep related-work and paper outputs review-required. |

## Current Risk Posture

Beta is viable if it stays release-hardening-first. The highest-risk items are
binary export and real artifact replay; both need strict evidence and safety
gates before promotion.
