# v0.5.0 Roadmap

Status: planning draft.

## Candidate Directions

1. Lightweight Web UI / Dashboard
2. Real Advisor PDF/PPTX Export
3. Modal Run Dashboard UI
4. Project Template Generator
5. Public Release Hardening
6. MCP Marketplace / package polish
7. Benchmark / demo suite
8. Real VGGT dogfooding replay

## Recommended Sprint 1 Focus

If the next phase prioritizes usability, start with a lightweight local dashboard
that renders existing Markdown/JSON reports without introducing a heavy UI.

If the next phase prioritizes advisor communication, start with real PDF/PPTX
export from the v0.4 Markdown bundle while preserving limitations and evidence
boundaries.

If the next phase prioritizes public release quality, start with release
hardening, package polish, MCP registry consistency, and a benchmark/demo suite.

## Non-Goals Until Explicitly Scoped

- No default live remote sync.
- No remote command execution.
- No automatic VGGT experiment execution.
- No final paper writing.
- No paywall bypass.
- No raw data or SMPL-X model packaging.
