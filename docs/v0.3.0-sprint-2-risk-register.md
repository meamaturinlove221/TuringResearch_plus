# v0.3.0 Sprint 2 Risk Register

Status: planning.

Round 70 keeps this risk register active as the Sprint 2 scope-lock register.

## R1 - Default Network Leakage

- Area: Web Fetch / Apify
- Risk: implementation accidentally reaches network during default tests.
- Mitigation: fake adapter by default, explicit live markers, no API key required.

## R2 - Source Hygiene Bypass

- Area: Web Fetch / Apify
- Risk: adapter fetches restricted, login-only, paywalled, or private content.
- Mitigation: source hygiene gate, blocklist rules, live test policy, manual review.

## R3 - Retrieved Means Verified Confusion

- Area: Web content, related work positioning
- Risk: fetched or cached content is treated as human-verified evidence.
- Mitigation: source metadata must carry `human_verified: false`; positioning output
  must require human review.

## R4 - Related Work Overclaiming

- Area: Related Work Positioning Generator
- Risk: generator produces definitive novelty or no-collision claims.
- Mitigation: reuse Collision Risk Detector conservative safe/unsafe claims.

## R5 - Skill Routing Drift

- Area: ENTRY.md / Routing Table
- Risk: routing table references missing skills, stale lanes, or old names.
- Mitigation: contract test for skill/lane existence and naming integrity.

## R6 - Vault Graph Noise

- Area: Vault Graph Enhancement
- Risk: graph gains edges without evidence or produces confusing traversal.
- Mitigation: edge audit, typed edge validation, missing evidence lint.

## R7 - Ontology SOP Sprawl

- Area: Knowledge Graph / Ontology SOPs
- Risk: SOPs grow too abstract without executable tests.
- Mitigation: each SOP requires fixture, expected graph diff, and markdown output.

## R8 - Premature Sync Expansion

- Area: NAS / SSH / GitHub sync
- Risk: planning drifts back into sync adapters before source and routing are stable.
- Mitigation: keep sync as deferred non-goal for Sprint 2.

## R9 - Web Content Cache Pollution

- Area: Web Fetch / Apify
- Risk: cached web content stores restricted material or loses provenance.
- Mitigation: require source URL, retrieval time, content hash, source hygiene status,
  and `human_verified: false`.

## R10 - Ontology Edge Overconfidence

- Area: Vault Graph / Ontology SOPs
- Risk: ontology edges are treated as facts without evidence.
- Mitigation: require evidence refs, confidence labels, and edge audit outputs.
