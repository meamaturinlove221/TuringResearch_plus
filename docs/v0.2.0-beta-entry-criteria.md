# TuringResearch Plus v0.2.0 Beta Entry Criteria

Status: proposed beta gate.

TuringResearch Plus can enter v0.2.0 beta only when the alpha state remains
green and beta scope decisions are explicit.

## Required Gates

- v0.2 alpha tests pass.
- Name integrity tests pass.
- Package import tests pass.
- Sprint 1 integration tests pass.
- Sprint 2 integration tests pass.
- No legacy project naming outside migration docs.
- No fake result is written as observed.
- Live adapters remain optional and disabled by default.
- Route pack status remains planned until real VGGT-side evidence arrives.
- Public MCP API changes are reflected in contracts, docs, and tests.

## Required Decisions

- Promote or defer Sprint 2 proposed tools in public MCP contracts.
- Keep Modal execution outside TuringResearch or define a separate ingestion-only
  contract.
- Define beta scope for live Semantic Scholar adapter.
- Decide whether Paper Collision Risk Detector enters beta.
- Decide whether Handoff Bundle Export / Import enters beta.

## Required Evidence

- Full pytest pass.
- `mypy src` pass.
- Name integrity pass.
- Package import pass.
- Contract suite pass.
- Workflow suite pass.
- Clean alpha issue ledger review.

## Explicit Non-Entry Conditions

Do not enter beta if:

- fake or planned artifacts are presented as observed results;
- SparseConv3D success is claimed without real evidence;
- live adapters run by default;
- package imports fail;
- legacy naming appears outside migration docs;
- route pack status is changed without VGGT-side artifacts.
