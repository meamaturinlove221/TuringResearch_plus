# v0.9.0 Non-goals

Status: planning boundary.

Round: 167.

## Non-goals

v0.9 split planning does not do the following:

- create real GitHub repositories;
- move code out of the monorepo by default;
- split `turingresearch-core`;
- split paper, artifact, dashboard, or plugin packages before API maturity;
- publish the VGGT case study;
- publish private VGGT files;
- bundle raw data;
- bundle SMPL-X or restricted model files;
- claim VGGT experiment success;
- claim SparseConv3D success;
- generate fake users, fake benchmarks, or fake adoption evidence;
- make live adapters default;
- execute plugins;
- relax privacy, compliance, or release gates.

## What v0.9 May Do

v0.9 may prepare:

- exact export manifests for `turingresearch-vggt-case`;
- exact export manifests for `turingresearch-examples`;
- standalone README drafts;
- safety checklists;
- privacy/compliance/claim-safety gates for export trees;
- clear links back to the flagship repo.

## Main Repo Rule

The flagship repository must remain useful on its own. A split is invalid if a
new repo makes the main repo harder to install, demo, explain, or star.
