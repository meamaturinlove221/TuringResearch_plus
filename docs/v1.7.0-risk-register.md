# v1.7.0 Risk Register

Status: planned risk register.

Round: 389.

| Risk | Impact | Mitigation |
| --- | --- | --- |
| real Pages deployment exposes private material | public leak | rerun privacy/name/no-secret gates on deployment bundle |
| fake URL enters README or docs | user confusion | insert real URL only after deployment exists |
| split repository created with unsafe files | public data leak | use split safety gate and human checklist before creation |
| PyPI package includes private or stale files | package leak / broken install | inspect wheel/sdist before upload |
| package rename breaks compatibility | user disruption | keep compatibility import path until migration plan is approved |
| optional live runner logs secrets | credential leak | require redaction and explicit private env |
| optional live runner becomes default | privacy/security regression | default live disabled and skipped tests preserved |
| plugin polish enables unknown execution | code execution risk | keep plugins review-only unless explicitly approved |
| dashboard polish adds analytics or external assets | privacy regression | static/local-first and no analytics by default |
| ARIS scope creep | runtime complexity and safety risk | study-only track, no implementation by default |

## Hard No-Go Conditions

- secrets or API keys in repo;
- private paths in public surfaces;
- raw data or restricted model payloads included;
- fake/demo output marked as observed evidence;
- live network enabled by default;
- remote command execution enabled by default;
- child repo URL written before the repo exists;
- PyPI publication without license/package review;
- ARIS implementation without a separate approved scope.
