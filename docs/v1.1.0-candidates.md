## Recommended Sprint-1 Set

The recommended first v1.1 sprint should focus on:

1. `turingresearch-vggt-case` physical split preparation/execution after human
   approval.
2. Public documentation site.
3. Local server dashboard scope and prototype plan.

Paper writing beta and more public demos should be prepared as follow-up work
unless sprint capacity is larger than expected.

## Deferred From Sprint 1

- `turingresearch-examples` physical split: wait until the case repo validates
  the spoke pattern.
- Plugin ecosystem expansion: wait for real contributor demand.
- Real web live mode polish: keep fake/default and explicit live opt-in.
- CI/CD release polish: important, but can be a supporting lane unless release
  branch friction becomes the bottleneck.

## Candidate Safety Baseline

All candidates must preserve:

- no private data;
- no secrets;
- no raw data;
- no restricted model payloads;
- no unsupported experiment success claims;
- no automatic experiment execution;
- no automatic final paper generation;
- no unknown plugin execution;
- flagship repository remains the main install and star target.
