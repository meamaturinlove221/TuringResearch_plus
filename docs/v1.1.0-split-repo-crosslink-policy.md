# v1.1.0 Split Repo Crosslink Policy

Status: policy.

Round: 211.

This policy defines how future split repositories should link back to the
flagship TuringResearch repository.

## Principle

The flagship repository remains the primary project. Split repositories are
optional spokes for case studies and examples.

## Required Crosslink Pattern

Every future spoke README should include, above the fold:

```text
Flagship placeholder: TuringResearch main repository URL goes here after human
publication approval.
```

Before a real repository exists, do not write a GitHub URL. After repository
creation and human approval, replace the placeholder with the approved flagship
URL.

## Required Message

Spoke repositories must say:

- the main repository remains the flagship;
- users should use and star the main repository;
- the spoke does not replace install, docs, release, quickstart, or public API;
- the spoke contains demo/case material only;
- no core framework source is included.

## Forbidden Crosslinks

- Do not link to nonexistent split repositories.
- Do not present a local `split_ready/` bundle as a published repo.
- Do not claim a spoke is the canonical package.
- Do not duplicate package install instructions in a way that shifts the main
  install path away from the flagship.

## Review Gate

Before publishing a spoke repository, reviewers must check:

- the flagship URL is real and approved;
- README still sends users back to the flagship;
- privacy and claim-safety sections remain visible;
- fake/demo boundaries are not weakened;
- no unsupported result claims were introduced.
