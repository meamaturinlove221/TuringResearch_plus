# v0.3 Sprint 1 Candidates

Recommended Sprint 1 theme: Git-based Context Handoff / Pod Workflow.

## Candidate Items

| Candidate | Priority | Why |
| --- | --- | --- |
| Git Context Bundle Manifest | P0 | Extends v0.2 handoff bundle with Git-friendly context metadata. |
| Pod Workflow Route Spec | P0 | Captures local -> pod -> local workflow without SSH teleport assumptions. |
| Durable Context Pack | P0 | Packages `CLAUDE.md`-style instructions and memory snapshots as explicit review artifacts. |
| Structured Pod Output Import | P1 | Consumes files pushed back via Git without touching private VGGT paths. |
| Dotfile / Secret Guard | P0 | Prevents `.env`, `.codex`, secrets, and raw data from entering context bundles. |
| Memory Sync Policy | P1 | Documents that memory is not bidirectionally synced by default. |
| Handoff Compatibility Check | P1 | Ensures v0.2 handoff bundles can be referenced or embedded safely. |

## Deferred From Sprint 1

- General NAS / SMB shared artifact store.
- SSH / SFTP remote artifact reader.
- GitHub Artifact Sync as a broad artifact-sync system.
- Cloud object storage.
- Large UI.

## Required Tests

- bundle manifest validation;
- no secret / dotfile leakage;
- pod output import produces proposed updates only;
- no bidirectional memory sync by default;
- fake/local-only workflow test;
- no old naming.
