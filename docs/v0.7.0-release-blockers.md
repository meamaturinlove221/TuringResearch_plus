# v0.7 Release Blockers

Status: blockers documented.

## Blocking Before Actual Public Release

- Clean release branch review is still required.
- Maintainer approval is still required.
- License posture must be confirmed before public distribution.
- Release notes and public README must be reviewed.
- Current local worktree contains many historical uncommitted changes and must
  not be pushed wholesale.

## Not Release Blockers For RC Review

- Optional PDF/PPTX backends may be absent when skipped reasons are recorded.
- Compliance findings may record restricted or unknown assets as long as the
  report does not claim legal approval.
- Paper assembly and deep review may remain blocked as long as no final paper
  conclusion is generated.
- Plugin loading may remain metadata-only as long as unknown code is not
  executed.

## Hard Blockers If Reintroduced

- Credential values.
- Private local paths.
- Data payloads.
- Private model payload files.
- Fake/demo results marked observed.
- Legacy project naming in new public files.
- Live networking enabled by default.
- Unknown plugin execution.
- Unsupported experiment success claims.

## Current Gate

No new hard blocker was added by Round 145. Public release still requires human
maintainer review.
