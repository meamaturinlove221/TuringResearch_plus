# v1.3.0 Sprint 1 Recommendation

Status: sprint recommendation.

Round: 257.

## Recommendation

Start v1.3 with ARIS study artifacts, not runtime implementation.

## Sprint 1 Order

1. Define the ARIS study harness and fake fixture format.
2. Write a cross-model review loop threat model.
3. Draft a result-to-claim verification matrix.
4. Draft an experiment audit checklist.
5. Draft a paper-claim audit taxonomy.
6. Decide whether one small fake prototype is safe enough for Sprint 2.

## Do Not Start With

- production cross-model review;
- proof-checker runtime;
- meta-optimize runtime;
- paper compile automation;
- live provider calls;
- automatic evidence mutation.

## Sprint 1 Exit Criteria

- Study docs exist and are internally consistent.
- Risks and non-goals are explicit.
- Fake prototype candidates are ranked.
- No ARIS feature is presented as implemented production behavior.
- Human review remains the release-facing authority.
