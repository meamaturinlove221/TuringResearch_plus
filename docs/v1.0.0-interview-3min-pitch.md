# v1.0.0 Interview 3min Pitch

Status: final interview pitch.

Round: 185.

## 3 分钟讲法

### 1. 背景问题

研究项目真正难的地方不只是生成文本，而是状态会散掉：证据、artifact、路线、
失败原因、paper notes、advisor feedback、dashboard 和 release checklist 经常分散在
不同地方。时间一长，很容易把 planned 写成 observed，或者把缺失证据包装成漂亮结论。

### 2. 项目定位

TuringResearch Plus 是 local-first Research OS。它把一个研究项目拆成可审查的状态：

- evidence ledger 记录 claim 状态；
- artifact audit 检查文件是否存在、是否可公开、是否足够支持 claim；
- route DSL 和 hard gates 区分 planned route、blocked route 和 executed evidence；
- paper intelligence 生成 digest、method card、related work scaffold 和 deep review checklist；
- advisor pack 和 dashboard 把当前状态变成 review meeting 能看的材料；
- plugin safety、privacy gate、quality gate 和 replay gate 防止默认路径越界。

### 3. 架构演进

项目从 monorepo 开始，因为 README、quickstart、public demo、tests、release gates 和 star
入口需要集中。后来逐步演进成 modular architecture：

- `turing_research_plus` 保留为 compatibility namespace；
- `turing_research_core`、`turing_research_paper`、`turing_research_artifact`、
  `turing_research_experiment`、`turing_research_dashboard`、`turing_research_plugins`、
  `turing_research_cases` 作为 facade namespaces；
- 每个模块有 public API contracts、stability matrix 和 import compatibility tests；
- 未来拆仓前先做 split readiness gate。

### 4. 工程难点

这个项目不是简单 prompt 工程。更核心的工程点是边界设计和验证：

- fake/live boundary；
- planned vs observed boundary；
- privacy-first public demo；
- plugin manifest 和 sandbox policy；
- contracts / tests / gates；
- mypy、ruff、contract tests、workflow tests、privacy tests 和 replay tests。

### 5. VGGT dogfooding

VGGT 是 dogfooding case。它展示系统如何组织研究 intent、route changes、failures、
evidence management、advisor pack、paper scaffold 和 public-safe case study。它不是实验成功
宣传，也不替代真实研究验证。

### 6. 结尾

我会把它总结为一个偏系统工程的 research tooling 项目：它不承诺自动科研，而是让复杂研究流程
更可追踪、更可复核、更安全地走向公开 demo 和 release。
