# Public Example Policy

Status: active.

## Requirements

Public examples must be:

- fake, demo, or fully authorized;
- small enough for repository review;
- free of secrets and tokens;
- free of raw data and restricted datasets;
- free of private model files;
- explicit about demo-only status;
- clear that fake results are not observed evidence.

## Required Labels

Use clear status labels such as:

- `demo only`
- `fake-data`
- `planned`
- `not-enough-evidence`
- `requires-human-review`

## Not Allowed

- private project path config
- real API credentials
- private papers
- raw datasets
- model payload files
- unreviewed generated claims
- planned work written as observed
