Status: GO for human review, NO-GO for automation.

Round: 343.

## GO

The following are allowed after Round 343:

- human review of `split_manual/turingresearch-vggt-case/`;
- human review of `split_manual/turingresearch-examples/`;
- manual comparison against `split_ready/` source bundles;
- manual owner/name/visibility decision;
- manual privacy, compliance, license, and README review;
- manual decision about whether to create child repositories later.

## NO-GO

The following remain blocked:

- automatic GitHub repository creation;
- automatic external push;
- automatic release publication;
- automatic tag creation;
- automatic real URL insertion;
- automatic issue template publication;
- uploading private data;
- uploading raw data;
- committing secrets;
- publishing unsupported research or benchmark claims.

## Required Before Any Real Child Repository Exists

1. Maintainer approves the exact source bundle.
2. Maintainer approves the manual pack.
3. Privacy and public-release checks pass.
4. License posture is reviewed.
5. README backlink to the flagship is approved.
6. Repository owner/name/visibility are approved.
7. The external repository is created manually.
8. The real URL is inserted only after the repository exists.

## Flagship Rule

The main TuringResearch repository remains the install, quickstart, public API,
release, docs, and star entry. Child repositories are optional case/demo spokes
and must not replace the flagship.
