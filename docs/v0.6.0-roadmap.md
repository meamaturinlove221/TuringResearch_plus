# v0.6.0 Roadmap

Status: planning.

Round: 102.

v0.6 should move TuringResearch Plus from a single flagship case study toward a
general research operating system. The immediate goal is not more automation for
its own sake. The goal is to make evidence, artifacts, project templates,
dashboards, writing support, and plugin boundaries reusable across projects
while preserving human review and safety gates.

## Roadmap Theme

v0.5 alpha/beta planning made the product usable as a local, fake/default,
review-first package. v0.6 should make the same architecture portable across
research projects:

- project templates become first-class workspace assets;
- dashboards compare multiple projects and runs;
- remote execution outputs are ingested as structured evidence, not trusted
  blindly;
- paper writing support remains claim-checked and human-authored;
- plugins and MCP surfaces become discoverable, testable, and release-ready.

## Recommended v0.6 Sprints

### Sprint 1: Multi-project Workspace Foundation

- General research project template stabilization.
- Workspace registry for multiple research projects.
- Cross-project evidence graph planning and minimal export.
- Privacy and data policy layer for workspace-level scans.

### Sprint 2: Research OS Dashboard

- Multi-project dashboard index.
- Project and run status comparison.
- Advisor-ready workspace summary.
- Static-first UI export; no SaaS account system.

### Sprint 3: Writing and Review Assistants

- Paper digest expansion.
- Related-work positioning refinement.
- Paper writing assistant with claim gates.
- Advisor communication templates.

### Sprint 4: Plugin and Marketplace Readiness

- Capability manifest.
- Plugin contract schema.
- MCP marketplace metadata preparation.
- Public release compatibility checks.

### Sprint 5: VGGT Case Study Hardening

- Replay using only newly supplied real artifacts.
- Evidence ledger reconciliation.
- Advisor pack regeneration.
- Public case-study package if and only if evidence supports it.

## Near-term Items

- Research OS positioning.

## Long-term Items

- Public plugin ecosystem.
- MCP marketplace readiness.

- Optional paper writing assistant with strict claim provenance.

## Requires Human Review

- Any evidence promotion from proposed/imported/retrieved to observed.
- Any remote execution output.
- Any paper digest, citation graph, related-work positioning, or generated prose.
- Any advisor-facing export that could be read as a final result.
- Any plugin or marketplace metadata intended for public release.
- Any VGGT claim involving SparseConv3D, backend success, or visual improvement.

## Privacy / Data / License Risk

- Remote execution artifacts may contain raw data, paths, secrets, or private
  model files.
- Multi-project workspaces can accidentally mix evidence between projects.
- Paper writing assistants can reproduce copyrighted text if not constrained.
- Plugin ecosystems can introduce license and supply-chain risk.
- Public case studies can expose private project names, files, logs, or datasets.

## Cannot Be Fully Automated

- Human paper reading and interpretation.
- Final paper conclusions.
- Evidence promotion to observed.
- Advisor approval.
- Privacy/license approval for public release.
- Experiment success claims.
- Cross-project transfer of evidence.

## Non-Goals for v0.6 Planning

- No implementation in Round 102.
- No network access.
- No remote execution.
- No cloud deployment.
- No user account system.
- No automatic paper writing.
- No automatic experiment success claim.
- No future sync adapter implementation.
