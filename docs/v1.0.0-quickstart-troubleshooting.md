# v1.0.0 Quickstart Troubleshooting

Status: troubleshooting draft.

Round: 179.

## Install Fails

Check Python version:

```powershell
python --version
```

Expected: Python 3.11 or newer.

Then retry:

```powershell
python -m pip install -e .[dev]
```

## Pytest Cannot Find Packages

Run from the repository root and use the editable install:

```powershell
python -m pip install -e .[dev]
python -m pytest tests/workflow/test_public_demo_suite.py -q
```

## MCP Command Not Found

Install MCP extras locally:

```powershell
python -m pip install -e .[dev,mcp]
```

Or use the module path:

```powershell
python -m turing_research.mcp_server --manifest
```

## API Key Error

The public quickstart should not require API keys. If a command asks for a real
key, you are likely running a live/manual path instead of the fake/default
quickstart path.

Use:

```text
TURINGRESEARCH_MODE=fake
TURINGRESEARCH_ENABLE_LIVE_TESTS=0
```

## Dashboard Does Not Open

The dashboard is static HTML. Open the file directly in a browser:

- `examples/public_demo/dashboard/index.html`

No server is required.

## Demo Looks Like A Real Result

It should not. Public demo outputs must be marked demo-only and require human
review. If a demo appears to claim observed research success, treat it as a
documentation bug.

## What Not To Debug In Quickstart

- VGGT private data.
- SMPL-X model files.
- Modal or remote experiment execution.
- live provider credentials.
- unknown plugin execution.
- final paper generation.
