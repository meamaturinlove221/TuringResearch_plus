# v1.1.0 Final Risk Register

Status: final risk register.

Round: 205.

## High Risks

| Risk | Severity | Mitigation |
| --- | --- | --- |
| v1.1 starts from unclear branch state | high | Make baseline stabilization the first workstream. |
| Dirty worktree contaminates v1.1 commits | high | Require selective commits and clean branch review. |
| Physical split weakens flagship | high | Split only case/examples after human approval; child READMEs point back to flagship. |
| Child repo URLs appear before repos exist | high | Add URL checks; use local `split_ready/` paths until repos exist. |
| Local server dashboard becomes SaaS-like | high | Local-only, no login, no cloud user system, static fallback required. |
| Paper beta is mistaken for final paper writing | high | Keep beta labels, evidence refs, and human-review gates. |
| Public demos imply real results | high | Keep demo-only labels and no observed fake result checks. |
| Plugin ecosystem expands without review capacity | high | Keep unknown third-party plugin execution disabled; defer ecosystem expansion. |
| Live mode becomes default | high | Keep fake/default tests and explicit live opt-in. |

## Medium Risks

| Risk | Severity | Mitigation |
| --- | --- | --- |
| Docs site drifts from README | medium | Link-check and keep README as flagship entry. |
| Examples repo sync policy is unclear | medium | Finalize examples sync before any physical split. |
| CI/CD hardening over-automates release | medium | Start with checks; no publish/tag automation without human approval. |
| More case studies increase claim risk | medium | Require claim-safety report for every case. |
| Compliance helpers are read as legal advice | medium | Keep not-legal-advice disclaimers visible. |

## Release Blockers

- name integrity failure;
- privacy/security failure;
- private/local path in public output;
- secrets or raw data in repo;
- restricted model payload;
- fake/demo output marked as observed evidence;
- unsupported VGGT or SparseConv3D success claim;
- unknown plugin execution enabled;
- live networking required by default;
- nonexistent child repo URL in README/docs;
- full tests or mypy fail at RC gate.

## Carryover From v1.0

- v1.0 release branch not observed locally;
- current branch remains `pre-rename-round38-checkpoint`;
- worktree remains historically dirty;
- v1.0 public launch still requires human review.
