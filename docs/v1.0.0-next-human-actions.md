### Public Demo Review

- Confirm demo is marked demo-only.

- Confirm dashboard and advisor bundle wording stays review-first.

### Tag / GitHub Release Decision

- Create or check out a clean release branch.
- Re-run full tests and mypy on that branch.
- Review final diff.
- Create tag only after maintainer approval.
- Create GitHub release only after tag and release draft are approved.

### Split Repository Decision

- Start with `turingresearch-vggt-case` only if approved.
- Create `turingresearch-examples` only after the case repo validates the spoke
  model.
- Delay plugins until real ecosystem demand appears.
- Keep core, paper, artifact, dashboard, and plugin framework in the flagship.

### Screenshot / GIF Preparation

- Use public demo only.
- Do not include private data.
- Do not show restricted model files.
- Do not imply observed experiment success.
- Keep visuals consistent with README limitations.
