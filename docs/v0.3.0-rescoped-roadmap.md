# TuringResearch Plus v0.3.0 Rescoped Roadmap

Round 63 rescopes v0.3 around manual upstream signals and the current
TuringResearch Plus v0.2 beta surface.

## v0.3 Direction Split

### Sprint 1: Git-based Context Handoff / Pod Workflow

Goal: make cross-machine and pod workflows explicit, reviewable, and safer than
ad hoc file copying.

Candidate capabilities:

- Git-based context bundle export/import.
- Durable `CLAUDE.md` / `MEMORY`-style context packaging.
- Structured output files pushed back from pod to local.
- Dotfile and secret exclusion rules.
- Handoff bundle compatibility with v0.2.

### Sprint 2: Scholar Pipeline Refinement

Goal: improve paper discovery, cached markdown, references, and reading
workflows while preserving fake/offline defaults.

Candidate capabilities:

- arXiv > Semantic Scholar > Unpaywall planning.
- references pagination support.
- local cached markdown reads.
- paper reference fallback from markdown.
- Keshav three-pass reading workflow.

### Sprint 3: Web Fetch Adapter / Apify Adapter

Goal: refine web fetching and content caching with explicit optional live mode.

Candidate capabilities:

- Apify REST adapter behind live adapter policy.
- `web_fetching` / `web_content` refinement.
- `.mcp.json` env-block pattern.
- real integration tests marked live/manual.

### Sprint 4: Knowledge Graph / Wiki / Ontology / Skill Routing

Goal: improve knowledge graph memory, edge audit, and skill routing.

Candidate capabilities:

- wiki edge audit.
- inline `[[wikilink]]` parsing.
- ontology SOPs.
- campaign `ENTRY.md` / routing table pattern.
- skill routing documentation.

## Near-Term Plan Change

Yes, the near-term plan changes. v0.3 Sprint 1 is now Git-based Context Handoff
/ Pod Workflow.

NAS / SMB / SSH / GitHub Artifact Sync is temporarily not the Sprint 1 mainline.
Git-based handoff is closer to the latest Neocortica-Session route and better
matches the current dual-machine + VGGT + Modal operating pattern.

## Non-Goals

- Do not implement code in Round 63.
- Do not copy upstream code.
- Do not require default networking.
- Do not read private VGGT paths.
- Do not claim manual snapshot items are newly detected without baseline.
