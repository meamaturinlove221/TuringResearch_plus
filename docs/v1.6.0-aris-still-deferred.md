# v1.6.0 ARIS Still Deferred

Status: deferred.

Round: 361.

ARIS remains outside the v1.6 implementation line. v1.6 is for public release
execution readiness: docs, package/install checks, split manual packs, optional
live smoke readiness, release artifacts, launch checklist, demo assets, full
regression, and v1.7 planning.

## Not Implemented in v1.6

- ARIS runtime.
- Cross-model review execution.
- Proof-checker.
- Meta-optimize.
- Paper-claim-audit.
- Automated research loop.
- Automatic evidence promotion.
- Final claim authority from ARIS-like reports.

## Why Deferred

v1.6 crosses closer to public release execution. Adding ARIS in the same line
would mix release engineering with a higher-risk research automation layer.
That would make the safety story harder to verify and could create false
authority around claims.

## Allowed Future Shape

ARIS can return only as a separately scoped study-only track, with:

- explicit scope lock;
- fake/demo data only at first;
- disabled-by-default behavior;
- privacy/security gate;
- claim authority boundary;
- no automatic observed evidence.

## v1.6 Decision

Keep ARIS deferred. Do not start ARIS implementation, cross-model review,
proof-checking, meta-optimization, or paper-claim auditing in v1.6.
