# TuringResearch Plus v0.2.0 Sprint 2 P0 Implementation Order

Round 45 keeps the Round 44 implementation order:

1. Hard Gate Library
2. Experiment Route DSL
3. Failure Taxonomy Engine
4. Paper-to-Method Card
5. Figure-to-Architecture

## Why This Order

- Hard Gate Library is the evidence and promotion boundary used by every later
  Sprint 2 capsule.
- Experiment Route DSL gives VGGT long-running work a structured plan before
  any live execution or artifact ingestion.
- Failure Taxonomy Engine turns gate failures and route blockers into shared
  next-action language.
- Paper-to-Method Card needs provenance and failure/gate language before
  extracting method structure.
- Figure-to-Architecture should wait until method cards and figure provenance
  exist, so architecture drafts do not fabricate unsupported edges.

## Required Contracts

- `contracts/hard_gates.yaml`
- `contracts/experiment_route.yaml`
- `contracts/failure_taxonomy.yaml`
- `contracts/method_cards.yaml`
- `contracts/figure_architecture.yaml`

The first implementation slice should be:

2. Experiment Route DSL

Failure Taxonomy, Paper-to-Method Card, and Figure-to-Architecture should wait
