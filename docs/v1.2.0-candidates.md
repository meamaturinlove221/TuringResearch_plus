## Recommended Sprint-1 Set

1. Physical split execution for `turingresearch-vggt-case`.
2. Physical split execution for `turingresearch-examples`, after the case repo
   validates the spoke pattern.
3. Public docs deployment.
4. Local server dashboard polish.
5. Pod Lifecycle Manager scope/prototype.

Paper writing beta refinement should remain close behind, but the first sprint
should prioritize public ecosystem shape and local usability.

## Deferred Or Research-only

- MinerU / heavy PDF fallback research.
- OS-level plugin sandbox research.
- Apify workflow templates beyond static examples.
- Broad live web/scholar behavior before fake/default regression remains clean.
- Any remote execution orchestrator.

## Candidate Safety Baseline

All candidates must preserve:

- no private data;
- no secrets;
- no raw data;
- no restricted model payloads;
- no unsupported experiment success claims;
- no automatic experiment execution;
- no automatic final paper generation;
- no default live networking;
- no unknown plugin execution;
- flagship repository remains the main install and star target.
