# Dashboard Screenshot Checklist

Status: planning checklist.

Round: 164.

Use this checklist before adding dashboard screenshots to README or public
docs.

## Required Screenshots

- [ ] Public demo dashboard.
- [ ] Refined VGGT dashboard.
- [ ] Advisor export quality report.
- [ ] Public case-study claim safety view.
- [ ] Vault UI or graph view.

## Safety Checks

- [ ] Screenshot uses fake/demo or redacted material only.
- [ ] No private local paths.
- [ ] No raw data.
- [ ] No model payloads.
- [ ] No API keys or tokens.
- [ ] No private advisor feedback.
- [ ] No unsupported VGGT or SparseConv3D success claim.
- [ ] Safe demo mode or review boundary is visible.

## Visual Checks

- [ ] Text is readable.
- [ ] Dashboard cards are not cropped.
- [ ] The screenshot works in README width.
- [ ] The image has descriptive alt text.
- [ ] The image filename is stable and lowercase.

## Approval

- [ ] Privacy scan complete.
- [ ] Claim safety review complete.
- [ ] Maintainer approval complete.
