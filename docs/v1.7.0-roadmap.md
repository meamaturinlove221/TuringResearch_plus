# v1.7.0 Roadmap

Status: planned roadmap.

Round: 389.

v1.7 should follow the v1.6 public release execution pack by deciding which
manual public actions to perform after maintainer review. It should not expand
the core research runtime until the public release surface is stable.

## Candidate Direction

1. Actual GitHub Pages enablement.
2. Actual split repo creation after human confirmation.
3. PyPI release preparation.
4. Optional live guarded runner.
5. Plugin ecosystem polish.
6. Dashboard UX polish.
7. ARIS study only, not implementation.

## Default Recommendation

- Start with GitHub Pages only if the v1.6 docs release bundle is reviewed and
  a real public URL will be inserted after deployment exists.
- Create split repositories only after a human reviews each split pack,
  license posture, README backlinks, privacy scan, and URL placeholder policy.
- Treat PyPI as preparation first: package name, license, artifact inspection,
  README rendering, and compatibility policy before upload.
- Keep optional live work private, guarded, redacted, and disabled by default.
- Polish plugins and dashboard UX without enabling unknown plugin execution,
  analytics, tracking, or default live network access.
- Keep ARIS as a study-only candidate with no runtime implementation in v1.7.

## Release Boundary

This roadmap does not deploy docs, create repositories, publish to PyPI, create
tags, publish GitHub releases, run live providers, upload data, execute remote
commands, or implement ARIS.
