# v1.2.0 Release Notes

Status: release-candidate notes.

Round: 256.

TuringResearch Plus v1.2.0 is an original-reference parity release-candidate
line. It keeps the project local-first and fake/demo-first while aligning the
stable ideas from the tracked reference repositories with TuringResearch's own
contracts, docs, tests, public demo, and safety gates.

This release does not publish a SaaS product, does not execute experiments
automatically, does not enable live networking by default, does not create child
repositories automatically, and does not adopt ARIS runtime features into v1.2.

## Highlights

- Original reference parity strategy.
- Neocortica Session parity for context pack safety and structured returns.
- Neocortica Scholar parity for paper source priority, tool lists, MCP usage,
  and fallback policy.
- Neocortica Web parity for fake/default web fetching, web content, Apify usage
  guidance, cache/source metadata, and no-key graceful skip.
- MCP config parity with fake mode defaults and optional live env flags.
- Skill SOP parity for key workflows.
- yogsoth campaign parity with strategy book and preconditions.
- vault/wiki/edge audit parity.
- ontology SOP parity.
- stress test parity.
- experiment execution parity through safe runbooks and artifact requirements.
- TuringResearch Research Catalog integration.
- Reference Parity Dashboard and v1.2 public demo refresh.
- ARIS deferral roadmap for v1.3+ study.

## What Is New Since v1.1

- v1.2 narrows the roadmap around original-reference parity first.
- Public docs now explain why ARIS remains deferred.
- Reference parity surfaces are backed by contracts and workflow tests.
- Public demo now shows Research Catalog, reference parity, and stress-test
  review with fake/demo data.
- Security/privacy gate and full regression now cover v1.2 parity additions.

## What Remains Review-gated

- Release branch push, tag creation, GitHub release, and PyPI publishing.
- Live provider tests and live adapter polish.
- ARIS study and any future cross-model review loops.
- MinerU or heavy PDF fallback.
- Remote execution orchestration.
- Physical child repository creation.

## Release Boundary

v1.2.0 remains a local-first Research OS release candidate. Human review is
required before publication, claims, external repository creation, live
provider usage, or public release actions.
