# v1.3.0 ARIS Study Roadmap

Status: study roadmap.

Round: 257.

v1.3 opens an ARIS study track after v1.2 original-reference parity. The goal is
to evaluate ARIS-inspired ideas with safety, evidence, and human-review
boundaries before any selective adoption.

This roadmap does not implement ARIS features. It does not merge an ARIS runtime
into TuringResearch.

## Default Recommendation

v1.3 should do study / matrix / small fake prototype only.

Do not do a full ARIS merge in v1.3.

## Candidate Studies

| Candidate | v1.3 target | Study output |
| --- | --- | --- |
| cross-model review loop study | study only | design note, threat model, fake review transcript fixture |
| result-to-claim verification study | study only | evidence mapping matrix, missing-result guard design |
| experiment audit study | study only | audit checklist, fake run manifest review |
| paper-claim audit study | study only | paper claim taxonomy, reviewer boundary note |
| proof-checker study | study only | checklist framing, correctness-risk memo |
| meta-optimize study | study only | tuning-risk memo, configuration boundary proposal |
| effort levels study | study only | effort-level schema proposal and fake examples |
| paper compile audit study | study only | compile checklist and artifact requirement matrix |

## Study Principles

- Keep fake/demo mode as the default.
- Require human review for every ARIS-inspired output.
- Treat model review as advice, not authority.
- Do not infer missing results.
- Do not mutate evidence automatically.
- Do not write final papers automatically.
- Do not enable live provider calls by default.
- Do not add unknown remote execution.

## Expected v1.3 Deliverables

- ARIS feature evaluation matrix.
- ARIS risk register.
- ARIS non-goals.
- One or two small fake prototypes if they stay local, deterministic, and
  review-only.
- Go/no-go recommendation for possible v1.4 selective adoption.

## Success Criteria

v1.3 succeeds if TuringResearch can explain which ARIS ideas are useful, which
are too risky, and which small subset might be mature enough for later adoption.

It should not end with an autonomous research runtime.
