1. Neocortica Session runtime parity.

5. yogsoth Campaign execution trace.

## Rationale

Session, Scholar, Web, MCP, and Skill parity form the user-facing reference
workflow foundation. The yogsoth campaign/vault/stress/runbook surfaces should
then become a richer traceable demo. Only after those are coherent should v1.3
run a full original parity replay and prepare release materials.

## Exit Criteria Per Workstream

| Workstream | Exit criteria |
| --- | --- |
| Session runtime parity | local fake workflow, context pack, return manifest, no remote execution by default |
| Scholar tool surface | tool list, source priority, MCP usage, fallback, fake tests |
| Web tool surface | web_fetching, web_content, Apify optional guide, no-key skip, fake tests |
| MCP / SKILL / README | docs and skill routing point to the same runnable fake path |
| Campaign trace | deterministic trace, preconditions, outputs, handoff notes |
| Vault / Ontology / Wiki demo | public-safe graph, backlinks, dangling links, gap report |
| Stress / Experiment runbooks | convergence/stress findings feed runbook requirements |
| Full replay | one fake replay covers original parity end to end |
| Public dashboard | dashboard shows complete/partial/deferred/rejected status clearly |
| Release prep | notes, limitations, tests, security/privacy, and handoff updated |

## Dependency Rule

Do not start ARIS implementation to fill a parity gap. If original parity is
blocked, document the blocker and keep ARIS deferred.
