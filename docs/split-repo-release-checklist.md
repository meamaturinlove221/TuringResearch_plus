## Required Checklist Items

Every split repo release checklist must include:

- repo created manually;
- README reviewed;
- license reviewed;
- privacy reviewed;
- no secrets;
- no raw data;
- no private paths;
- main repo linked;
- first release draft;
- issue templates optional.

## Shared Release Boundary

The main TuringResearch repository remains the flagship install, docs, release,
public API, and star entry point. Split repositories may only be public-safe
spokes after human approval.
