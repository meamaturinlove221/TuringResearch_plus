# v1.0.0 Public Release Plan

Status: planning draft.

Round: 173.

v1.0 should be a public release only after the flagship repository is clear,
safe, and stable enough for users to install, inspect, and demo locally.

## Public Release Goals

- public README is honest and compelling;
- quickstart works locally;
- public demo is fake/demo-first and privacy-safe;
- docs index is complete;
- release notes and upgrade guide are current;
- plugin manifests are stable and disabled/review-gated by default;
- privacy, compliance, and quality gates pass;
- no private data, raw data, secrets, or model payloads are included.

## Public README Requirements

The README must say:

- local-first Research OS;
- fake/demo first;
- live adapters optional;
- human review required;
- no automatic experiment execution;
- no automatic final paper;
- VGGT is a dogfooding case, not proof of final research success;
- sensitive files are excluded.

## Release Candidate Hardening

Required gates:

- full pytest;
- `python -m mypy src`;
- name integrity;
- privacy/security audit;
- compliance audit;
- public demo tests;
- split export safety checks if optional spokes are included;
- package metadata and import surface tests;
- docs link check;
- pre-push sensitive scan.

## Publication Non-goals

- no SaaS;
- no cloud user system;
- no default live networking;
- no automatic remote experiment execution;
- no private VGGT data;
- no final paper claims;
- no unreviewed plugin execution.

## Human Approval

The v1.0 release requires maintainer approval for:

- license posture;
- public README;
- release notes;
- optional split repo links;
- package version;
- public launch timing.
