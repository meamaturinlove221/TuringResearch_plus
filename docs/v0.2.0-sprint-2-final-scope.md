## Sprint 2 Goals

- Upgrade VGGT long-route planning from hand-written prompts to structured route
  DSL.
- Standardize hard gates so fast-return, report-only, fallback-only, and
  not-ready routes cannot be promoted silently.
- Standardize failure attribution so evidence ledgers, advisor packs, and route
  reports use the same vocabulary.
- Convert paper understanding into Method Cards for NeuralBody, HumanRAM, HART,
  and related work only when provenance is available.
- Convert method cards and route structures into Mermaid / Graphviz architecture
  drafts without claiming unsupported implementation success.

## Acceptance Criteria

- Each selected item has a contract before implementation.
- Each selected item has deterministic unit tests and at least one workflow
  dry-run test.
- Outputs serialize to JSON and Markdown.
- All gates preserve the difference between `planned`, `observed`,
  `local-observed`, `requires-human-review`, `not-enough-evidence`, and
  `hard-blocked`.
- SparseConv3D success remains unclaimed unless real evidence appears in the
  evidence ledger.
- Missing paper or figure inputs become explicit `requires-human-review` or
  `requires-real-paper` outputs instead of fabricated method claims.
- No network, real VGGT execution, Modal live execution, or sync adapter is
  required for default tests.

## Deferred From Sprint 2

- Modal Experiment Run Ingestor.
- Collision Risk Detector.
- Live Semantic Scholar adapter.
- Real citation graph expansion.
- Vault graph enhancement.
- NAS / SSH / GitHub artifact sync.
- Large UI.
- OCR heavy pipeline.

## Release Boundary

Sprint 2 may produce contracts, models, deterministic local services, docs, and
fake/dry-run examples. It must not claim live VGGT experiment completion, live
paper analysis completion, or advisor-ready proof unless the corresponding
evidence is present.
