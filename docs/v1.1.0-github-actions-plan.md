# v1.1.0 GitHub Actions Plan

Status: planning only.

Round: 226.

This document describes the intended GitHub Actions shape. It does not modify
workflows in this round.

## Existing Workflows

- `lint.yml`: ruff and optional mypy.
- `test.yml`: Python 3.11 / 3.12 pytest matrix plus fake workflow examples.

## Recommended Workflow Layout

### `lint.yml`

Recommended checks:

- install `.[dev]`;
- run `python -m ruff check .`;
- run `python -m mypy src` on release branches;
- keep lint free of live adapter credentials.

### `test.yml`

Recommended checks:

- unit tests;
- contract tests;
- workflow tests in fake/default mode;
- full default pytest suite;
- Python 3.11 as the release baseline;
- Python 3.12 as compatibility signal.

### Future `release-check.yml`

Recommended manual workflow dispatch only:

- verify version files;
- verify changelog and release notes;
- run name integrity;
- run privacy/security gates;
- run docs-site build;
- verify release draft exists;
- produce a release readiness summary.

This future workflow should not create tags or GitHub releases.

## Environment Rules

- No repository secrets are required for default CI.
- Live adapter secrets must not be used by default.
- Live jobs must require explicit operator action.
- Do not upload private data, raw data, restricted model files, or local
  machine paths.

## Suggested Job Groups

| Group | Command shape | Notes |
| --- | --- | --- |
| Lint | `python -m ruff check .` | Required |
| Unit | `python -m pytest tests/unit` | Required |
| Contract | `python -m pytest tests/contract` | Required |
| Workflow | `python -m pytest tests/workflow` | Fake/default only |
| Docs | docs-site build tests | Required before release |
| Privacy | privacy/security gates | Required before release |
| Live | marked live tests | Skipped by default |

## Manual Approval

GitHub Actions may support release readiness reporting, but tag creation,
GitHub release creation, and package publication must stay manual for v1.1.
