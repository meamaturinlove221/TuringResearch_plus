# v1.0.0 Upgrade Guide

Status: release-candidate upgrade guide.

Round: 195.

## Version

The v1.0 release-candidate version is `1.0.0rc0`.

## Install

Use the main repository as the install target:

```bash
python -m pip install -e .[dev]
```

For MCP-oriented local checks:

```bash
python -m pip install -e .[dev,mcp]
```

## Import Paths

The compatibility namespace remains available:

```python
import turing_research_plus
```

v1 also documents facade namespaces:

```python
import turing_research_core
import turing_research_paper
import turing_research_artifact
import turing_research_experiment
import turing_research_dashboard
import turing_research_plugins
import turing_research_cases
```

Do not remove `turing_research_plus` imports before v1.0. It remains the
compatibility namespace.

## Configuration

`.mcp.example.json` uses fake/default mode. Live adapters require explicit
opt-in and credentials supplied by the user. Do not commit real keys.

## Split Repositories

No split repository is required to upgrade. `split_ready/` contains local export
bundles only and does not change install paths.

## Behavioral Notes

- Live adapters remain optional.
- Unknown plugin execution remains blocked.
- Optional PDF/PPTX export backends skip gracefully when unavailable.
- Compliance and privacy reports remain review aids.
