# v0.7.0 Risk Register

Status: scope locked.

| Risk | Severity | Mitigation |
| --- | --- | --- |
| Plugin loading executes untrusted code | Critical | Trusted-local allowlist, sandbox policy, disabled third-party defaults, no unknown entrypoints. |
| Sandbox policy is only documentation | High | Contract tests must require explicit permission model and safety reports. |
| Export adapters pull in unsafe dependencies | High | Optional adapters only, fake/default tests, dependency review before enablement. |
| PDF/PPTX outputs imply unsupported claims | High | Export from reviewed advisor bundles only; no fabricated charts or results. |
| Dataset/license assistant reads like legal advice | High | Label as review aid, require human/legal review, no compliance guarantees. |
| Vault UI becomes SaaS-like scope creep | Medium | Keep local-first static/lightweight path first. |
| Paper deep review fabricates citation confidence | High | Require source status and human paper review; block fake fixtures. |
| VGGT case study overclaims results | Critical | Evidence ledger gating; no SparseConv3D success without real evidence. |
| Public release includes private data | Critical | Privacy gate, public release hygiene, demo-safe policy, maintainer review. |
| Roadmap promises more than implementation supports | Medium | Keep planned/optional/future labels explicit. |

## Release Blockers

- Any committed real secret.
- Any raw data or licensed/private model payload.
- Any `local_project_links.yaml` in public release artifacts.
- Any unknown third-party plugin enabled by default.
- Any final paper conclusion generated without human review.
- Any fake/demo result marked observed.
- Any old project naming in v0.7 scope files.
