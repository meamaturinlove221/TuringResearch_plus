# v0.5.0 Implementation Order

Status: scope locked.

## Recommended Order

1. Project Template Generator
2. MCP / CLI Packaging Polish
3. Public Demo Suite
4. Lightweight Web UI / Dashboard
5. Real Advisor PDF / PPTX Export
6. Real VGGT Dogfooding Replay
7. v0.5 Integration Gate

## Rationale

Project templates, packaging polish, and demo suite work reduce release risk
before adding UI and binary export surfaces. UI and PDF/PPTX export should be
fed by stable fixture workflows. Real VGGT dogfooding replay should remain last
because it depends on project template, packaging, demo, dashboard, and export
boundaries being clear.

## Checkpoints

- Checkpoint A: templates + packaging polish pass fake workflow tests.
- Checkpoint B: demo suite can run without network or secrets.
- Checkpoint C: local dashboard renders existing artifacts without claiming
  experiment execution.
- Checkpoint D: PDF/PPTX export preserves limitations.
- Checkpoint E: VGGT replay generates proposed updates only.
