# TuringResearch Plus v0.2.0 Sprint 1 Report

## Summary

Sprint 1 delivered the local evidence infrastructure needed for VGGT / SMPL-X
dogfooding without claiming live experiment success. The sprint is useful for
turning messy local scan context into conservative evidence, missing-evidence
reports, advisor-readable Markdown, and fixture-safe PDF asset extraction.

## Remaining Limitations

- Visual Evidence Auditor is currently a dry-run artifact surface.
- `local_scan_evidence_ledger.json` is missing.
- `local_scan_visual_inventory.md` is missing.
- No real VGGT experiment was run.
- SparseConv3D success is not claimed.
- PDF Phase B remains lightweight and does not include OCR or layout-heavy parsing.

## Known Risks

- Advisor pack wording can still over-imply readiness if not reviewed by a human.
- Missing visual inventory limits VGGT usefulness for visual claims.
- Simple PDF table extraction can miss publisher-formatted or scanned tables.
- Capsule-local proposed namespaces need a later public API decision.

## VGGT Usefulness

Sprint 1 is useful for VGGT dogfooding as a status discipline layer. It captures
what can be said, what is missing, and what must remain human-review or
not-enough-evidence. It does not prove VGGT route success.

## Advisor Pack Readiness

Advisor Pack Builder is ready for Markdown review packages. It is not a final
advisor acceptance workflow and does not create PPTX or PDF.

## PDF Phase B Status

PDF Phase B is implemented as a lightweight local extractor for fixture PDFs:
embedded image extraction, simple text table extraction, section tree, page map,
and paper figure registry integration.

## Upstream Learning Applied

No current `docs/upstream-learning-report.md` or `docs/upstream-adoption-plan.md`
was available in this checkout. Sprint 1 still follows the documented upstream
learning policy: adopt architecture patterns and testing discipline only, do not
copy incompatible code, and keep source hygiene gates visible.

## Non-Goals Preserved

- No network.
- No live API adapters.
- No private `D:/vggt` reads.
- No VGGT execution.
- No PPTX/PDF generation.
- No NAS/SSH/GitHub sync.
- No heavy OCR pipeline.
