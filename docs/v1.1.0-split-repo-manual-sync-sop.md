# v1.1.0 Split Repo Manual Sync SOP

Status: manual workflow.

Round: 212.

This SOP describes the manual workflow for syncing public-safe material from the
flagship repository into future split repositories. It is intentionally manual
for v1.1.

## Preconditions

- The flagship repository is reviewed and tests pass.
- The target split repository exists or has explicit creation approval.
- The source material is public-safe.
- The maintainer has approved the exact file set.
- The spoke README still routes users back to the flagship.

## Manual Sync Steps

1. Select the source release, branch, or reviewed bundle in the flagship.
2. Compare source files against `split_ready/split_manifest.yaml`.
3. Copy only files listed as public-safe.
4. Do not copy raw data, private paths, secrets, restricted model files, huge
   artifacts, or private logs.
5. Update the spoke README crosslink placeholder only after the target URL is
   real and approved.
6. Update the spoke manifest with the source version and sync date.
7. Re-run split safety tests, privacy gate, and public demo tests if examples
   changed.
8. Record the sync result in the flagship ledger or release notes.
9. Ask for final human approval before any external push.

## Required Review Checklist

- Source of truth remains the flagship.
- Spoke contains demo/case material only.
- No core framework source is introduced.
- No unsupported result claim is introduced.
- No fake/demo output is promoted to observed evidence.
- No nonexistent repository URL is written.
- Issues and security reports still route back to flagship policy.

## Rollback

If a sync introduces ambiguity or unsafe content:

1. stop publication;
2. remove the candidate spoke update;
3. restore the previous reviewed spoke state;
4. file a flagship issue or ledger note;
5. re-run safety checks before retrying.
