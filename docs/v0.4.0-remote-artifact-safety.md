# v0.4.0 Remote Artifact Safety

Status: active for the v0.4 remote artifact track.

Remote artifact features are transport and indexing helpers. They do not prove
that an experiment succeeded and do not replace Evidence Ledger review.

## Shared Rules

- No default networking.
- No credential storage.
- No raw data import.
- No SMPL-X model file import.
- Large files are metadata-only by default.
- Unsafe files are intercepted.
- Proposed imports are review items, not Evidence Ledger writes.
- Remote artifacts are not human verified by default.

## Source Boundaries

- GitHub Artifact Sync lists release assets, workflow artifacts, or fixture
  indexes in fake/default mode.
- SSH / SFTP Remote Reader is read-only and does not execute remote commands.
- NAS / SMB Shared Store reads only user-mounted local paths and does not mount
  shares.
- Cloud Object Artifact Index is provider-neutral and does not use cloud SDKs in
  the minimal implementation.

## VGGT Boundary

For VGGT work, remote artifact reports can help review `final_status.json`,
`failure_report.md`, artifact indexes, board inventories, and sha256 manifests.
They must not claim SparseConv3D success, promote planned routes to observed, or
import raw private data.
