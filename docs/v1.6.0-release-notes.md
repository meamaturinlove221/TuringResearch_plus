# v1.6.0 Release Notes

Status: release-candidate notes.

Round: 387.

TuringResearch v1.6.0 is a public release execution pack. It organizes the
project for human-reviewed public launch actions: docs deployment readiness,
GitHub Pages workflow drafting, split repository creation packs, optional live
smoke policy, release artifact review, dashboard/demo assets, and launch
checklists.

This release candidate does not publish a public site, create tags, create
child repositories, publish to PyPI, run live providers, enable live networking
by default, or implement ARIS.

## Highlights

- Docs deployment ready.
- GitHub Pages workflow draft.
- Split repo creation packs.
- Optional live smoke.
- Live output redaction gate.
- Packaging readiness.
- Release artifact build.
- Dashboard / demo asset pack.
- Public launch checklist.

## What Is New Since v1.5

- Docs deployment is now GitHub Pages-ready for human review, with preflight,
  dry-run workflow, release bundle, manifest, and gate docs.
- Split repository material is packaged as final manual creation packs for
  human review; no child repository is created automatically.
- Optional live smoke is prepared for Scholar, Web/Apify, and SFTP while live
  tests remain skipped by default.
- Live output redaction policy is part of the safety surface for future private
  live runs.
- Package/install readiness is documented around local editable installs and
  compatibility names.
- Release artifacts are review-only local outputs, not uploaded packages.
- Dashboard and demo asset pack now records screenshot placeholders and capture
  requirements without fabricating screenshots.
- Public launch checklist separates maintainer decisions for tags, GitHub
  release, PyPI, Pages, and split repository creation.
- v1.6 full regression confirms docs, split, optional live, package,
  dashboard, release artifact, privacy/security, and ARIS-deferred surfaces.

## Public Naming

The public project name is **TuringResearch**. Compatibility package and import
surfaces remain stable for now:

- package distribution: `turingresearch-plus`;
- Python compatibility namespace: `turing_research_plus`;
- MCP compatibility keys and commands remain available.

Package renaming and PyPI publication remain separate maintainer decisions.

## What Remains Review-gated

- Git tag creation.
- GitHub release publication.
- PyPI publication.
- GitHub Pages deployment.
- Split repository creation.
- Any live smoke run with private credentials.
- Any public claim based on retrieved or remote-returned material.
- ARIS study or implementation.

## Release Boundary

v1.6.0 remains a local-first release candidate. Human review is required before
publication, claims, external repository creation, live provider usage, live
SSH/SFTP usage, package upload, or public docs deployment.
