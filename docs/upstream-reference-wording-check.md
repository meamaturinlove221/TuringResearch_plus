# Upstream Reference Wording Check

Round: 397R
Status: completed

## Result

Use upstream wording as Reference / Inspiration / Related Projects only.

Do not claim migrated academic publications.

## Accepted Wording

Allowed:

- Upstream Reference Docs
- Workflow Inspiration
- Related Projects
- reference parity
- docs/tool-surface parity
- independently implemented TuringResearch equivalent

## Forbidden Wording

Not allowed:

- Authorized Academic Showcase
- Academic Showcase Migration
- migrated academic publications
- migrated upstream papers
- original author academic outputs
- upstream publication migration
- academic output migration complete

## Current Sweep Finding

The forbidden phrases appear only in no-go or policy documents as examples of
what not to say. They are not present as launch claims in README.

`examples/original-author-showcase/` is absent on this branch, so PR #1
showcase content has not entered the release-derived public launch prep line.

## README Guidance

If README references upstream, use the short wording from
`docs/readme-upstream-reference-wording.md`:

```markdown
Upstream projects are referenced as workflow inspiration and docs/tool-surface
parity targets. No upstream academic publication package has been found or
migrated.
```

Do not add upstream publication claims unless a future publication package gate
finds concrete paper artifacts.
