# TulingResearch Plus v0.1.0 Announcement Draft

TulingResearch Plus `v0.1.0` is ready for final human review as the first release-candidate foundation of a Python MCP-first research workflow engine.

This release focuses on architecture, contracts, local tools, deterministic dry-run workflows, fake-mode examples, repo-scoped skills, and release gates. It does not claim fully automatic research or live API execution by default.

## What Is Included

- Single-window multi-agent lane architecture.
- Contracts-first MCP tool design.
- TulingResearch Core local tools.
- PDF input to Markdown Phase A.
- Semantic Graph dry-run.
- Literature Survey dry-run.
- Vault / Context base.
- Race Mode base with Source Hygiene.
- Feature Capsule skeletons.
- DocFlow Article Blocks.
- SOP Graph Generator.
- Figure Asset Registry.
- Paper Draft Gate.
- Codex-compatible skills.
- Fake-mode examples.
- Local install smoke checks and public repo hygiene files.

## What To Try First

```powershell
python -m pip install -e ".[dev]"
python -m pytest
python -m tuling_research.mcp_server --manifest
```

Optional PDF smoke:

```powershell
python -m pip install -e ".[dev,pdf]"
python -m pytest tests/workflow/test_example_pdf_to_markdown.py
```

## What Is Not Included Yet

- Default live network adapters.
- Heavy OCR.
- Advanced PDF layout parsing.
- Full automatic paper writing.
- Real GPU experiment execution.
- Automatic PyPI publishing.
- Private or unauthorized source idea implementation.

## Safety

TulingResearch Plus keeps Source Hygiene as a release gate. Private repository content, leaked roadmap material, NDA content, proprietary code, copied incompatible-license implementation details, private papers without authorization, restricted datasets, and secrets must not become implementation inputs.

## Roadmap

Next planned work after `v0.1.0` includes opt-in live scholarly adapters, stronger PDF conversion, citation graph expansion, OCR and layout parsing, advanced survey workflows, and later optional LLM/cloud/GPU adapters.
