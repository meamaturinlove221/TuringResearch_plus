# Research OS Positioning v2

Status: planning draft.

TuringResearch Plus should be positioned as a local-first research operating
system for evidence-led work, not as a SaaS product, not as an autonomous paper
writer, and not as a remote experiment runner.

## Core Position

TuringResearch Plus helps researchers organize:

- project workspaces;
- evidence and artifact status;
- route and experiment plans;
- paper reading notes and method cards;
- dashboard and advisor review packs;
- privacy and release gates;
- plugin and capability metadata;
- benchmark and replay checks.

It keeps the researcher in the loop for evidence interpretation, paper claims,
privacy decisions, and release approval.

## What Makes It Different

- Review-first instead of automation-first.
- Local-first instead of cloud-first.
- Evidence status instead of result storytelling.
- Fake/default tests instead of live-by-default dependencies.
- Manifest-only plugin strategy before runtime plugin execution.
- Public demo safety before public marketing.

## v0.7 Positioning Goals

- Make public onboarding easier without hiding limitations.
- Improve dashboard and export outputs so users can inspect work quickly.
- Clarify plugin ecosystem direction without unsafe execution.
- Add more demo-safe projects to show generality beyond VGGT.
- Prepare real-world dogfooding cases with strict evidence boundaries.

## Messaging Guardrails

Do not claim:

- autonomous research completion;
- automatic paper writing;
- automatic remote experiment execution;
- legal compliance;
- benchmark success from fake replay;
- VGGT or SparseConv3D success without real evidence.

## Audience

- Individual researchers organizing multi-stage projects.
- Small labs coordinating evidence, artifacts, and paper drafts.
- Tool builders designing local research workflow extensions.
- Advisors or reviewers who need transparent project status.

## Near-term Product Shape

- Python package and local CLI/MCP smoke surface.
- Static docs and demo-safe examples.
- Static dashboards and Markdown bundles.
- Manifest registries for tools/plugins/skills.
- Local replay and quality gates.

## Long-term Product Shape

- Local-first vault UI.
- Reviewed plugin ecosystem.
- Stronger dataset/license policy assistance.
- Optional export adapters.
- Better cross-project learning without transferring evidence automatically.
