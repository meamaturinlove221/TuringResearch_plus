# v1.0.0 Split Readiness Summary

Status: ready for human review, not approved for automatic split.

Round: 192.

This summary explains how v1.0 treats future split repositories after the final
local export bundles were prepared.

## Decision

TuringResearch Plus should launch the main repository first. The main repo
remains the only flagship, install target, quickstart path, public API home,
and release-gate surface.

Physical split is delayed until after explicit human approval.

## No External URLs Yet

The README and docs must not link to nonexistent GitHub repositories. Until a
real repository exists, references must use local docs and `split_ready/`
paths only.

## Main Repo Responsibilities

The main repository keeps:

- install and quickstart;
- CLI/MCP entry points;
- public API contracts;
- public demo suite;
- plugin loading and safety framework;
- privacy, compliance, and regression gates;
- release notes and launch readiness reports.

## Release Boundary

For v1.0, split-ready bundles are launch-supporting artifacts. They do not
change package behavior, dependency behavior, install instructions, or public
API stability.

## Remaining Before Real Split

- maintainer approval;
- repository creation approval;
- final license review;
- final privacy/compliance/claim-safety review;
- README and issue-template review for each spoke;
- flagship README link update after the repository exists.
