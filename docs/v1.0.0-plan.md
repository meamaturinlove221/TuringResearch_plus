# TuringResearch Plus v1.0.0 Plan

## Goals

- Release TuringResearch Plus as a stable Python MCP-first research workflow engine.
- Freeze public APIs, MCP tool namespaces, contracts, and artifact schemas.
- Provide complete docs and tested examples for fake-mode and opt-in live workflows.
- Support optional LLM, cloud, and GPU adapters through explicit protocols.

## Features

- Full research workflow engine from vague intent to research brief, survey, graph expansion, insight, hypothesis, experiment plan, evidence vault, and paper artifacts.
- Live MCP adapters for scholarly search, paper metadata, web content, and workflow services.
- Stable public API for `turing_research` and `turing_research_plus`.
- Full documentation for installation, MCP usage, workflows, adapters, testing, source hygiene, and release operations.
- Tested examples across Core, PDF, Semantic Graph, Literature Survey, Vault, Context, Race Mode, SOP Graph, and Paper Pipeline.
- Optional LLM integrations with fake-mode parity and manual/live test markers.
- Optional cloud and GPU execution adapters for experiment workflows.

## Non-Goals

- No hidden network activity during import, dry-run, or default tests.
- No default use of private or unauthorized sources.
- No unsupported copying from license-incompatible projects.
- No guarantee that live adapters work without user-provided credentials and provider availability.
- No paper result fabrication when ExperimentReport is missing.

## Risks

- Public API stability can freeze poor abstractions if v0.2.0 through v0.4.0 feedback is not incorporated.
- Live adapter support can become provider-specific without strict protocols.
- Optional cloud and GPU adapters introduce security, cost, and reproducibility risks.
- End-to-end workflow claims need continuous EvidenceRef integrity across many modules.

## Required Tests

- Full unit, contract, workflow, integration_fake, package, namespace, and example test matrix.
- API regression tests for stable public models and tool contracts.
- Full fake-mode end-to-end research workflow tests.
- Manual/live adapter tests skipped by default.
- Source Hygiene, EvidenceRef, BudgetGate, StateLedger, and Paper Draft Gate integrity tests.
- Packaging and entry point tests for `turingresearch-plus` and `turingresearch-plus-mcp`.

## Required Docs

- Stable API reference.
- MCP tool reference.
- Adapter configuration guide.
- End-to-end workflow guide.
- Security and Source Hygiene policy.
- License review and contribution guide.
- Release checklist and migration guide.

## Release Blockers

- Any public MCP tool missing contract, docs, and test coverage.
- Any default path requiring network access, API keys, LLM provider credentials, cloud, or GPU.
- Any artifact that cannot serialize to markdown and json.
- Any research conclusion, paper section, or implementation task without evidence and source-hygiene gates.
- Any naming, import, packaging, or namespace drift.
