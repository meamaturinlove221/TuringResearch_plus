# v0.8.0 Candidates

Status: planning draft.

Round: 149.

This document records candidate work for v0.8. It is not a scope lock and does
not authorize implementation, publishing, networking, or plugin execution.

## Candidate Details

### Local Server Dashboard

Goal: provide a local-only dashboard server for browsing workspaces, evidence,
case studies, exports, vault UI outputs, and public demos.

Acceptance questions:

- Does it run without cloud services?
- Does it avoid reading private locations unless explicitly configured?
- Can it fall back to static HTML/Markdown export?
- Does it clearly mark fake/demo and not-ready evidence?

### Research Paper Writing Beta

Goal: assemble review-ready paper sections from existing scaffold outputs while
preserving evidence, citation, experiment, and claim gates.

Acceptance questions:

- Are final abstract/results/conclusion still blocked without evidence and
  human approval?
- Are citation-grade and fake/manual-note sources separated?
- Are unsafe claims blocked before export?

### Public Plugin Registry Draft

Goal: define a local registry draft for public plugins with metadata,
compatibility reports, safety labels, and review status.

Acceptance questions:

- Are third-party plugins disabled by default?
- Does the registry avoid loading or executing code?
- Are permissions, safety level, compatibility, docs, and tests visible?

### More Case Studies

Goal: add public demo-safe case studies that show the workflow across project
types.

Acceptance questions:

- Are all examples fake/demo or evidence-reviewed?
- Are private paths, raw data, model files, and advisor-private comments absent?
- Are limitations and human-review requirements prominent?

### Real OS-level Plugin Sandbox Research

Goal: research and design a real sandbox path for future plugin execution.

Acceptance questions:

- Is policy-level safety clearly separated from runtime isolation?
- Are shell, network, secrets, remote write, and file-system permissions tested?
- Is unknown plugin execution still blocked until a later gate?

## Deferred Or Supporting Candidates

- Real PPTX/PDF polish should follow dashboard and paper beta needs.
- Dataset/license compliance expansion should follow case-study asset needs.
- MCP marketplace polish should follow the public plugin registry draft.
- Multi-project analytics should follow local dashboard foundations.
- Collaboration workflow should wait until privacy and local review semantics
  are clearer.
