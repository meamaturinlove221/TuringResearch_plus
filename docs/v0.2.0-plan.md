# TulingResearch Plus v0.2.0 Plan

## Goals

- Move from dry-run-only graph discovery toward opt-in live scholarly adapters.
- Improve PDF Markdown Phase A with better converters and local figure/table extraction.
- Make citation graph expansion and Vault traversal more useful for evidence-backed research.
- Expand examples while keeping default execution fake-mode and network-free.

## Features

- Live Semantic Scholar adapter through a service protocol.
- Live arXiv adapter through a service protocol.
- Converter selection for PDF Markdown with clearer quality scoring.
- Local PDF figure and table extraction with artifact metadata.
- Real citation graph expansion with depth, node, year, citation count, and open-access gates.
- Stronger Vault graph traversal for claim, evidence, concept, and relation pages.
- Better examples for citation graph, survey, Vault, and PDF workflows.

## Non-Goals

- No default live API execution.
- No OCR pipeline.
- No advanced layout-aware parser.
- No automatic full paper generation.
- No cloud or GPU experiment execution.
- No implementation from private, leaked, or unauthorized sources.

## Risks

- Live adapters can introduce flaky behavior through rate limits or upstream schema changes.
- PDF figure/table extraction can produce incomplete artifacts without clear warnings.
- Citation graph expansion can exceed budgets if frontier control is incomplete.
- Vault traversal can surface unsupported claims if edge typing is too loose.

## Required Tests

- Mocked contract tests for Semantic Scholar and arXiv adapters.
- Live/manual tests skipped by default for any real provider call.
- PDF fixture tests for figure and table extraction.
- Citation graph expansion tests for depth, deduplication, filters, frontier nodes, and recommended next reads.
- Vault traversal tests for supported_by, contradicts, derived_from, and related_to edges.
- Example dry-run tests that require no API keys.

## Required Docs

- `docs/install.md` adapter extras and environment variable guidance.
- `docs/mcp-tools.md` status updates for live-capable graph and PDF tools.
- `docs/pdf_markdown.md` extraction limits and quality interpretation.
- `docs/examples.md` for updated fake-mode examples.
- `docs/security-and-source-hygiene.md` notes for live-source provenance.

## Release Blockers

- Default tests require network access or real API keys.
- Any adapter bypasses the service protocol boundary.
- Any live source can enter implementation without Source Hygiene.
- Any extracted PDF artifact lacks metadata or serialization support.
- Any graph expansion path lacks budget and depth controls.
