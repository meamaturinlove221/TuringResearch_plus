# Demo GIF Plan

Status: planning draft.

Round: 157.

This plan defines future GIF concepts for public README positioning. It does
not generate media.

## GIF 1: Public Demo Walkthrough

Length target: 20 to 30 seconds.

Flow:

1. open public demo dashboard;
2. filter or scan demo projects;
3. open evidence ledger;
4. open advisor pack;
5. show safe demo mode badge.

Boundary:

- all content must be fake/demo;
- no real project data;
- no result success claims.

## GIF 2: Research Case Review

Length target: 20 to 30 seconds.

Flow:

1. open VGGT case-study draft;
2. show redaction report;
3. show claim safety report;
4. show blocked claims and human-review markers.

Boundary:

- dogfooding case only;
- no SparseConv3D success claim;
- no raw data or private paths.

## GIF 3: Plugin Safety Story

Length target: 15 to 25 seconds.

Flow:

1. show plugin manifest;
2. show sandbox policy;
3. show compatibility report;
4. show disabled-by-default status.

Boundary:

- do not execute plugin code;
- do not show live credentials;
- no network requirement.

## GIF Acceptance Gate

Before a GIF is published, it must pass:

- privacy scan;
- old-name scan;
- no secrets scan;
- no raw data scan;
- claim safety review;
- README wording review.
