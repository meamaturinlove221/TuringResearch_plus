Status: planned matrix.

The v1.1 test matrix keeps fake/default local behavior as the release baseline.

## Matrix

| Area | Command | Release blocking | Notes |
| --- | --- | --- | --- |
| Ruff | `python -m ruff check .` | Yes | lint and import hygiene |
| Unit | `python -m pytest tests/unit` | Yes | deterministic local behavior |
| Contract | `python -m pytest tests/contract` | Yes | package, API, release surface |
| Workflow | `python -m pytest tests/workflow` | Yes | fake/default workflow checks |
| Full pytest | `python -m pytest` | Yes | excludes live/manual by default |
| Mypy | `python -m mypy src` | Yes for RC | full source typing signal |
| Name integrity | `python -m pytest tests/contract/test_name_integrity.py -q` | Yes | old-name prevention |
| Public hygiene | `python -m pytest tests/contract/test_public_release_hygiene.py -q` | Yes | public release safety |
| Privacy gate | `python -m pytest tests/workflow/test_public_demo_privacy_gate.py -q` | Yes | public demo privacy |

| Paper/demo | `python -m pytest tests/workflow/test_v1_1_paper_demo_integration.py -q` | Yes | v1.1 paper/demo line |
| Split readiness | `python -m pytest tests/workflow/test_v1_1_split_ready_bundles.py -q` | Review | only if split bundle changes |
| Live tests | `python -m pytest tests/live -m live` | No | skipped unless explicitly enabled |

## Python Versions

- Python 3.11: release baseline.
- Python 3.12: compatibility signal.

## Skip Rules

- Live tests are skipped by default.
- Manual release operations are not part of CI.
- Child repository creation is not part of CI.
- PyPI publish is not part of CI.

## Required v1.1 RC Summary

The v1.1 release candidate summary should record:

- full pytest result;
- mypy result;
- name integrity result;
- privacy/security result;
- docs/dashboard integration result;
- paper/demo integration result;
- release notes and changelog review status;
- manual approval status.
