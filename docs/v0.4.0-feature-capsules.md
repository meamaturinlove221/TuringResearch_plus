# v0.4.0 Feature Capsules

Status: created for planning.

Round 79 creates v0.4 feature capsules for the remote artifact, dashboard,
paper digest, advisor export, and release-hardening surfaces.

## Capsules

| Capsule | Purpose | Status |
| --- | --- | --- |
| `github_artifact_sync` | Controlled GitHub-based artifact manifest sync. | skeleton |
| `ssh_sftp_remote_reader` | Read-only remote artifact listing and summary. | skeleton |
| `nas_smb_artifact_store` | Shared local-network artifact store planning. | skeleton |
| `cloud_object_artifact_index` | Cloud object index-only adapter planning. | skeleton |
| `modal_run_dashboard` | Static/read-only run dashboard from structured reports. | skeleton |
| `paper_digest_three_pass` | Expanded paper digest using three-pass reading. | skeleton |
| `advisor_export_pdf_pptx` | Advisor export from reviewed Markdown sections. | skeleton |
| `public_release_hardening` | Final release gates, hygiene, and packaging hardening. | skeleton |

## Shared Safety Rules

- No default network.
- No permission bypass.
- No automatic large-file download.
- No remote code execution.
- No secret persistence.
- No raw data or SMPL-X model file packaging.
- Remote artifacts are retrieved/indexed, not verified.
- No legacy project naming.
