# v0.7.0 Roadmap

Status: planning draft.

v0.7 should build on the v0.6 release candidate by turning the local,
review-first foundation into a more usable public-facing research workflow
system. The main theme is controlled exposure: better dashboards, safer plugin
distribution, richer exports, deeper public demos, and real-world dogfooding
without relaxing privacy or evidence gates.

This is a roadmap, not an implementation commitment.

## Candidate Directions

| Candidate | Horizon | Priority | Notes |
| --- | --- | --- | --- |
| Real plugin loading with sandbox policy | Medium term | High | Requires explicit sandbox policy, permission gates, and no default execution. |
| Public plugin registry draft | Near term | High | Can start as static registry metadata and review workflow. |
| Dashboard refinement | Near term | High | Improve static dashboard readability before any server UI. |
| Real PPTX/PDF export adapters | Near term | Medium | Optional adapter path only; defaults remain Markdown/source bundles. |
| Real VGGT case study | Medium term | High | Requires real evidence artifacts and human research review. |
| More public demo projects | Near term | High | Best public-release value with low privacy risk if fake/demo only. |
| Related work deep review mode | Medium term | Medium | Requires citation-grade source handling and human paper review. |
| Local-first research vault UI | Longer term | Medium | Useful, but should follow dashboard and privacy hardening. |
| Dataset/license compliance assistant | Medium term | High | High trust value, but needs careful disclaimers and human legal review. |
| MCP distribution polish | Near term | High | Installation, smoke tests, config examples, and package metadata polish. |

## Recommended v0.7 Sprint Order

1. Public plugin registry draft.
2. MCP distribution polish.
3. Dashboard refinement.
4. More public demo projects.
5. Real PPTX/PDF export adapters.
6. Dataset/license compliance assistant planning.
7. Real VGGT case study preparation.
8. Related work deep review mode.
9. Real plugin loading with sandbox policy.
10. Local-first research vault UI.

## Near-term Work

- Keep plugin distribution manifest-only until safety and sandbox policy are
  reviewed.
- Improve public demo breadth with fully fake projects.
- Refine static dashboards and docs navigation.
- Add optional export adapters behind explicit tests and clear dependency
  boundaries.
- Polish MCP install and smoke-test documentation.

## Longer-term Work

- Runtime plugin loading with explicit sandboxing.
- Local-first vault UI across projects.
- Related work deep review with citation-grade source status.
- Dataset/license compliance assistant with strong human-review disclaimers.
- Full VGGT case study only after real evidence and artifact review exists.

## Human Review Required

- Plugin permission approval.
- Third-party plugin registry inclusion.
- Dataset/license risk interpretation.
- Related work citation quality.
- Real VGGT case study claims.
- Public release examples and README claims.

## Non-goals For v0.7

- No SaaS.
- No cloud user system.
- No default live networking.
- No automatic upload of private research data.
- No automatic remote experiment execution.
- No final paper writing.
- No plugin execution without explicit sandbox policy.
- No demo result promoted to observed evidence.
- No SparseConv3D success claim without real evidence.
