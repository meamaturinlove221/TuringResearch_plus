# Post-launch Metrics

Status: metrics plan.

Round: 198.

These metrics are lightweight public-interest and maintenance signals. They do
not guarantee project success and must not be exaggerated in public posts.

## Metrics To Watch

| Metric | Why it matters | How to use |
| --- | --- | --- |
| Install failure count | Setup friction | Improve install docs or package metadata |
| Quickstart failure count | First-run reliability | Patch quickstart/demo tests |
| Docs confusion themes | README clarity | Update FAQ/troubleshooting |
| Privacy/security reports | Safety posture | Triage privately and patch fast |
| Live adapter issues | Optional-live clarity | Improve fake/live docs |
| Plugin safety reports | Extension trust boundary | Tighten plugin policy/tests |
| Demo breakage reports | Public demo reliability | Refresh fixtures |
| User feedback themes | Roadmap signal | Feed v1.1 planning |
| Star/fork/watch trend | Discovery signal | Track only as interest, not proof |
| Interview usage notes | Portfolio usefulness | Improve demo script and FAQ |

## Cadence

- Day 1: install, quickstart, security/privacy, demo breakage.
- Week 1: repeated docs confusion and feedback themes.
- Month 1: roadmap adjustment and patch-release decision.

## Metrics Boundaries

- Do not invent users.
- Do not invent stars.
- Do not claim benchmark wins.
- Do not treat stars as proof of scientific value.
- Do not collect private telemetry.
