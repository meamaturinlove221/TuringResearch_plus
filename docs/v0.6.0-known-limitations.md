# v0.6.0 Known Limitations

Status: release candidate limitations.

## Runtime And Live Features

- Default workflows are fake/local/review-first.
- Live adapters are optional and skipped by default.
- No live remote artifact sync runs by default.
- No automatic remote command execution.
- No automatic remote experiment execution.
- No default live networking.
- No SaaS.
- No user system.
- No cloud deployment.

## Data And Privacy

- Privacy gate is report-only and does not delete or redact files in place.
- Privacy scan is not a legal compliance guarantee.
- Public demos are fake/demo only.
- Raw data and SMPL-X model files must not be packaged.
- Private project links must stay outside release artifacts.

## Plugin System

- Plugins are manifest-only unless explicitly enabled by future reviewed
  integrations.
- MCP plugin registry does not start a server or load plugin code.
- Third-party plugins stay disabled by default.
- Extension safety reports require human review.

## Paper And Research Claims

- Paper assembly generates scaffolds and gate reports, not final papers.
- No final paper writing.
- Paper drafts require human review.
- Related work still requires real paper reading and citation review.
- Experiment sections remain blocked without real run outputs.
- No result values are generated.
- SparseConv3D success is not established.

## Release Process

- v0.6 RC still needs clean branch review.
- Release notes and version metadata are prepared in Round 125.
- Public release requires maintainer approval.
