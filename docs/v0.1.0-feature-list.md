# TulingResearch Plus v0.1.0 Feature List

## Included Features

- Single-window multi-agent lane architecture.
- Contracts-first design.
- TulingResearch Core local tools.
- PDF input to Markdown Phase A.
- Semantic Graph dry-run.
- Literature Survey dry-run.
- Vault / Context base.
- Race Mode base.
- Feature Capsule skeleton.
- DocFlow Article Blocks.
- SOP Graph Generator.
- Figure Asset Registry.
- Paper Draft Gate.
- Codex-compatible skills.

## Supporting Release Assets

- MCP contracts for `core.*`, `pdf.*`, `graph.*`, `research.*`, `vault.*`, `context.*`, `race.*`, and `paper.*`.
- Pydantic models for core artifacts and workflow boundaries.
- Local MCP smoke entry point for `tulingresearch-plus`.
- Fake-mode examples.
- Contract tests and workflow dry-run tests.
- Local install smoke docs.
- Repo-scoped skills index.
- Public repo hygiene files and Source Hygiene policy.

## Package And Server Surface

- Core package: `tuling_research`
- Plus package: `tuling_research_plus`
- MCP server: `tulingresearch-plus`
- Distribution metadata name: `tulingresearch-plus`

## Implementation Status Boundary

- `implemented_minimal`: local deterministic tools or minimal local workflows.
- `implemented_dry_run`: fake-service or dry-run workflows.
- `contract_only`: documented public contract with implementation planned after `v0.1.0`.

## Release Tests

The release candidate includes unit, contract, workflow, package, namespace, skills, MCP, local install, CI, and example dry-run tests.
