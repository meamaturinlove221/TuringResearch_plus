# v1.5.0 Risk Register

Status: scope locked risk register.

Round: 331.

| Risk | Severity | Mitigation | Gate |
| --- | --- | --- | --- |
| release action happens automatically | high | Keep release, tag, GitHub release, PyPI publish, and child repo creation manual. | release prep gate |
| docs deployment leaks private data | high | Run privacy/security gates before deployment and use static public-safe inputs. | privacy gate |
| split repo weakens flagship | medium | Child repos must point back to the flagship and avoid install authority drift. | split execution gate |
| split execution creates repos too early | high | Produce manual packs first; require maintainer approval before repository creation. | split execution gate |
| live SSH/SFTP becomes default | high | Keep live tests skipped unless explicitly enabled by private config. | live skip tests |
| live Scholar/Web uses credentials unsafely | high | Keep provider keys out of repo and require explicit opt-in. | secret scan |
| dashboards overclaim production parity | medium | Label fake/default, deferred, and review-only states visibly. | dashboard review |
| ARIS scope creep | high | Keep ARIS deferred docs active and require a separate v1.6 study lock. | ARIS deferred gate |
| public release implies star growth guarantee | medium | Keep release docs honest: no public-attention guarantee. | README/release review |
| mainline regression | high | Run full regression and mypy before release prep. | full regression |

## Risk Posture

v1.5 is acceptable only if public release work remains privacy-first, manual,
review-gated, and compatible with the v1.4 fake/default production parity
baseline.
