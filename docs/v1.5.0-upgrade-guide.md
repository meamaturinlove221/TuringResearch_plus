# v1.5.0 Upgrade Guide

Status: release-candidate upgrade guide.

Round: 357.

## Version

The v1.5 release-candidate version is `1.5.0rc0`.

## Install

Use the main repository as the install target:

```bash
python -m pip install -e .[dev]
```

For MCP-oriented local checks:

```bash
python -m pip install -e .[dev,mcp]
```

## Import Paths

The compatibility namespace remains available:

```python
import turing_research_plus
```

Facade namespaces remain importable:

```python
import turing_research_core
import turing_research_paper
import turing_research_artifact
import turing_research_experiment
import turing_research_dashboard
import turing_research_plugins
import turing_research_cases
```

## v1.5 Additions

- Use `docs-site/dist/` and `docs-site/dist_manifest.yaml` for the local docs
  deployment dry-run package.
- Use `split_manual/` for human-only split repo creation packs.
- Use optional live guides for private Scholar, Web/Apify, and SFTP opt-in
  policy.
- Use `examples/public_demo/dashboard_showcase/landing.html` for the dashboard
  landing page.
- Use `examples/public_demo/dashboard_showcase/parity.html` for parity progress.
- Use `examples/public_demo/dashboard_showcase/interview.html` for the
  interview demo.
- Treat ARIS docs as deferred study material, not implemented runtime behavior.

- Public deployment remains manual.
