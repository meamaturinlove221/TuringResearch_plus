# v1.0.0 Final Scope

Status: scope locked after upstream refresh.

Round: 178 upstream-adjusted scope lock.

This document locks the intended v1.0 scope for TuringResearch Plus after the
Round 175-177 upstream absorption line. It does not implement code, publish a
release, create split repositories, run remote work, or change package
behavior.

## Release Positioning

v1.0 presents TuringResearch Plus as the flagship local-first Research OS for
evidence-led research workflows.

The release is public-facing, review-first, fake/demo-safe by default, and
honest about what still requires human judgment.

## Locked v1.0 Scope

### 1. Flagship Main Repository

The main repository remains the flagship. It keeps:

- README and product positioning;
- install and quickstart;
- public demo suite;
- docs index;
- release gates;
- package metadata;
- integration story;
- links to optional case/examples split material.

Optional split repos may be prepared later, but they must not replace the
flagship or move the install path.

### 2. Local-first Research OS

v1.0 messaging must emphasize:

- local-first operation;
- fake/demo workflows by default;
- optional live adapters;
- human review required;
- no automatic research completion;
- no default network requirement.

### 3. Core Research Capability Families

The following capability families are in scope:

- Evidence:
  - evidence status semantics;
  - planned vs observed boundary;
  - evidence ledger and claim safety.
- Artifact:
  - artifact audit;
  - handoff metadata;
  - export quality checks.
- Paper:
  - paper digest;
  - method cards;
  - writing scaffold;
  - deep review checklist;
  - no final paper generation claim.
- Route:
  - route DSL;
  - hard gates;
  - failure taxonomy;
  - run/replay status.
- Dashboard:
  - static/local dashboard;
  - public-safe demo dashboard;
  - advisor export surface with optional backends.
- Plugin:
  - plugin manifest;
  - trusted local loading policy;
  - sandbox policy;
  - compatibility harness;
  - no unknown plugin execution by default.
- Workspace:
  - multi-project workspace;
  - project templates;
  - cross-project evidence graph.

### 4. Campaign Catalog

v1.0 includes the static TuringResearch Campaign Catalog as a review-only
strategy book. It may recommend campaigns and skills, but it does not execute
skills, call an LLM, start MCP, use the network, or replace the master
orchestrator.

Campaign Catalog is in scope as documentation, local metadata, and release
positioning. Complex strategy runtime remains deferred.

### 5. MCP / Fake-live Config Polish

v1.0 includes MCP distribution clarity and `.mcp.example.json` fake/live
configuration polish:

- env block style;
- blank credential placeholders only;
- fake mode usable by default;
- live tests require explicit opt-in;
- Semantic Scholar, Apify, and Web live adapters disabled by default;
- plugin tools disabled by default.

### 6. Pod Context Lifecycle Safety Plan

v1.0 includes the Pod Context Lifecycle Safety Plan:

- durable context package policy;
- `PROJECT_CONTEXT.md` / `MEMORY.md` / `ROUTE_SPEC.yaml` context surfaces;
- preflight checklist;
- transfer policy;
- return verification;
- memory conflict policy;
- proposed updates only.

This is not a remote execution runtime. It does not run SSH, Modal, tmux, shell
commands, or git push.

### 7. Public Demo Suite

v1.0 includes public-safe demos clearly marked fake/demo. They must not include
private data, raw data, model payloads, secrets, or unsupported result claims.

### 8. VGGT Public-safe Case Study

v1.0 may include the VGGT dogfooding case as a public-safe case study. It must
be framed as workflow dogfooding, not proof of final experiment success.

The case study must not claim VGGT experiment success, SparseConv3D success,
private data access, or final research conclusions.

### 9. Plugin Manifest And Safety Policy

v1.0 stabilizes the plugin manifest and safety policy surfaces. It does not
stabilize arbitrary plugin execution.

Required boundaries:

- third-party plugins disabled by default;
- execute-code permission blocked unless future sandbox review approves it;
- secrets access forbidden;
- live/network access explicit opt-in only;
- compatibility and safety reports required for review.

### 10. Monorepo Modular Layout

v1.0 keeps the monorepo but supports modular namespace facades:

- `turing_research_core`;
- `turing_research_paper`;
- `turing_research_artifact`;
- `turing_research_experiment`;
- `turing_research_dashboard`;
- `turing_research_plugins`;
- `turing_research_cases`;
- `turing_research_plus` compatibility namespace.

Physical split is not required for v1.0.

### 11. Stable Public API Contracts

v1.0 must lock reviewed public contracts for stable or beta surfaces. Contract
docs and tests must stay aligned with import behavior.

Stable candidates:

- workspace/project summary DTOs;
- privacy report DTOs;
- quality/regression report DTOs;
- evidence status labels;
- route DSL summary schema;
- plugin manifest schema;
- benchmark/replay report schema.

Beta candidates:

- artifact audit/handoff DTOs;
- paper scaffold/deep-review DTOs;
- dashboard/export quality DTOs;
- repo split dry-run manifest;
- campaign catalog local metadata;
- pod context lifecycle safety DTOs.

### 12. Public README And Quickstart

The README and quickstart must be clear enough for a new public reviewer to:

- understand the Research OS positioning;
- install locally;
- run fake/demo checks;
- inspect public demos;
- understand limitations;
- find safety and privacy policies.

### 13. Release, Privacy, And Regression Gates

v1.0 requires:

- full tests;
- mypy;
- name integrity;
- privacy/security gate;
- compliance gate;
- public demo gate;
- quality/regression gate;
- contract tests;
- package/import tests;
- docs link review;
- pre-push sensitive scan.

## Deferred To v1.1

- Pod Lifecycle Manager.
- Context Return Verifier beyond the current safety-plan helper.
- Web live mode polish.
- Paper source fallback refinement.
- Campaign Router model beyond current deterministic local routing.

## Deferred To v1.2 Or Later

- MinerU / heavy PDF fallback.
- Apify workflow templates.
- Remote execution orchestration research.
- Research strategy runtime experiments.
- OS-level plugin sandbox.
- Public plugin marketplace.
- Local server dashboard as a stable product.
- Broad physical split of core/paper/artifact/dashboard/plugins repos.

## Final Scope Decision

v1.0 scope is **locked for release hardening**.

New features after this point should be deferred unless they fix a release
blocker, close a safety gap, repair documentation drift, or preserve API
compatibility.
