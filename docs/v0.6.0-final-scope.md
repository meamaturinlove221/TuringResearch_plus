# v0.6.0 Final Scope

Status: scope locked.

Round: 103.

v0.6 focuses on moving TuringResearch Plus from a single-project research
workflow engine toward a multi-project research operating system. The release
scope is planning- and architecture-first: workspace foundations, reusable
project templates, cross-project evidence views, privacy policy, plugin
boundaries, capability manifests, and writing scaffolds.

## Scope Summary

1. Multi-project Workspace.
2. General Research Project Template.
3. Cross-project Evidence Graph.
4. Privacy / Data Policy.
5. Plugin Architecture.
6. MCP Plugin Registry.
7. Tool Capability Manifest.
8. Skill Marketplace Layout.
9. Paper Writing Scaffold.
10. Benchmark / Replay Suite.
11. v0.6 RC Gate.

## Sprint Structure

### Sprint 1: Multi-project Workspace Foundation

- Workspace registry for multiple research projects.
- Project metadata and status conventions.
- Workspace overview and demo workspace examples.
- Cross-project privacy boundaries.

### Sprint 2: General Research Project Template

- Template stabilization beyond VGGT.
- Reusable evidence, artifact, route, paper, advisor, and dashboard skeletons.
- Template safety labels for planned, missing, and requires-human-review states.

### Sprint 3: Cross-project Evidence Graph

- Compare methods, claims, artifacts, failures, and routes across projects.
- Identify reusable patterns without transferring evidence automatically.
- Export JSON and Markdown graph summaries.

### Sprint 4: Privacy / Data Policy

- Workspace-level privacy and data classification.
- Secret/raw-data/private-model scan policy.
- Release and sharing gates for examples and dashboards.

### Sprint 5: Plugin / MCP / Capability Surface

- Plugin architecture draft.
- MCP plugin registry draft.
- Tool capability manifest.
- Skill marketplace layout.
- Compatibility and safety metadata.

### Sprint 6: Writing and Replay

- Paper writing scaffold with claim gates.
- Benchmark / replay suite for fake/default workflows.
- Public/demo replay hygiene.
- v0.6 RC gate.

## Recommended Baseline

v0.6 should be treated as a foundation release for multi-project and extension
readiness. It should not expand live networking, remote execution, or automatic
paper writing by default.

## Acceptance Gate

- All default tests pass without network access.
- Name and skills integrity pass.
- Feature capsule files are complete.
- No legacy naming appears in new scope material.
- No secrets, raw data, local project links, or private model files are added.
- Demo and replay outputs remain fake/default or review-only.
- Cross-project evidence remains advisory and requires human review.
- Plugin and MCP registry surfaces remain draft until contract tests mature.
