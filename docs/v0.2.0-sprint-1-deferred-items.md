## Re-entry Criteria

A deferred item can re-enter planning only when:

- its contract is updated first
- tests can run without network or private data by default
- source hygiene and credential policy are documented
- evidence provenance is preserved
- Lane 14 or a later sprint lane explicitly accepts the item
