# v0.6.0 Test Plan

Status: scope locked.

Round: 103.

v0.6 tests should preserve the fake/default, local-first behavior established
in earlier releases while adding multi-project, privacy, plugin, and writing
scaffold checks.

## Required Test Groups

| Area | Required tests |
| --- | --- |
| Multi-project workspace | unit tests for workspace models, registry, overview export; workflow demo workspace test. |
| General research project template | unit tests for template metadata and generation; workflow new-project fixture test. |
| Cross-project evidence graph | unit tests for graph nodes, edges, comparator, Markdown export; workflow fake workspace test. |
| Privacy / data policy | unit tests for privacy models, scanners, classification; contract checks for no secret/raw-data leakage. |
| Plugin architecture | unit tests for plugin manifest, safety levels, loader validation; contract tests for draft schema. |
| MCP plugin registry | unit tests for registry metadata and compatibility; contract tests for example registry. |
| Tool capability manifest | unit tests for capability models, collector, manifest export; workflow fake capability index. |
| Skill marketplace layout | contract tests for skill metadata and marketplace layout; no automatic install behavior. |
| Paper writing scaffold | unit tests for outline, claim checks, section scaffolds; workflow fake paper plan. |
| Benchmark / replay suite | workflow tests for fake/demo replay only; no live remote execution. |

## Global Gates

- `python -m pytest tests/contract/test_name_integrity.py`
- `python -m pytest tests/contract/test_skills_integrity.py`
- package import tests
- full contract suite before integration gates
- full workflow suite before release prep
- `python -m mypy src`
- focused `ruff check`

## Safety Checks

- No `.env` committed outside explicit safety-test allowlists.
- No `local_project_links.yaml`.
- No real API key, token, or private credential value.
- No raw data.
- No private model files.
- No large `npz` payloads.
- No demo result marked observed.
- No cross-project evidence transferred automatically.

## Live / Network Policy

Default tests must not require network access. v0.6 planning does not expand
live sync or remote execution. Any future live path must be explicitly marked,
skipped by default, and fake-equivalent.
