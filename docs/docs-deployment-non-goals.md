# Docs Deployment Non-goals

Status: final non-goals for Round 332.

Round: 332.

The docs deployment strategy is intentionally conservative. v1.5 prepares for
public docs but does not publish them automatically.

## Non-goals

- No automatic GitHub Pages deployment.
- No automatic Cloudflare Pages, Netlify, or similar provider deployment.
- No real public URL written into docs before human deployment.
- No analytics or tracking.
- No provider secrets in repository files.
- No private file upload.
- No raw research data upload.
- No restricted model payload upload.
- No live web fetch during docs build.
- No automatic release publication.
- No automatic PyPI publication.
- No automatic child repository creation.
- No ARIS implementation.
- No guarantee of star growth, traffic, citations, or public attention.

## Allowed in v1.5

- Deployment strategy docs.
- GitHub Pages-ready checklist.
- Static hosting comparison.
- Local-only docs plan.
- No-deploy release bundle planning.
- Public-safety gates for docs source and generated output.
