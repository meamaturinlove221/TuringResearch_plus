# v1.0.0 Import Guide

Status: import guide draft.

Round: 177.

This guide explains the v1.0 import story for TuringResearch Plus.

## Recommended Imports

For new code, prefer domain namespace facades:

```python
from turing_research_core import Workspace, load_workspace_registry
from turing_research_experiment import ExperimentRouteSpec, compile_experiment_route
from turing_research_plugins import PluginManifest, load_plugin_manifest
```

These facades expose reviewed public API surfaces while the implementation
continues to live under `turing_research_plus`.

## Compatibility Imports

Existing imports remain supported:

```python
import turing_research_plus
from turing_research_plus.workspace import Workspace
from turing_research_plus.experiment_route import compile_experiment_route
from turing_research_plus.plugins import PluginManifest
```

Do not remove compatibility imports during v1.0 release hardening.

## Namespace Map

| Use case | Recommended namespace | Compatibility namespace |
| --- | --- | --- |
| Workspace, privacy, quality, templates | `turing_research_core` | `turing_research_plus` |
| Paper digest, scaffold, review | `turing_research_paper` | `turing_research_plus` |
| Artifact audit, handoff, run ingest | `turing_research_artifact` | `turing_research_plus` |
| Routes, gates, failures | `turing_research_experiment` | `turing_research_plus` |
| Dashboard, advisor export, vault UI | `turing_research_dashboard` | `turing_research_plus` |
| Plugin manifests, MCP, safety | `turing_research_plugins` | `turing_research_plus` |
| Public demos, case studies, replay | `turing_research_cases` | `turing_research_plus` |

## Compatibility Alias Registry

The compatibility alias registry documents legacy mapping:

```python
from turing_research_plus.compat import legacy_module_for

legacy = legacy_module_for("turing_research_core", "workspace")
assert legacy == "turing_research_plus.workspace"
```

## Stability Notes

- `turing_research_core` and `turing_research_experiment` are beta facade
  namespaces with stable candidates.
- `turing_research_paper`, `turing_research_artifact`,
  `turing_research_dashboard`, `turing_research_plugins`, and
  `turing_research_cases` remain experimental.
- Experimental namespace availability is not a stable promise for every helper
  inside that domain.

## Import Safety Boundary

Imports must not:

- require network access;
- require API keys;
- execute plugins;
- run experiments;
- read private paths;
- require optional PDF/PPTX backends.

## Migration Rule

For v1.0, migration is recommended but not required. Existing
`turing_research_plus` users should not need to change imports to keep running
default tests and demos.
