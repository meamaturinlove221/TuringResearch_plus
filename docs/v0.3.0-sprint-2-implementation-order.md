# v0.3.0 Sprint 2 Implementation Order

Status: locked.

## Order

1. Web Fetch Adapter + Apify Adapter
2. Related Work Positioning Generator
3. Skill `ENTRY.md` / Routing Table
4. Vault Graph Enhancement
5. Ontology SOPs

## Rationale

Web Fetch and Apify come first because they provide public source intake for
project pages, GitHub README files, method pages, and arXiv HTML pages.

Related Work Positioning follows because it consumes method cards, citation
graphs, collision risk, and fetched/cached source summaries.

Skill routing comes next so future workflows can dispatch consistently across
the expanding skill and lane surface.

Vault Graph Enhancement and Ontology SOPs complete the graph-memory side after
the source and routing paths are stable.

## Dependencies

- Web Fetch depends on live adapter contracts and source hygiene.
- Related Work Positioning depends on PaperMethodCard, CitationGraph, and
  CollisionRiskReport.
- Skill Routing depends on skills integrity and lane registry.
- Vault Graph Enhancement depends on vault schema and edge audit.
- Ontology SOPs depend on vault graph and routing conventions.
