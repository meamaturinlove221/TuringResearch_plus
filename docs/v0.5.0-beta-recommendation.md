# v0.5.0 Beta Recommendation

Status: recommended.

Round: 100.

## Recommended Priority

1. Public Release Hardening
2. Real PPTX/PDF Export
3. Dashboard refinement
4. MCP installation flow
5. Benchmark/demo test suite

## Rationale

v0.5 alpha already proves the fake/default local path:

- static dashboard
- advisor export plan
- project template generator
- packaging docs/tests
- public demo suite
- VGGT dogfooding replay

Beta should turn that into a cleaner public-facing package before chasing large
new systems. The safest next move is hardening: secret scans, docs coherence,
package metadata, public examples, and explicit boundaries.

## Recommended Implementation Order

1. Public Release Hardening
2. MCP installation flow
3. Dashboard refinement
4. Real PPTX/PDF optional adapter
5. Benchmark/demo test suite
6. Beta Integration Gate

## Defer

- Plugin architecture, unless beta hardening shows a stable extension boundary.
- Real VGGT replay with new artifacts, unless new real artifacts are provided.
- Community contribution guide, unless public release is imminent.
- More public demos beyond a small curated expansion.

## Acceptance Gate

- Full tests pass.
- No legacy naming.
- No real secrets in examples.
- No raw data or private model files.
- Live adapters remain optional.
- Binary export remains opt-in.
- Public docs do not overclaim experiment or paper results.
