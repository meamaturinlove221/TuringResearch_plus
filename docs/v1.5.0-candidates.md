# v1.5.0 Candidates

Status: planned candidates.

Round: 328.

v1.5 candidates focus on release readiness, public-facing polish, optional live
opt-in ergonomics, and split execution after v1.4 production parity.

## Selected Candidates

| Candidate | Purpose | Default status | Safety boundary |
| --- | --- | --- | --- |
| public docs deployment | Publish static docs or prepare a deployable docs bundle. | selected | no private data, no fake URLs |
| physical split execution | Create or prepare approved public-safe split repositories after maintainer review. | selected | no automatic repo creation |
| optional live SSH/SFTP polish | Improve private opt-in live transfer ergonomics and skip behavior. | selected | no default SSH/SFTP |
| optional live Scholar/Web polish | Improve private opt-in provider docs and skipped-live checks. | selected | no default network |
| dashboard UX polish | Make dashboards easier to scan and review. | selected | no unsupported claims |
| public release packaging | Harden release notes, package metadata, install checks, and manual release SOP. | selected | manual release only |

## Deferred Candidates

| Candidate | Status | Reason |
| --- | --- | --- |
| ARIS study matrix | maybe_v1.6_study_only | Useful later, but not a default v1.5 implementation. |
| result-to-claim verification prototype | deferred | Needs separate ARIS study scope and claim safety review. |
| experiment audit prototype | deferred | Must not blur into automatic experiment execution. |
| claim audit prototype | deferred | Needs stronger false-authority controls. |
| cross-review fake prototype | deferred | Belongs in a future ARIS study lane, not v1.5 release polish. |

## Candidate Decision

The best v1.5 starting point is public release polish around the stable v1.4
production parity surface. ARIS remains reference-only for this release line.
