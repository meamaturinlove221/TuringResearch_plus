# TuringResearch Plus v0.2.0 Beta Implementation Order

Status: beta implementation order locked.

## Order

1. Live Adapter Contracts
2. Live Semantic Scholar Adapter
3. Citation Graph Expansion
4. Collision Risk Detector
5. Modal / Experiment Run Ingestor
6. Handoff Bundle Export / Import
7. Beta Integration Gate

## Rationale

| Step | Why First / Why Now |
| --- | --- |
| Live Adapter Contracts | Prevents live code from drifting outside Protocol, timeout, retry, cache, and live-test policy boundaries. |
| Live Semantic Scholar Adapter | Provides the first opt-in live source while preserving fake adapter parity. |
| Citation Graph Expansion | Depends on Semantic Scholar adapter, budget gates, and cache behavior. |
| Collision Risk Detector | Needs method cards plus citation/manual evidence to detect duplicate or unsupported route claims. |
| Modal / Experiment Run Ingestor | Should ingest exported artifacts only after route/gate/failure semantics are stable. |
| Handoff Bundle Export / Import | Needs stable manifest rules after live/citation/ingestion outputs are defined. |
| Beta Integration Gate | Verifies fake/live boundary, no-key behavior, JSON/Markdown exports, and Sprint 1/2 preservation. |

## Dependency Notes

- Citation Graph Expansion depends on Live Semantic Scholar Adapter.
- Collision Risk Detector depends on Paper Method Cards and citation/manual
  evidence.
- Modal / Experiment Run Ingestor depends on Evidence Ledger, Failure Taxonomy,
  Hard Gates, and Route Pack artifact requirements.
- Handoff Bundle Export / Import depends on packaging rules from the ingestor
  and must exclude raw data, secrets, and SMPL-X body model files.
