# v1.0.0 Split Repo Plan

Status: planning draft.

Round: 173.

Physical split remains optional for v1.0. The main repository stays the
flagship even if case/examples spokes are created.

## Eligible Split Candidates

Eligible after human approval:

1. `turingresearch-vggt-case`
2. `turingresearch-examples`

Not eligible for v1.0 physical split by default:

- core;
- paper;
- artifact;
- dashboard;
- plugins.

## Why Case / Examples First

Case and examples can stand alone as public-safe demo material without moving
core runtime, package APIs, install instructions, or release gates.

They also help public users understand the project without hollowing out the
flagship.

## Split Preconditions

Before any v1.0 split:

- exact export tree frozen;
- dry-run export passes;
- privacy and compliance gates pass;
- no secrets;
- no raw data;
- no SMPL-X/model payloads;
- no private paths;
- no unsupported claims;
- README points to flagship;
- main repo quickstart remains complete;
- maintainer approves repository creation.

## Main Repo Protection

The main repo must retain:

- install path;
- quickstart;
- docs index;
- public demo overview;
- capability map;
- release gates;
- links to optional spokes.

## Split Non-goals

- no automatic repo creation;
- no automatic push;
- no moving core runtime;
- no package rename;
- no split that weakens main repo star strategy.
