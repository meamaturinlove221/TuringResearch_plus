# TuringResearch Plus v0.2.0 P0 Implementation Order

Round 37 translates the Sprint 1 Feature Capsules into an implementation order
for later rounds. This document is a plan only.

## P0 Order

1. `vggt_smplx_evidence_ledger`
2. `artifact_auditor`
3. `visual_evidence_auditor`
4. `advisor_pack_builder`
5. `pdf_phase_b_figure_table_extraction`

## Why This Order

| Order | Capsule | Reason |
| ---: | --- | --- |
| 1 | `vggt_smplx_evidence_ledger` | Evidence Ledger is the evidence substrate. It defines safe status labels before other tools write reports. |
| 2 | `artifact_auditor` | Artifact Auditor records bundle and board inventory for the ledger. |
| 3 | `visual_evidence_auditor` | Visual Evidence Auditor depends on Artifact Auditor board inventory. |
| 4 | `advisor_pack_builder` | Advisor Pack Builder depends on ledger, artifact audit, and visual audit outputs. |
| 5 | `pdf_phase_b_figure_table_extraction` | PDF Phase B can proceed independently but must align with paper figure registry and method-card provenance. |

## First Implementation Round Should Do

- Promote or reject capsule-local proposed namespaces.
- Update root contracts before public tools are added.
- Add Pydantic models.
- Add tests using fake/local fixtures.
- Implement minimal deterministic behavior.

## First Implementation Round Should Not Do

- Do not run VGGT.
- Do not read private local VGGT paths unless explicitly configured by the user.
- Do not fabricate experiment or advisor results.
- Do not implement Future Sync Adapters.
- Do not turn capsule-local proposed tools into public API without contract and docs updates.
