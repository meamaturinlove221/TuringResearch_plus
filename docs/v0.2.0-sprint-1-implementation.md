# TuringResearch Plus v0.2.0 Sprint 1 Implementation

Round 38 starts the v0.2.0 Sprint 1 implementation with the two evidence
foundation features only:

1. VGGT/SMPL-X Evidence Ledger
2. Artifact Auditor

Visual Evidence Auditor, Advisor Pack Builder, and PDF Phase B full extraction
remain planned for later rounds.

## Implemented In Round 38

| Feature | Module | Status |

| VGGT/SMPL-X Evidence Ledger | `src/turing_research_plus/vggt/` | implemented minimal |
| Artifact Auditor | `src/turing_research_plus/artifact_audit/` | implemented minimal |

## Evidence Ledger

The Evidence Ledger supports these statuses:

- `observed`
- `local-observed`
- `planned`
- `fake-data`
- `failed`
- `hard-blocked`
- `requires-real-paper`
- `requires-real-experiment`
- `requires-human-review`
- `not-enough-evidence`

It records:

- `run_id`
- `version_label`
- `claim`
- `status`
- `evidence_refs`
- `source_files`
- `limitations`
- `blockers`
- `next_actions`

Round 38 keeps V120 and V121 as `requires-human-review` when
`local_scan_evidence_ledger.json` and visual inventory inputs are missing.
SparseConv3D success remains `not-enough-evidence` unless real evidence exists.

## Artifact Auditor

The Artifact Auditor supports:

- manifest-like JSON indexes
- Markdown `local_scan_artifact_index.md`
- artifact path, type, size, optional sha256
- included/omitted status
- omitted reason
- safety flags
- NPZ summary metadata

It does not read `D:/vggt` paths by default and marks private/external path
references as omitted with safety flags.

## NPZ Summary

The minimal NPZ summary reads only `.npy` member headers from an `.npz` file and
records:

- key
- shape
- dtype
- file size
- summary status

It does not load large arrays.

## Public API Boundary

`vggt.evidence_ledger_build` and `artifact.audit` remain capsule-local proposed
tools. They are not added to the frozen public MCP namespace in Round 38.

## Explicit Non-Goals

- No network access.
- No VGGT execution.
- No private VGGT path reads by default.
- No fake experiment results.
- No Visual Evidence Auditor implementation.
- No Advisor Pack Builder implementation.
- No PDF Phase B full extraction implementation.
- No Future Sync Adapters.
