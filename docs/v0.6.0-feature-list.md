# v0.6.0 Feature List

Status: release candidate feature list.

## Core Features

1. Multi-project Workspace
   - Local workspace registry for multiple research cases.
   - Project status, privacy level, paths, and Markdown overview export.
   - Workspace index is not an evidence source.

2. General Research Project Template
   - Reusable template schema and template registry.
   - Template types for VGGT-like experiment projects, paper surveys,
     experiment-heavy projects, software tooling projects, and mixed research.
   - Generated content is marked template / placeholder.

3. Cross-project Evidence Graph
   - JSON / Markdown graph for claims, artifacts, methods, failures, and
     routes across projects.
   - Reports reusable patterns without applying one project's evidence to
     another project.

4. Privacy / Data Policy Layer
   - Safety levels from `public-demo` to `secret-forbidden`.
   - Scanner and report for secrets, private paths, raw data, model payloads,
     personal paths, private advisor feedback, and licensed model files.
   - Report-only by default; no deletion or in-place redaction.

5. Plugin Architecture
   - Manifest model, registry, validation, and Markdown export.
   - Supports adapter, exporter, workflow, skill, validator, and renderer
     plugin types.
   - Does not execute plugin code.

6. MCP Plugin Registry
   - Describes which plugin manifests can map to MCP tool declarations.
   - Requires namespace, permissions, safety level, API key declaration, and
     fake/live mode metadata.
   - Prevents core tool override in validation.

7. Tool Capability Manifest
   - Unified static catalog of core tools, adapters, exporters, workflows, and
     review surfaces.
   - Covers evidence, artifact, visual, advisor, pdf, paper, citation,
     collision, related work, route, failure, dashboard, remote artifact,
     handoff, plugin, and workspace capabilities.

8. Skill Marketplace Layout
   - Local skill catalog built from `.agents/skills`.
   - Categories include orchestration, evidence, artifact, visual, advisor,
     pdf, paper, web, remote, route, failure, dashboard, workspace, plugin, and
     release.
   - This is a local documentation layout, not an online marketplace.

9. Extension Safety Gate
   - Permission policy for plugins, MCP plugins, skills, and adapters.
   - Defaults third-party extensions to disabled.
   - Treats code execution, remote write, secrets access, and raw data access
     as blocked or restricted by policy.

10. Paper Writing Scaffold
    - Organizes existing evidence, method cards, related work, and experiment
      routes into a paper writing structure.
    - Does not write final abstract, results, or conclusions.

11. Method Section Builder
    - Builds evidence-linked method section skeletons from method cards,
      architecture diagrams, and route specs.
    - Keeps figure placeholders and limitations explicit.

12. Related Work Draft Assistant
    - Builds related work skeletons and citation safety reports from related
      work positioning, collision risk, and paper digest material.
    - Requires source status for every citation candidate.

13. Experiment Section Builder
    - Builds experiment section skeletons from route, run ingest, dashboard,
      and failure taxonomy material.
    - Missing result tables remain missing; no numeric results are fabricated.

14. Paper Assembly Gate
    - Integrates paper scaffold, method skeleton, related work skeleton, and
      experiment skeleton.
    - Marks sections as `ready`, `partial`, `blocked`, or
      `requires-human-review`.

15. Benchmark / Replay Suite
    - Local replay scenarios for public demo, VGGT fake replay, demo workspace,
      and paper assembly outputs.
    - Checks expected outputs and regression flags without running real
      experiments.

16. Quality / Regression Gate
    - Quality metrics for docs completeness, test coverage proxy, contract
      consistency, example readiness, safety readiness, fake/live boundary,
      old-name absence, privacy gate pass, and release readiness.
    - Regression gate blocks old names, secrets, missing demos, contract drift,
      fake observed results, and required live tests by default.

17. v0.6 Documentation Consolidation
    - Top-level README refresh.
    - `docs/README.md`, docs index, user guide, developer guide, architecture
      overview, and docs audit.
