# v1.6.0 Roadmap

Status: planned roadmap.

Round: 359.

v1.6 should follow v1.5 public externalization by deciding whether to perform
real public actions that v1.5 only prepared as dry-run or manual packs. The
default recommendation is to keep v1.4 production parity stable, preserve v1.5
public-safety boundaries, and treat each external action as separately reviewed.

## Candidate Direction

1. Real GitHub Pages deployment.
2. Actual physical split repo creation.
3. Optional live SSH/SFTP guarded runner.
4. Optional live Scholar/Web E2E.
5. Dashboard UX polish.
6. Plugin ecosystem polish.
7. ARIS study only.

## Default Recommendation

- Start with a real docs deployment decision only after reviewing the v1.5
  dry-run package.
- Create split repositories only after manual privacy, license, and README
  review.
- Keep optional live work private, explicit, and skipped by default.
- Improve dashboard UX without adding a server or analytics by default.
- Polish plugin ecosystem docs and safety metadata before executing unknown
  plugins.
- Keep ARIS as study-only and never default-enabled in v1.6.

## Release Boundary

This roadmap does not deploy, create repositories, push remotes, run live
providers, execute SSH/SFTP transfers, publish releases, upload data, or
implement ARIS.
