# ARIS Reference-Only Policy

Status: active.

Round: 262.

ARIS may be discussed as a reference, but it must not be implemented in the
v1.3 original parity line.

## Policy

ARIS is:

- a future reference;
- a deferred study topic;
- a source of questions for safety design;
- not a v1.3 runtime dependency;
- not a v1.3 feature claim.

## Acceptable References

Acceptable v1.3 references:

- risk analysis;
- feature evaluation matrix;
- study roadmap;
- backlog item;
- comparison against TuringResearch positioning;
- note that a concept requires design and safety review.

## Unacceptable References

Unacceptable v1.3 references:

- code paths named or documented as active ARIS runtime;
- automatic cross-model review;
- proof-checker execution;
- meta-optimize execution;
- paper-claim-audit execution;
- session stop hook execution;
- automated sleep research loop execution;
- claims that ARIS features are ready.

## Relationship To Original Parity

Original parity work should continue independently:

- make Neocortica parity more runnable;
- make yogsoth parity more traceable;
- improve fake/default demos;
- strengthen runtime dashboards;
- keep human review as the decision boundary.

ARIS can inform future questions, not v1.3 implementation.
