# v1.5.0 Sprint 1 Recommendation

Status: planned recommendation.

Round: 328.

## Recommendation

Start v1.5 with public release infrastructure, not ARIS implementation.

## Sprint 1 Order

1. Public docs deployment plan and static docs checks.
2. Public release packaging smoke: install, metadata, entry points, and release
   draft review.
3. Physical split execution readiness: case/examples/plugin bundles, privacy
   gates, flagship backlink policy, and no nonexistent public URLs.
4. Dashboard UX polish for parity, session, catalog, and public demo views.
5. Optional live SSH/SFTP polish with live tests skipped by default.
6. Optional live Scholar/Web polish with provider credentials kept out of repo.

## Why This Order

v1.4 already made original repo production parity runnable in fake/default mode.
The next bottleneck is public review and release reliability: docs deployment,
split readiness, packaging, and dashboard clarity. Optional live polish should
arrive only after the public path is stable.

## ARIS Position

ARIS is still deferred. If it returns, it should be v1.6 study-only by default,
with a separate safety model and no automatic research or evidence promotion.
