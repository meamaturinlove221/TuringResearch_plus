# Interview Demo Walkthrough

Status: demo script.

Round: 174.

## 10-minute Route

### 0:00 - 1:00: Positioning

Open the README and say:

> TuringResearch Plus is a local-first Research OS. It helps keep evidence,
> artifacts, routes, papers, dashboards, advisor packs, plugins, and privacy
> gates connected. It does not automatically complete research.

### 1:00 - 2:00: Public Demo

Show `examples/public_demo/`.

Emphasize:

- fake/demo first;
- no private data;
- no live credentials;
- no result success claims.

### 2:00 - 3:00: Evidence And Artifacts

Show how evidence status and artifact readiness make claims reviewable.

Message:

> The core engineering idea is that a claim should have a status and evidence,
> not just persuasive prose.

### 3:00 - 4:00: Route DSL

Show route/hard-gate/failure materials.

Message:

> Planned route does not equal executed route. Failure state is useful state.

### 4:00 - 5:30: Paper Intelligence

Show paper scaffold or deep review examples.

Message:

> The system scaffolds writing work but keeps missing experiments, unsafe
> claims, and citation review visible.

### 5:30 - 6:30: Dashboard And Advisor Pack

Show static dashboard and advisor export examples.

Message:

> This is designed for review meetings: what is ready, what is blocked, and
> what decision is needed.

### 6:30 - 7:30: Plugin Safety

Show manifest/sandbox/compatibility docs.

Message:

> Plugin architecture exists, but unknown code is not executed by default.

### 7:30 - 8:30: VGGT Dogfooding

Show public case study or split dry-run report.

Message:

> VGGT is a dogfooding case for workflow discipline, not a success claim.

### 8:30 - 9:30: Testing And Contracts

Show contract tests / replay gates / mypy status summaries.

Message:

> The project treats docs, contracts, and fake replay as engineering surfaces.

### 9:30 - 10:00: Roadmap

Close with v1.0:

- stable core API;
- public demo;
- public README;
- optional case/examples split;
- no private data;
- release candidate hardening.
