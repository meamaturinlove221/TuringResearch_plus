# v0.6.0 Upgrade Guide

Status: release candidate upgrade notes.

This guide is for local users moving from the v0.5 alpha surface to the v0.6
release candidate surface.

## Version Metadata

v0.6 RC uses:

```text
0.6.0rc0
```

The version is updated in `VERSION`, `pyproject.toml`, `turing_research`, and
`turing_research_plus`.

## Install Locally

```powershell
python -m pip install -e .[dev]
python -m pytest -q
python -m mypy src
```

Optional MCP smoke checks remain local:

```powershell
python -m pip install -e .[dev,mcp]
python -m turing_research.mcp_server --manifest
turingresearch-plus-mcp --health-check
```

## What Changed From v0.5 Alpha

- v0.5 alpha focused on static dashboard, advisor export planning, project
  templates, packaging polish, public demo, and VGGT replay.
- v0.6 adds multi-project workspace, cross-project evidence graph, privacy
  policy, plugin registry, capability manifest, skill marketplace, extension
  safety, paper assembly scaffolds, benchmark replay, quality gates, and docs
  consolidation.

## Compatibility Notes

- Existing local fake/demo workflows should remain fake/default.
- Plugins are manifest-only unless explicitly enabled by a future integration.
- Paper writing helpers produce scaffolds and reports, not final paper text.
- Remote/live adapters remain optional and safety-gated.
- Workspace and cross-project outputs are review aids, not evidence sources.

## Required Human Review

Review is still required for:

- private data and release safety decisions;
- plugin permissions and extension safety reports;
- related work citations and paper digest sources;
- experiment result interpretation;
- VGGT dogfooding conclusions.

## Not An Upgrade To Live Automation

v0.6 does not add SaaS, user accounts, cloud deployment, automatic remote
experiment execution, final paper writing, or default live networking.
