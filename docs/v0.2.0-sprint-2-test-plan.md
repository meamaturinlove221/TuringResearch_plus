# TuringResearch Plus v0.2.0 Sprint 2 Test Plan

Status: scope locked.

Sprint 2 tests must remain local, deterministic, and fake/dry-run by default.
No live API key, network access, private VGGT path, real experiment run, or
large PDF fixture is required.

## Test Groups

| Group | Purpose | Required Coverage |
| --- | --- | --- |
| Unit | Validate each model and pure function. | Schema validation, status labels, serialization, deterministic scoring. |
| Contract | Keep public and capsule-local contracts aligned. | Required fields, implementation status, allowed namespaces, naming integrity. |
| Workflow | Exercise the local Sprint 2 chain. | Route DSL -> gates -> failures -> method cards -> architecture draft. |
| Regression | Preserve Sprint 1 safety boundaries. | Planned not observed, SparseConv3D unclaimed, missing inputs recorded. |

## Hard Gate Library Tests

- Gate passes when all required evidence is present.
- Gate blocks when a hard blocker is present.
- Gate returns `requires-human-review` for ambiguous evidence.
- Gate preserves `not-enough-evidence`.
- Gate Markdown and JSON serialization are stable.
- Gate labels match existing Sprint 1 status language.

## Experiment Route DSL Tests

- Route spec validates required fields.
- Route parser accepts a minimal VGGT route fixture.
- Route parser rejects unknown gate labels.
- Route Markdown export includes steps, gates, expected artifacts, and stop
  conditions.
- V260 route remains hard-blocked when evidence says hard-blocked.
- Modal Real SparseConv3D remains planned, not observed.

## Failure Taxonomy Engine Tests

- Known failure labels normalize to canonical categories.
- Severity mapping is deterministic.
- Failure reports preserve blockers and next actions.
- Fast-return, fallback-only, report-only, visual-not-ready, and missing-evidence
  failures remain distinguishable.
- Failure taxonomy serializes to JSON and Markdown.

## Paper-to-Method Card Tests

- Method card validates with paper metadata and EvidenceRef fields.
- Missing real paper input returns `requires-real-paper`.
- Missing figure input returns `requires-human-review`.
- Method card does not fabricate method components from absent inputs.
- PDF Phase B asset report can be referenced without copying copyrighted PDF
  content.

## Figure-to-Architecture Tests

- Architecture draft validates from a minimal Method Card fixture.
- Mermaid export contains nodes and edges.
- Unsupported edges are reported as warnings.
- Figure provenance is preserved.
- Output is labeled draft unless all required evidence is present.

## End-to-End Dry Run

The minimal Sprint 2 workflow should test:

```text
VGGT route fixture
-> Hard Gate Library
-> Experiment Route DSL report
-> Failure Taxonomy report
-> Paper-to-Method Card fixture
-> Figure-to-Architecture draft
```

Expected assertions:

- all outputs serialize to JSON;
- all outputs serialize to Markdown;
- missing inputs are recorded;
- no real VGGT experiment is run;
- no network is used;
- no public API key is required;
- no planned item is written as observed.
