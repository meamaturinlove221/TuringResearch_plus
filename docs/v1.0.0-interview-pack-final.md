This pack is the final interview-facing summary for TuringResearch Plus. It is
for internship interviews, GitHub portfolio review, and public release warmup.
It does not add functionality or claim research success.

## Core Positioning

TuringResearch Plus is a local-first Research OS for evidence-led research
workflows.

It helps organize:

- evidence ledger;
- artifact audit;
- route DSL and failure taxonomy;
- paper digest, method cards, related work, and deep review;
- dashboard and advisor pack;
- plugin manifest and safety policy;
- privacy, compliance, quality, and replay gates.

## What To Emphasize

### Research OS 定位

The project is not a paper summarizer or prompt wrapper. It is a system for
keeping research state explicit and reviewable.

### Monorepo To Modular To Future Split

The project keeps the main repository as the flagship, adds module facades and
API contracts, and only considers future split repos when docs, tests, demos,
and privacy gates are ready.

### Contracts / Tests / Gates

The strongest engineering story is that the project has many layers of
verification:

- unit tests;
- workflow tests;
- contract tests;
- namespace compatibility tests;
- public demo tests;
- privacy and release hygiene gates;
- replay gates;
- mypy and ruff.

### Fake / Live Boundary

Default workflows are fake/demo-first. Live adapters are optional. Public demo
and quickstart paths do not require credentials or network access.

### Privacy-first

The project treats public safety as a core requirement. Public demos and case
studies exclude private paths, raw data, private model payloads, credentials,
and unsupported claims.

### Plugin Safety

Plugin work is manifest-first. Unknown third-party plugins are disabled by
default, permissions are policy-gated, and compatibility reports are required.

### VGGT Dogfooding

VGGT is the dogfooding case. It demonstrates workflow discipline: evidence
management, route changes, failures, advisor packs, paper scaffolds, dashboards,
redaction, and claim safety. It does not claim final experiment success.

## What Not To Overclaim

- automatic research completion;
- automatic real experiment execution;
- automatic final paper writing;
- replacement for human review;
- legal compliance approval;
- unknown plugin execution;
- final VGGT experiment success;
- guaranteed GitHub star growth.
