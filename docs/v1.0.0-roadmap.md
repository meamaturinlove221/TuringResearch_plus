# TuringResearch v1.0.0 Roadmap

Status: planning roadmap.

Round: 173.

This roadmap defines the v1.0 direction after the v0.7 public release prep,
v0.8 repository strategy, and v0.9 split dry-run gates. It does not implement
code, create repositories, publish packages, or change release status.

## v1.0 Goal

TuringResearch v1.0 should be a stable, public, local-first Research OS with
review-first workflows, stable core APIs, clear public demos, and optional
case/examples split repositories.

## Primary Objectives

- stable core API;
- public README;
- public demo;
- plugin manifest stable;
- evidence/artifact/paper/route stable;
- optional split case/examples repos;
- no private data;
- release candidate hardening.

## Target Stable Surfaces

| Surface | v1.0 target |
| --- | --- |
| core | stable workspace, privacy, quality, project template, and evidence status APIs |
| evidence | stable evidence status semantics and no planned-as-observed rule |
| artifact | beta/stable artifact audit and handoff metadata APIs |
| paper | stable scaffold/deep-review DTOs, with final paper generation still out of scope |
| route | stable route DSL, hard gates, failure taxonomy, and run-status model |
| plugin | stable manifest and compatibility contract; execution remains policy-gated |
| dashboard/export | stable static/demo export contracts; optional backends remain optional |
| cases/examples | optional split repos only after human approval and public-safe export gates |

## Release Shape

v1.0 should ship as a flagship main repo release first.

The flagship must keep:

- install path;
- quickstart;
- docs index;
- public demo;
- capability map;
- privacy/compliance policy;
- plugin safety policy;
- quality/regression gates;
- optional links to case/examples spokes.

## Optional Split Repos

The only split repos eligible for v1.0 consideration are:

1. `turingresearch-vggt-case`
2. `turingresearch-examples`

They remain optional. They must not replace the main repo, move install
instructions, or weaken the flagship star strategy.

## v1.0 Non-Claims

- v1.0 does not automatically complete research.
- v1.0 does not automatically run VGGT or Modal experiments.
- v1.0 does not generate final paper conclusions without human review.
- v1.0 does not treat fake/demo replay as observed research evidence.
- v1.0 does not make live networking default.
- v1.0 does not make plugin code execution safe without a reviewed sandbox.

## Candidate Milestones

1. v1.0 API freeze draft.
2. Stable core API contract review.
3. Public README and docs hardening.
4. Public demo and split export gate.
5. Plugin manifest stability gate.
6. Full regression and privacy/security/compliance audit.
7. Release candidate hardening.
8. Maintainer go/no-go.
