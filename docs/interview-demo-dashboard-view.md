# Interview Demo Dashboard View

Status: static/local-first interview view.

Round: 353.

This dashboard view is designed for a 3-10 minute project walkthrough. It gives
an interviewer a compact path through the project architecture, module breadth,
original repo parity, safety gates, tests/contracts, public demo material, split
strategy, and ARIS deferral.

The generated file is:

- `examples/public_demo/dashboard_showcase/interview.html`

## Required Content

The view shows:

- architecture;
- modules;
- original repo parity;
- safety gates;
- tests/contracts;
- public demo;
- split strategy;
- why ARIS deferred.

## Suggested Talk Track

1. Start with the architecture: local-first Research OS, review-first docs,
   examples, contracts, tests, and dashboards.
2. Show the module breadth: Session, Scholar/Web, campaign/catalog,
   vault/ontology, stress/convergence, experiment runbooks, docs-site, UI.
3. Explain original repo parity: stable Neocortica and yogsoth-ai ideas are
   replicated through structural, runtime, and production gates.
4. Point at safety gates: no default network, no unsafe remote command, no raw
   data, no secrets, and no planned/demo output promoted to observed evidence.
5. Use tests/contracts as evidence that the repo is maintainable.
6. Show public demo and split strategy as externalization work.
7. Close with ARIS deferred: cross-model review, proof checking, meta
   optimization, and paper claim audit are future-study ideas because they carry
   research-automation and claim-authority risk.

## Runtime Boundary

The page is standalone static HTML:

- no JavaScript;
- no external assets;
- no analytics;
- no network requirement;
- no live provider call;
- no SSH or SFTP connection;
- no remote command execution;
- no experiment execution;
- no Evidence Ledger mutation.

It is a demo script, not evidence that live providers, remote systems,
experiments, or ARIS workflows ran.
