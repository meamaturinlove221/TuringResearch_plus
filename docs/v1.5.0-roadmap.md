# v1.5.0 Roadmap

Status: planned roadmap.

Round: 328.

v1.5 should follow v1.4 original repo production parity with public release and
distribution hardening. It should not default to ARIS implementation. The best
next investment is to make the now-runnable fake/default production parity
surfaces easier to publish, review, demo, and optionally exercise in private
live mode.

## Default Recommendation

- v1.5 should prioritize public release readiness and public infrastructure.
- v1.5 should keep original repo production parity stable.
- v1.5 should keep live paths explicit opt-in and skipped by default.
- v1.5 should not implement ARIS by default.
- ARIS may be reconsidered as a v1.6 study-only track after a separate scope
  lock.

## Recommended Priority

1. Public docs deployment.
2. Physical split execution.
3. Optional live SSH/SFTP polish.
4. Optional live Scholar/Web polish.
5. Dashboard UX polish.
6. Public release packaging.

## Sprint Shape

Sprint 1 should start with public docs deployment planning and release packaging
checks. Physical split execution can follow only after the docs, privacy, and
release surfaces are stable. Optional live polish should remain private opt-in
and must not become a default test path.

## ARIS Boundary

ARIS remains future reference only. v1.5 does not implement cross-model review,
proof-checker, meta-optimize, paper-claim-audit, automated sleep research loops,
or ARIS runtime machinery.

## Release Boundary

v1.5 planning does not publish, tag, create a release, enable live adapters,
enable remote execution, upload data, or create child repositories
automatically.
