# v1.2.0 Final Risk Register

Status: scope locked risk register.

Round: 231.

## Major Risks

| Risk | Severity | Mitigation |
| --- | --- | --- |
| ARIS work enters v1.2 too early | high | Keep ARIS in v1.3 study roadmap only; v1.2 implements reference parity first. |
| Parity work copies upstream code | high | Reimplement concepts locally; cite ideas in docs; no copied source code. |
| Upstream diff claims are overstated | high | Strictly label manual snapshots, live scans, unknowns, and unverified changes. |
| Session parity becomes remote execution | high | Implement local safety checks only; no SSH, tmux, Modal, shell execution, or git push. |
| Scholar/Web parity enables default live networking | high | Keep fake mode default and live tests disabled unless explicitly opted in. |
| Heavy PDF fallback becomes default path | medium | Keep MinerU/heavy PDF fallback research-only for v1.2. |
| Campaign/vault/ontology parity becomes agent runtime overreach | high | Keep catalog/SOP/parity gates local and review-only. |
| Paper writing beta becomes automation | high | Keep paper beta human-review-only; no final paper or camera-ready prose. |
| OS-level plugin sandbox is overpromised | high | Keep OS-level sandbox out of v1.2 implementation; research only after parity. |
| Current runnable mainline is destabilized | high | Require full regression and no breaking import/install changes. |

## Release Blockers

- old project name reintroduced;
- privacy/security gate failure;
- private path or local secret added;
- raw data or restricted model payload added;
- fake/demo output marked observed;
- unsupported SparseConv3D or VGGT success claim;
- live networking required by default;
- unknown plugin execution enabled;
- remote execution added without explicit future scope;
- automatic GitHub child repo creation added;
- ARIS runtime loop added to v1.2 implementation.

## Carryover Watch Items

- branch/worktree cleanliness;
- manual upstream snapshot freshness;
- MCP config drift;
- live adapter provider drift;
- split-ready bundle drift;
- public demo confusion around fake/live boundaries;
- paper beta overclaim risk;
- local dashboard scope creep.
