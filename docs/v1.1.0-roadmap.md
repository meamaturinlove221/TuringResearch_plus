# v1.1.0 Roadmap

Status: planning draft.

Round: 199.

v1.1 should follow the v1.0 public launch with focused, reviewable work that
improves discovery, onboarding, and daily local usability without changing the
core promise of TuringResearch Plus: a local-first Research OS for evidence,
artifacts, routes, papers, dashboards, plugins, and public-safe demos.

This roadmap is planning only. It does not implement code, create GitHub
repositories, publish release artifacts, or claim user growth.

## v1.1 Direction

The recommended v1.1 theme is:

> Turn the v1.0 public launch into a clearer public product surface.

That means:

- create the safest first split repository only after human approval;
- make public docs easier to browse;
- improve the local dashboard experience;
- advance paper-writing assistance as beta tooling, not final paper generation;
- add more public demos that stay fake/demo-first and privacy-safe.

## Recommended Priority

1. `turingresearch-vggt-case` physical split after human approval.
2. Public documentation site.
3. Local server dashboard.
4. Research paper writing beta.
5. More public demos.

## Upstream-driven Candidates Carried Forward

The v1.0 upstream refresh also identified candidates that may influence v1.1:

- Pod Lifecycle Manager;
- Context Return Verifier;
- web live mode polish;
- paper source fallback refinement;
- Campaign Router model.

These should be considered only if they fit v1.1 capacity and preserve the
default no-network, no-remote-execution, no-fake-result-promotion boundaries.

## Sequencing

1. Review v1.0 launch feedback and monitoring signals.
2. Final human review of `split_ready/turingresearch-vggt-case`.
3. If approved, create the case repository and update flagship links only after
   the repository exists.
4. Draft the public documentation site from existing docs, with no private data.
5. Scope the local server dashboard with a static fallback and no SaaS behavior.
6. Scope paper writing beta around claim-checked drafts and advisor review.
7. Add more public demos only after privacy and public-demo gates pass.

## Release Gates

Every v1.1 item should keep these gates:

- no private data;
- no secrets;
- no raw data;
- no restricted model payloads;
- no unsupported experiment claims;
- no automatic experiment execution;
- no automatic final paper writing;
- live features remain opt-in;
- plugin execution remains restricted;
- main repository remains the flagship.

## Non-goal Reminder

v1.1 should not become a broad rewrite. The goal is a clean public growth sprint,
not a new agent runtime, cloud platform, marketplace, or remote execution system.
