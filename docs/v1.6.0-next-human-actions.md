# v1.6.0 Next Human Actions

Status: maintainer checklist.

Round: 390.

## Before Any Public Release

1. Review `README.md`, `LICENSE`, `CITATION.cff`, `CONTRIBUTING.md`,
   `CODE_OF_CONDUCT.md`, and `SECURITY.md`.
2. Make a final license decision. The current repository remains proprietary
   until maintainers explicitly approve a license change.
3. Review `docs/open-source-hygiene-gate-report.md` and
   `docs/open-source-blockers.md`.
4. Run a clean checkout validation on the release branch.
5. Review `docs-site/release_bundle/` and manifests.
6. Decide whether to enable GitHub Pages manually.
7. Review split manual packs and decide whether to create child repositories.
8. Review package naming and PyPI readiness before any publication.
9. Review optional live guides before any private live smoke.
10. Review release notes, changelog, and GitHub release draft.
11. Review screenshot/demo asset manifest and capture real screenshots only if
    needed.
12. Review v1.7 roadmap as planning only, not as implementation approval.

## If Deploying Docs Later

- Enable Pages only after reviewing the dry-run workflow.
- Insert a real public URL only after deployment exists.
- Re-run privacy, name integrity, and no-secret gates after deployment files
  are generated.

## If Creating Split Repositories Later

- Create repositories manually.
- Use placeholder remote URLs until real repositories exist.
- Re-run split safety and privacy checks.
- Confirm each child README links back to the flagship repository only after a
  real URL exists.

## If Running Optional Live Smoke Later

- Use private local env only.
- Do not commit credentials.
- Keep live tests skipped by default.
- Treat live output as retrieved context until human review.

## If Publishing Package Later

- Decide package name and compatibility policy.
- Confirm license and metadata.
- Build artifacts in a clean environment.
- Inspect artifacts before upload.
- Do not publish to PyPI automatically.

## If Publishing A GitHub Release Later

- Create the tag only after choosing the exact source commit.
- Use `docs/github-release-draft-v1.6.0.md` as draft text.
- Run `docs/github-release-final-checklist-v1.6.0.md`.
- Run `docs/post-release-verification-v1.6.0.md` after publication.
- Do not imply Pages, PyPI, split repositories, or live smoke happened unless
  those actions were actually performed.

## If Resuming ARIS Later

- Scope ARIS as a study-only track.
- Keep disabled by default.
- Add a separate safety gate before any implementation.
