# v1.0.0 Release Notes

Status: release-candidate notes.

Round: 195.

TuringResearch Plus v1.0.0 is the first release-candidate line that presents
the project as a local-first Research OS with public API contracts, public demo
materials, split-readiness documentation, and launch hardening.

This release is fake/demo-first by default. Live adapters remain optional and
explicitly gated.

## Highlights

- Local-first Research OS positioning.
- Multi-project workspace and public demo suite.
- Evidence, artifact, visual, advisor, dashboard, and export review surfaces.
- Paper intelligence surfaces for digest, citation, related work, method
  cards, deep review, and collision risk.
- Experiment Route DSL, hard gates, failure taxonomy, run ingest, and benchmark
  replay.
- Remote artifact, handoff, and pod-context lifecycle safety planning.
- Plugin manifest, safety policy, compatibility harness, and MCP registry.
- Campaign Catalog and MCP fake/live configuration polish.
- VGGT public-safe case study and split-ready export bundle.
- Modular monorepo namespace facades and v1 public API contracts.
- Release, privacy, security, regression, and public launch gates.

## What Is Stable Enough For v1 Review

- Public package identity: `turingresearch-plus`.
- Compatibility namespace: `turing_research_plus`.
- New facade namespaces for core, paper, artifact, experiment, dashboard,
  plugins, and cases.
- Fake/default quickstart and public demo path.
- Plugin manifest safety boundaries.
- Privacy/security/compliance review gates.

## What Remains Review-gated

- Live adapters.
- Optional PDF/PPTX backends.
- Physical split repository creation.
- Plugin contribution ecosystem.
- Any real experiment execution.
- Any final paper conclusion.

## Release Boundary

v1.0.0 does not publish SaaS, cloud users, automatic experiment execution,
automatic final paper writing, unknown plugin execution, or physical split
repositories by default.
