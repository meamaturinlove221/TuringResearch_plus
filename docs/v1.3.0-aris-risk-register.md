# v1.3.0 ARIS Risk Register

Status: study risk register.

Round: 257.

## High-level Risk

ARIS-style ideas can improve review depth, but they can also make the system
look more authoritative than the evidence supports. v1.3 must study this
carefully before any adoption.

## Risks

| Risk | Severity | Mitigation |
| --- | --- | --- |
| Cross-model review creates false authority | high | Treat all model review as advisory and require human review. |
| Result-to-claim verification infers missing evidence | high | Missing results must block claims instead of filling them. |
| Experiment audit implies real execution success | high | Audit output must stay proposed/review-only. |
| Paper-claim audit becomes paper writing automation | high | No final conclusion, camera-ready generation, or automatic resubmission. |
| Proof checker overstates correctness | high | Use checklist language and avoid proof guarantees. |
| Meta-optimize silently changes behavior | high | Require explicit config and no silent self-modification. |
| Effort levels hide quality assumptions | medium | Make level definitions visible and testable. |
| Paper compile audit encourages premature publishing | medium | Treat compile output as completeness checklist only. |
| Live provider drift | medium | Keep live calls disabled by default. |
| Evidence mutation drift | high | No automatic evidence ledger mutation from ARIS study outputs. |

## Release Safety Rule

If an ARIS study item cannot be explained as local, fake/default, review-only,
and human-reviewed, it should remain deferred.
