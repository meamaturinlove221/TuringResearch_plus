# TuringResearch Plus v0.2.0 Sprint 2 Known Limitations

Sprint 2 creates a local fake/dry-run integration surface for experiment route
planning, hard gate validation, failure attribution, paper method cards, and
text architecture drafts. It does not prove live experiment success.

## Limitations

- Route compile is a planning artifact, not a VGGT or Modal run.
- Hard gate failures describe plan readiness, not completed experiment results.
- Failure attribution is conservative and requires EvidenceRef-compatible input
  or `requires_human_review`.
- The NeuralBody and HumanRAM method-card fixtures are fake/manual notes and do
  not represent complete paper review.
- Architecture diagrams are derived from fixtures and require human review
  before being used as paper figures or advisor-ready proof.
- Proposed `experiment.*` and new `paper.*` tools remain capsule-local until a
  future contracts-first public API decision.
- The current integration does not call live Semantic Scholar, arXiv, Apify,
  OpenAI-compatible LLMs, image models, or graph rendering APIs.
- The current integration does not read `D:/vggt`, execute VGGT, execute Modal,
  or inspect private artifacts.
- Future Sync Adapters remain non-goal: no NAS, SSH, SFTP, GitHub artifact sync,
  handoff bundle, or cloud object storage adapter is included.
- No real related-work conclusion, SparseConv3D success claim, or advisor-ready
  proof should be inferred from the fixtures.

## Release Risk

The Sprint 2 surface is suitable for local maintainer review and follow-on
implementation planning. It is not suitable for public claims of experiment
success, complete paper analysis, or autonomous research execution.
