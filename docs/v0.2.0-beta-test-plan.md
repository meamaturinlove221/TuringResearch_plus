# TuringResearch Plus v0.2.0 Beta Test Plan

Status: beta test plan locked.

## Required Test Groups

| Test Group | Required Coverage |
| --- | --- |
| Unit tests | Models, parsers, adapter fake implementations, collision rules, bundle manifests. |
| Contract tests | Live adapter contracts, citation graph contracts, collision detector contracts, run ingestor contracts, handoff contracts. |
| Workflow tests | Fake mode end-to-end beta flows and no-key behavior. |
| Name integrity tests | No legacy project naming outside migration docs. |
| Package import tests | `turing_research`, `turing_research_plus`, and new beta modules import without optional live deps. |
| Sprint 1 integration tests | Must keep passing. |
| Sprint 2 integration tests | Must keep passing. |
| Live tests | Marked `live` and skipped by default; must require explicit env flag and keys. |

## Acceptance Checks

- All live adapters can run in fake mode.
- Missing API keys do not fail default tests.
- Live tests require explicit opt-in.
- Collision detector supports manual evidence and fake citation graph.
- Modal Run Ingestor reads local exported result packages only.
- Handoff bundle excludes raw data, secrets, and SMPL-X body model files.
- Every new output serializes to JSON and Markdown.
- No fake result is written as observed.
- Live search results remain unverified until human review.

## Suggested Focused Test Commands

```powershell
python -m pytest tests/unit
python -m pytest tests/contract
python -m pytest tests/workflow
python -m pytest tests/contract/test_name_integrity.py
python -m mypy src
```

Live tests must not be part of default CI.
