# v1.1.0 Issue Triage Policy

Status: complete.

Round: 207.

This policy adapts the post-launch triage plan for v1.1 stabilization.

## Triage Order

1. Security/privacy reports.
2. Install or quickstart failures.
3. Public demo breakage.
4. Plugin safety concerns.
5. Docs confusion around fake/live boundaries.
6. Research case requests.
7. Feature requests and roadmap ideas.

## Required First Check

Before replying publicly, check whether the issue includes:

- API keys or tokens;
- `.env` content;
- raw data;
- restricted model files;
- private local paths;
- private advisor feedback;
- exploit payloads that should be private.

If yes, ask the reporter to redact and move to private handling if necessary.

## Labels

| Label | Use |
| --- | --- |
| `bug` | Reproducible behavior defect |
| `enhancement` | Feature request |
| `security` | Security concern |
| `privacy` | Privacy, raw data, private path, or redaction concern |
| `case-study` | Research case proposal |
| `plugin` | Plugin proposal or plugin metadata |
| `plugin-safety` | Plugin permission or sandbox concern |
| `quickstart` | Quickstart failure |
| `demo-breakage` | Public demo fixture or dashboard issue |
| `docs-confusion` | Docs or wording confusion |

## Severity

| Severity | Examples | Action |
| --- | --- | --- |
| P0 | secret leak, private data leak, unsafe plugin execution | private triage and hotfix review |
| P1 | install broken, quickstart broken, public demo unusable | patch candidate |
| P2 | confusing docs, live/fake ambiguity, minor fixture issue | docs or test update |
| P3 | roadmap idea, case proposal, plugin idea | backlog review |

## Public Response Rules

- Do not ask users to post secrets.
- Do not ask users to upload raw data.
- Do not ask users to upload SMPL-X model files or restricted model payloads.
- Do not debug private VGGT artifacts in public.
- Do not mark planned/fake work as observed to close an issue.
- Keep live adapter troubleshooting opt-in and credential-free.

## Close Criteria

An issue may be closed when:

- a fix, doc update, or test update is merged;
- the report cannot be reproduced and lacks public-safe reproduction details;
- it is a duplicate with a linked active issue;
- it requests a non-goal such as SaaS, automatic real experiment execution, or
  automatic final paper generation.
