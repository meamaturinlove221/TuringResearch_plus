# TuringResearch Plus v0.2.0 Alpha Roadmap to Beta

The beta should turn the alpha planning surface into a stronger, still-safe
research workflow layer without overclaiming live execution.

## Recommended Beta Work

| Item | Goal | Risk |
| --- | --- | --- |
| Live Semantic Scholar adapter | Add opt-in paper lookup with live tests skipped by default. | API limits and key handling. |
| Real citation graph expansion | Expand graph workflows from fake adapter to optional live adapter. | Source quality and rate limits. |
| Paper Collision Risk Detector | Compare method cards against planned VGGT routes to avoid duplicate or unsafe claims. | Requires better evidence mapping. |
| Modal Run Ingestor | Ingest VGGT-side run artifacts into evidence ledger and advisor pack. | Must not execute Modal from TuringResearch. |
| Handoff Bundle Export / Import | Move route packs and artifact summaries between machines without private path leakage. | Sync discipline and privacy. |
| Vault graph enhancement | Strengthen claim/evidence graph traversal and edge audit. | Schema drift. |
| Richer advisor pack export | Add richer Markdown bundles and optional static assets. | Avoid PPT/PDF overclaiming. |

## Beta Non-Goals Unless Explicitly Reopened

- Default network execution.
- Heavy OCR pipeline.
- Direct Modal execution.
- Direct VGGT project mutation.
- NAS / SSH / GitHub artifact sync beyond reviewed handoff bundle design.
- Public claims of experiment success without real evidence artifacts.
