# v1.1.0 Feedback Templates

Status: complete.

Round: 207.

This document summarizes the public feedback templates prepared for v1.1
post-launch stabilization. The templates are designed to collect useful reports
without asking users to upload secrets, raw data, restricted model files, or
private paths.

## Templates

| Template | Purpose |
| --- | --- |
| `.github/ISSUE_TEMPLATE/bug_report.md` | Reproducible bug reports with environment, fake/live status, and safety notes |
| `.github/ISSUE_TEMPLATE/feature_request.md` | Feature ideas with public-safe workflow and test expectations |
| `.github/ISSUE_TEMPLATE/research_case_request.md` | Public-safe research case and demo case proposals |
| `.github/ISSUE_TEMPLATE/plugin_proposal.md` | Plugin proposals with explicit permission declarations |
| `.github/ISSUE_TEMPLATE/security_privacy_report.md` | Redacted security/privacy reports and triage prompts |
| `.github/PULL_REQUEST_TEMPLATE.md` | PR safety, source hygiene, data sensitivity, and test checklist |

## Required Fields

Templates now ask for:

- expected behavior;
- actual behavior;
- reproduction steps;
- environment;
- whether live mode is enabled;
- whether plugin mode is enabled;
- whether data is public/demo/private;
- safety/privacy notes;
- optional screenshots/logs;
- no-secrets warning.

## Safety Warnings

Every relevant template tells users not to upload:

- API keys;
- tokens;
- `.env` files;
- raw data;
- SMPL-X model files or other restricted model payloads;
- private local paths;
- private advisor feedback.

## Plugin Proposal Boundary

Plugin proposals must declare requested permissions. Unknown third-party
plugins remain disabled by default. `execute_code`, shell access, and secrets
access remain denied or forbidden unless a future approved safety gate changes
that policy.

## Research Case Boundary

Research case requests must declare data sensitivity. Cases that require private
data, raw data, restricted model files, or unsupported result claims should not
be handled as public issues without redaction and human review.
