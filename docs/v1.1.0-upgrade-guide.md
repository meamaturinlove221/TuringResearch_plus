# v1.1.0 Upgrade Guide

Status: release-candidate upgrade guide.

Round: 229.

## Version

The v1.1 release-candidate version is `1.1.0rc0`.

## Install

Use the main repository as the install target:

```bash
python -m pip install -e .[dev]
```

For MCP-oriented local checks:

```bash
python -m pip install -e .[dev,mcp]
```

## Import Paths

The compatibility namespace remains available:

```python
import turing_research_plus
```

Facade namespaces remain importable:

```python
import turing_research_core
import turing_research_paper
import turing_research_artifact
import turing_research_experiment
import turing_research_dashboard
import turing_research_plugins
import turing_research_cases
```

## Configuration

`.mcp.example.json` remains fake/default. Live adapters and live tests require
explicit opt-in from a private local config. Do not commit real credentials.

## v1.1 Additions

- Use `docs-site/` for the local-first docs skeleton.
- Use the static docs-site builder for local HTML output.
- Use the local server dashboard only on localhost in read-only mode.
- Use the Dashboard Data API for read-only public demo summaries.
- Treat paper writing beta output as review-only draft material.
- Treat `split_ready/` bundles as manual export candidates, not published
  repositories.

## Behavioral Notes

- Live adapters remain optional.
- Unknown plugin execution remains blocked.
- CI workflows run fake/default checks and skip live tests.
- Release publication remains manual.
