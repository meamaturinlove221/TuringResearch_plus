# TulingResearch Plus v0.3.0 Plan

## Goals

- Add opt-in OCR and layout-aware PDF parsing.
- Improve advanced literature survey workflows.
- Polish the path from validated gaps to experiments.
- Add optional LLM-assisted convergence and stress testing with deterministic fallbacks.
- Make Source Hygiene easier to inspect through a dashboard-oriented export.

## Features

- OCR pipeline for local PDFs with explicit quality warnings.
- Layout-aware PDF parsing for sections, captions, figures, tables, and page maps.
- Advanced literature survey planning, screening, snowball expansion, and evidence matrix output.
- Hypothesis-to-experiment workflow polish across gap prioritization, hypothesis generation, ideation, convergence, stress, and experiment planning.
- Optional LLM-assisted convergence and stress tests behind protocol adapters.
- Source Hygiene dashboard export for allowed, blocked, and watch-only ideas.

## Non-Goals

- No mandatory LLM dependency.
- No default live provider execution.
- No automatic implementation from watch-only sources.
- No claim, insight, gap, or paper section without EvidenceRef.
- No production cloud execution.

## Risks

- OCR can amplify poor scans into misleading text if quality warnings are weak.
- Layout parsing can overfit fixture PDFs and fail on publisher variation.
- Optional LLM paths can produce non-deterministic outputs unless prompts and gates are strict.
- Advanced survey workflows can overstate conclusions when full-text coverage is too low.

## Required Tests

- OCR fixture tests for valid, empty, low-quality, and mixed-page PDFs.
- Layout parser tests for section hierarchy, figures, tables, captions, and page maps.
- Deep survey tests enforcing full_text_ratio and evidence-backed final gaps.
- Hypothesis-to-experiment workflow dry-run tests across ResearchArtifact handoffs.
- Mocked optional LLM protocol tests and skipped live/manual provider tests.
- Source Hygiene dashboard export tests.

## Required Docs

- OCR pipeline guide.
- Layout-aware PDF parsing guide.
- Advanced survey workflow guide.
- Hypothesis-to-experiment operator guide.
- Optional LLM configuration and safety policy.
- Source Hygiene dashboard guide.

## Release Blockers

- OCR or layout extraction lacks quality warnings.
- Survey final gaps can pass without evidence refs.
- Optional LLM execution runs in default tests.
- Source Hygiene dashboard misclassifies blocked or unknown sources as implementation-ready.
- Hypothesis-to-experiment workflow loses artifact provenance.
