# v1.2.0 Sprint 1 Recommendation

Status: recommended sprint plan.

Round: 230.

## Recommended Sprint 1

The first v1.2 sprint should prioritize:

1. Physical split execution for `turingresearch-vggt-case`.
2. Physical split execution for `turingresearch-examples`.
3. Public docs deployment.
4. Local server dashboard polish.
5. Pod Lifecycle Manager scope/prototype.

Paper writing beta refinement should remain an immediate follow-up or a parallel
lane if capacity allows.

## Why This Order

### 1. `turingresearch-vggt-case` physical split

This is still the safest first child repo because:

- the bundle is already public-safe and split-ready;
- it is a dogfooding story, not a core runtime split;
- it can point back to the flagship and protect the main install/star path;
- it does not need private data or experiment-source release.

Execution must wait for human approval, final safety review, and actual child
repository creation permission.

### 2. `turingresearch-examples` physical split

This should follow the case repo once the sync pattern is validated. It gives
public users a lightweight demo/template mirror while preserving the main repo
as source of truth.

### 3. Public docs deployment

The docs site builder exists locally. v1.2 should make it useful publicly, but
only after private path, fake link, and safety checks pass.

### 4. Local server dashboard polish

The local server dashboard is the most practical daily usability improvement.
It must stay:

- localhost-only;
- read-only;
- no login;
- no upload;
- no default network;
- backed by static/read-only data APIs.

### 5. Pod Lifecycle Manager

The safety plan exists. v1.2 can introduce local lifecycle checks and return
metadata validation without implementing remote execution.

## Sprint 1 Gates

- name integrity;
- privacy/security gate;
- public demo privacy gate;
- split bundle safety gate;
- docs link check;
- no child repo URL until the repo exists;
- no private data;
- no unsupported claims;
- no default live networking;
- no unknown plugin execution.

## Expected Sprint 1 Deliverables

- human-approved child repo creation checklist;
- created case/examples repos or explicit no-go record;
- docs deployment plan or deployment-ready output;
- dashboard polish report and route test update;
- Pod Lifecycle Manager scope and contract draft.
