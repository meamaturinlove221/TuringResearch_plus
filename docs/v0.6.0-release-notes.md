# TuringResearch Plus v0.6.0 RC Release Notes

Status: release candidate preparation.

Version metadata: `0.6.0rc0`.

TuringResearch Plus v0.6 expands the project from a single-case research
workflow into a multi-project, review-first research operating system
foundation. It adds workspace management, reusable research templates,
cross-project evidence comparison, privacy gates, manifest-only plugin
metadata, paper assembly scaffolds, benchmark replay, quality gates, and a
consolidated documentation index.

This release candidate does not publish a package, create a tag, or push a
GitHub release. It prepares the local repository state for maintainer review.

## Highlights

- Multi-project workspace registry and Markdown overview.
- General research project templates for several research styles.
- Cross-project evidence graph for reusable pattern hints.
- Privacy / data policy scan reports and redaction proposals.
- Manifest-only plugin architecture and MCP plugin registry.
- Tool capability manifest and local skill marketplace layout.
- Extension safety gate for plugin, MCP plugin, skill, and adapter manifests.
- Paper writing scaffold with method, related-work, and experiment section
  builders.
- Paper assembly gate that blocks final abstract, results, and conclusion when
  evidence is missing.
- Benchmark / replay suite for public demo, VGGT fake replay, workspace, and
  paper assembly scenarios.
- Quality / regression gate for release-readiness checks.
- v0.6 documentation consolidation with user guide, developer guide,
  architecture overview, and docs index.

## Release Posture

v0.6 remains fake/default and review-first:

- default tests do not require network access;
- live tests remain optional and skipped by default;
- plugins are not executed by default;
- remote execution is not performed;
- paper scaffolds require human review;
- fake/demo outputs are not promoted to observed evidence.

## VGGT Dogfooding Boundary

The VGGT human-prior material remains a research dogfooding case. It is useful
for exercising evidence, artifact, replay, dashboard, paper scaffold, and
advisor workflows. It is not a claim of new VGGT experiment success, and
SparseConv3D success is still not established.

## Verification Summary

Round 125 release prep validates the local fake/default surface with full
pytest, static typing, version import checks, and pre-push safety scans. See
`docs/v0.6.0-test-summary.md` for the command-level summary.

## Not Included

- No SaaS product.
- No user account system.
- No default live networking.
- No automatic remote experiment execution.
- No final paper writing.
- No legal compliance guarantee from privacy scans.
- No package publication, tag creation, or GitHub release.
