# v1.2.0 Risk Register

Status: planning risk register.

Round: 230.

## Major Risks

| Risk | Severity | Mitigation |
| --- | --- | --- |
| Physical split fragments the flagship story | high | Create only case/examples first; child README must point back to flagship and not carry install authority. |
| Child repos drift from main repo source of truth | high | Use manual sync SOP, source-derived versions, and safety checks before every mirror update. |
| Public docs deployment leaks private/local paths | high | Build from reviewed docs only; run privacy and link checks before deployment. |
| Local server dashboard expands into SaaS scope | high | Keep localhost-only, read-only, no login, no upload, and no public network binding. |
| Pod Lifecycle Manager is mistaken for remote execution | high | Keep it local validation only; no SSH, tmux, Modal, shell execution, or git push. |
| Context Return Verifier writes unreviewed updates | high | Produce proposed updates only; require human review before evidence ledger changes. |
| Paper writing beta is mistaken for final paper writing | high | Keep results blocked, citation/claim guards visible, and human review required. |
| Real web/scholar live polish introduces default networking | high | Keep fake mode default and live opt-in only. |
| Apify templates imply live runs or cost assumptions | medium | Mark templates as optional and manual; no committed keys or automatic execution. |
| MinerU/heavy PDF fallback adds dependency/legal complexity | medium | Treat as research-only until licensing, reproducibility, and optional install path are clear. |
| OS-level plugin sandbox overpromises security | high | Keep as research, not guarantee; unknown plugin execution stays disabled. |

## Release Blockers

- name integrity failure;
- privacy/security gate failure;
- secrets detected;
- raw data or restricted model payload included;
- private local paths included;
- unsupported experiment success claim;
- fake/demo output promoted as observed evidence;
- child repository URL added before the repository exists;
- docs deployment with unreviewed private paths;
- live networking required by default;
- unknown plugin execution enabled.

## Watch Items

- child repo maintenance cost;
- main repo star/fork/watch trend after split;
- docs deployment feedback;
- install/quickstart confusion after child repos exist;
- local dashboard security expectations;
- live adapter provider drift;
- plugin contribution demand;
- paper beta overclaim risk.
