# v1.0.0 Final Risk Register

Status: locked after upstream refresh.

Round: 178 upstream-adjusted scope lock.

## Release Risks

| Risk | Severity | Mitigation | Release blocker |
| --- | --- | --- | --- |
| README overclaims automatic research | high | Keep fake/demo, human review, and no-final-paper wording visible | yes |
| Public demo leaks private data | high | Run privacy/security and public demo gates | yes |
| Fake result becomes observed evidence | high | Enforce evidence status and wording checks | yes |
| Stable API freezes weak abstractions | high | Freeze only reviewed contracts; label beta/experimental surfaces | yes |
| Plugin manifest implies execution safety | high | State manifest stability is not execution approval | yes |
| Unknown plugin execution becomes enabled | high | Keep execution blocked by default | yes |
| Live adapter becomes required by default | high | Keep live tests optional and fake/demo quickstart default | yes |
| Pod lifecycle docs are mistaken for remote execution support | high | Keep remote execution non-goals visible and test no-execution boundaries | yes |
| Campaign Catalog is mistaken for an autonomous agent runtime | medium | State it is local deterministic routing metadata only | no |
| License posture unclear | high | Require maintainer review before release | yes |
| Split repos weaken flagship | medium | Keep main repo install, README, docs, and demos complete | no |
| Optional backends appear required | medium | Keep PDF/PPTX optional and skipped gracefully | no |
| Docs drift from actual capabilities | medium | Run docs review and contract checks before RC | yes |
| Worktree includes unrelated historical changes | high | Require clean branch review and selective commit | yes |

## Security And Privacy Risks

| Risk | Mitigation |
| --- | --- |
| secret or API-key-like value appears | sensitive scan and public hygiene tests |
| private local path appears | privacy scan and release surface review |
| raw data appears | public demo gate and pre-push scan |
| SMPL-X/model payload appears | large/model payload scan |
| private advisor feedback leaks | case study redaction and split dry-run safety checks |
| pod context package includes forbidden dotfiles | pod preflight and transfer policy checks |
| unsafe archive path traversal appears | pod transfer path validation |

## Research Claim Risks

| Risk | Mitigation |
| --- | --- |
| VGGT case read as success proof | label as dogfooding case only |
| SparseConv3D described as successful | block unless real evidence ledger supports it |
| paper scaffold read as final paper | keep scaffold/blocked/human-review wording |
| dashboard read as experimental result | label dashboard as review surface |
| pod output read as accepted evidence | keep proposed-updates-only boundary |

## Upstream Adoption Risks

| Risk | Mitigation |
| --- | --- |
| upstream inspiration becomes copied implementation | document no-code-copy boundary |
| v1.1/v1.2 candidates creep into v1.0 | keep implementation freeze and non-goals active |
| live web/scholar behavior becomes expected | keep `.mcp.example.json` fake/default and live opt-in |
| heavy PDF fallback becomes implied | defer MinerU/heavy ingestion to v1.2 or later |

## Residual Risks

- Some users may still misunderstand fake/demo boundaries.
- Optional live providers may drift after release.
- Future split repos may create documentation drift.
- v1.x deprecations may be needed sooner than expected.
- Pod lifecycle safety docs may need stronger examples before remote execution
  research resumes.

## Final Risk Decision

The v1.0 risk posture is acceptable only for release-candidate hardening, not
for automatic public release. Final release still requires maintainer review.
