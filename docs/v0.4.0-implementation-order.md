# v0.4.0 Implementation Order

Status: scope locked.

## Default Order

1. GitHub Artifact Sync.
2. SSH / SFTP Remote Artifact Reader.
3. NAS / SMB Shared Artifact Store.
4. Cloud Object Artifact Index.
5. Remote Artifact Integration Gate.
6. Modal Run Dashboard.
7. Paper Digest / Three-pass Reading.
8. Advisor PDF / PPTX Export.

## Rationale

- GitHub Artifact Sync extends the existing Git-based context handoff and is
  the most natural first remote-artifact path.
- SSH / SFTP adds read-only remote inspection for GPU/pod outputs, but requires
  stronger credential and path safety gates.
- NAS / SMB is useful for local network labs, but should reuse the same manifest
  and safety model.
- Cloud object storage remains index-only first because credential and storage
  governance risk is high.
- The integration gate should happen before dashboard and export features, so
  remote artifact semantics are stable.
- Modal Run Dashboard should read structured outputs and never run Modal.
- Paper Digest and Advisor Export should consume already-reviewed material.

## Checkpoint Guidance

Long implementation rounds should push a checkpoint if more than half complete
and the diff is large. Recommended v0.4 branches:

- `feature/v0.4-remote-artifacts` for Rounds 79-84.
- `feature/v0.4-dashboard` for Rounds 85-88.
- `feature/v0.4-paper-digest` for Rounds 89-92.
- `feature/v0.4-advisor-export` for Rounds 93-96.
- `release/v0.4.0` for Rounds 97-102.
