# v1.4.0 Original Repo Production Parity

Status: scope adjusted.

Round: 291.

v1.4 is redirected from ARIS study prototypes to original repo production
parity. The goal is not to copy upstream code. The goal is to make the stable
reference ideas more runnable, visible, and maintainable in TuringResearch while
preserving local-first safety.

## Planning Input

The adjustment is based on a human upstream scan summary:

- Neocortica-Session remains centered on Git context, pod deployment planning,
  dotfile handling, shell safety, and cross-platform tar behavior.
- Neocortica-Scholar remains centered on MCP config, paper_content,
  paper_reference, paper_reading, SKILL.md, README tool lists, and arxiv2md
  fallback.
- Neocortica-Web remains centered on Apify, web_fetching, web_content, cache,
  MCP config, and dotenv removal.
- yogsoth-ai did not introduce a new large feature that should interrupt the
  mainline.
- ARIS remains outside the v1.4 implementation line.

This is not a strict machine diff. It must not be used to claim added,
modified, or deleted upstream files.

## Definition of Done

- fake/default workflows run locally;
- optional live paths are explicitly opt-in;
- docs and tests agree on the public story;
- no secrets, raw data, private paths, or restricted model payloads are added;
- planned work is not presented as observed evidence;
- ARIS is still deferred.
