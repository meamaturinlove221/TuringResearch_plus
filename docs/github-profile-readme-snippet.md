```markdown
### TuringResearch Plus

I am building **TuringResearch Plus**, a local-first Research OS for keeping
evidence, artifacts, experiment routes, paper review, dashboards, advisor
packs, plugin manifests, and privacy gates connected.

It is intentionally review-first: fake/demo workflows run locally by default,
live adapters are optional, unknown plugins are not executed, and paper or
experiment claims require human review.

Highlights:

- Evidence Ledger: separates planned work from observed evidence.
- Artifact Auditor: checks whether outputs are ready to support claims.
- Route DSL: tracks experiment routes, hard gates, failures, and next actions.
- Paper Intelligence: scaffolds paper sections without inventing results.
- Dashboard / Advisor Pack: packages project status for review.
- Plugin Safety: manifest-only plugin flow with sandbox and compatibility gates.
- Public Demo: fake/demo examples with privacy-first boundaries.

The project is a portfolio-scale engineering system, not a magic scientist.
```
