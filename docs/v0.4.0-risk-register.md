# v0.4.0 Risk Register

Status: scope locked.

| Risk | Severity | Likelihood | Mitigation |
| --- | --- | --- | --- |
| Artifact sync accidentally includes secrets. | high | medium | Reuse handoff safety rules; block `.env`, tokens, private configs, and credential files by default. |
| Raw data or SMPL-X model files are bundled. | high | medium | Enforce manifest-only / summary-only defaults and explicit deny patterns. |
| Remote reader becomes remote executor. | high | medium | Keep SSH / SFTP read-only; no command execution in Sprint 1. |
| GitHub sync pushes private artifacts. | high | medium | Use allowlisted review artifacts and dry-run reports before any push-capable workflow. |
| Modal dashboard implies Modal execution. | medium | medium | Dashboard must read fixture/exported outputs only and state it does not run Modal. |
| Advisor PDF/PPTX export looks more final than evidence supports. | medium | medium | Export must carry limitations, not-ready claims, and requires-human-review status. |
| Paper digest becomes fake complete reading. | medium | medium | Require source refs and manual review flags. |
| Related work refinement overstates novelty. | high | medium | Keep collision risk and safe/unsafe claim gates. |
| UI/dashboard scope grows too large. | medium | medium | Keep static or lightweight report-first UI only if selected. |
| Live/network behavior leaks into default tests. | high | low | Continue default fake mode and skipped live tests. |
| Remote artifact is treated as verified evidence. | high | medium | All remote/index outputs remain retrieved/indexed and require human review. |
| Cloud object storage expands before governance exists. | medium | medium | Keep v0.4 cloud work index-only unless explicitly rescoped. |
| PDF/PPTX export hides limitations. | medium | medium | Export tests must require limitations and not-ready claims. |

## Release Blockers

- Any default network requirement.
- Any secret, raw data, or SMPL-X model file included in a bundle.
- Any route output promoted without evidence.
- Any dashboard or export claiming VGGT / Modal execution that TuringResearch
  did not perform.
- Any old project naming outside historical migration docs.
- Any remote artifact output promoted to observed evidence without ledger review.
