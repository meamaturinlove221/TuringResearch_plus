# v0.3.0 Sprint 1 Known Limitations

## Git-based Context Handoff

- Git is treated as transport only.
- TuringResearch Plus does not run remote code.
- No SSH, Modal, RunPod, NAS, SMB, or cloud sync is implemented.
- `MEMORY.md` is a summary, not source of truth.

## Pod Workflow Pack

- The pack is review-only.
- It does not execute VGGT or Modal.
- Structured outputs must be filled by an external operator or future pod-side process.
- Evidence Ledger updates remain proposed until human review.

## Scholar Pipeline

- Default workflow uses cached Markdown and fake adapters.
- Live Semantic Scholar remains optional.
- Citation graph output is not complete related work.
- Paper method card fixtures are not full paper reading.
- Collision risk still requires human paper review.

## Web Fetch / Apify

- Live web fetching is not implemented in Sprint 1.
- Apify is not called.
- Proposed `web.*` tools are design-only and not public MCP tools.

## VGGT Boundary

- No `D:/vggt` path is read.
- No real VGGT experiment is run.
- SparseConv3D success is not claimed.
- Planned route output is not observed evidence.
