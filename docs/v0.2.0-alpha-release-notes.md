# TuringResearch Plus v0.2.0 Alpha Release Notes

Status: alpha release preparation, not a PyPI release and not an automatic tag.

TuringResearch Plus v0.2.0 alpha extends the v0.1.0 local MCP-first research
workflow skeleton with VGGT / SMPL-X dogfooding infrastructure, route planning,
hard gates, failure language, paper method cards, and text architecture
diagrams.

## What This Alpha Adds

Sprint 1 added the evidence and artifact foundation:

- VGGT/SMPL-X Evidence Ledger.
- Artifact Auditor.
- Visual Evidence Auditor dry-run surface.
- Advisor Pack Builder.
- PDF Phase B Figure/Table Extraction.

Sprint 2 added experiment planning and interpretation support:

- Experiment Route DSL.
- Hard Gate Library.
- Failure Taxonomy Engine.
- Paper-to-Method Card.
- Figure-to-Architecture.
- VGGT Modal SparseConv3D Route Pack.

## What This Alpha Is

- A local fake/dry-run workflow engine.
- A route and evidence management layer for VGGT dogfooding.
- A Markdown-first advisor and planning surface.
- A contracts-first implementation checkpoint.

## What This Alpha Is Not

- It is not a live VGGT experiment runner.
- It is not a Modal execution client.
- It is not a live paper search engine by default.
- It is not a complete OCR or layout parsing system.
- It is not an automatic paper writing system.
- It is not a public API freeze for new Sprint 2 proposed tools.

## Release Boundary

The alpha can be reviewed locally and used to prepare future VGGT-side work. It
must not be presented as proof that SparseConv3D has succeeded or that a full
related-work study has been completed.
