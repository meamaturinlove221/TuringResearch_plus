# v0.3.0 Sprint 2 Recommendation

Status: selected.

## Selected Scope

Sprint 2 should implement the selected Top 5 in this order:

1. Web Fetch Adapter / Apify Adapter implementation
2. Related Work Positioning Generator
3. Skill ENTRY.md / Routing Table
4. Vault Graph Enhancement
5. Knowledge Graph / Wiki / Ontology SOPs

## Implementation Order

### 1. Web Fetch Adapter / Apify Adapter implementation

Reason: Sprint 1 left live web / Apify as a documented gap. Public project
pages, GitHub README files, method pages, and arXiv HTML pages are immediately
useful for VGGT related-work context.

Required contracts:

- `contracts/web_fetch_adapter.yaml`
- `contracts/live_adapters.yaml`

Required tests:

- fake adapter tests
- cache metadata tests
- no-network default tests
- live tests skipped by default
- source hygiene blocking tests

### 2. Related Work Positioning Generator

Reason: TuringResearch already has method cards, citation graph, and collision
risk reports. The next useful layer is conservative positioning text and
claim-safety notes.

Required contracts:

- `contracts/paper_method_card.yaml`
- `contracts/citation_graph.yaml`
- `contracts/collision_risk.yaml`
- new positioning contract

Required tests:

- fake citation graph positioning
- unsafe claim rejection
- missing evidence report
- markdown export

### 3. Skill ENTRY.md / Routing Table

Reason: Upstream signals show value in explicit `ENTRY.md` and campaign routing.
TuringResearch has many skills and lanes; routing needs a compact, maintained
surface.

Required contracts:

- skills integrity contract
- campaign routing docs

Required tests:

- ENTRY exists
- every route points to an existing skill/lane
- no stale skill names

### 4. Vault Graph Enhancement

Reason: Sprint 2 positioning and method-card outputs need stronger graph
storage and traversal.

Required contracts:

- `contracts/vault_schema.yaml`
- vault graph tests

Required tests:

- typed edge traversal
- edge audit
- orphan / missing evidence detection
- graph markdown export

### 5. Knowledge Graph / Wiki / Ontology SOPs

Reason: Yogsoth and split-repo signals point toward ontology SOPs, inline
wikilinks, and structured graph maintenance.

Required contracts:

- vault schema
- ontology SOP draft

Required tests:

- concept page SOP
- alias resolution SOP
- edge batch creation SOP
- merge candidate SOP
- ontology export fixture

## Deferred

- NAS / SMB Shared Artifact Store
- SSH / SFTP Remote Artifact Reader
- GitHub Artifact Sync
- Modal Run Dashboard
- PPTX/PDF Advisor Pack export

These are useful but can expand the risk surface. They should follow after web
retrieval, positioning, and routing are stable.

## Release Criteria

- Default tests do not require network.
- Live tests remain explicit and skipped by default.
- No private or restricted content is fetched.
- No source is marked human verified without review.
- No final research claim is generated automatically.
- No old project naming appears.
