# v1.6.0 Optional Live Smoke Go / No-go

Round: 377
Status: GO for release-candidate review / NO-GO for default live

## Decision

`GO FOR OPTIONAL LIVE SMOKE RELEASE-CANDIDATE REVIEW / NO-GO FOR DEFAULT LIVE`.

The optional live smoke layer is release-ready as a fake/default and
skipped-live safety gate. It is not ready to run live providers by default.

## Go Conditions Met

- Scholar fake smoke pass.
- Web / Apify fake smoke pass.
- SFTP fake smoke pass.
- Live smoke tests skip by default.
- Live output redaction gate pass.
- No secrets.
- No default network.

## No-go Conditions Still In Force

- No live provider call by default.
- No API key, token, password, or private key path in repo.
- No paper download by default.
- No private scraping.
- No login bypass.
- No SSH or SFTP connection by default.
- No remote command.
- No remote delete.
- No raw live output retained.
- No automatic Evidence Ledger write.

## Human Actions Required Later

Before any live smoke is run privately, a maintainer must:

- explicitly enable live tests;
- provide credentials outside the repository;
- confirm target scope;
- run redaction guard before saving output;
- review sanitized reports manually.

## Current Recommendation

Keep optional live smoke enabled only as fake/default tests and skipped-live
checks. Do not enable default live networking.
