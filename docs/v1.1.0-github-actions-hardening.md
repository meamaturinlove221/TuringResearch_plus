# v1.1.0 GitHub Actions Hardening

Status: implemented workflow hardening.

Round: 227.

This round adds explicit PR / push GitHub Actions for local, fake/default
checks. It does not add automatic release, tag creation, PyPI publishing, live
tests, secrets, or artifact uploads.

## Added Workflows

- `.github/workflows/ci.yml`
- `.github/workflows/docs-check.yml`
- `.github/workflows/privacy-gate.yml`

## CI Coverage

The CI workflow covers:

- checkout;
- Python setup;
- minimal dependency install;
- ruff;
- unit tests;
- contract tests;
- name integrity;
- public demo privacy gate;
- optional mypy.

## Docs Check Coverage

The docs workflow covers:

- docs-site fake build;
- docs/dashboard integration;
- public release hygiene;
- public demo walkthrough checks.

## Privacy Gate Coverage

The privacy workflow covers:

- public demo privacy gate;
- v1 security/privacy gate;
- split bundle safety gate;
- paper/demo integration safety gate.

## Safety Boundary

- No secrets are configured.
- No API key is required.
- `TURINGRESEARCH_ENABLE_LIVE_TESTS` is set to `0`.
- Live tests are skipped by default.
- No artifact upload step is present.
- No private data is uploaded.
- No release, tag, or package publish step is present.
- No child repository creation step is present.

## Follow-up

Future release readiness workflows can be added as manual `workflow_dispatch`
checks, but they should still avoid automatic publishing unless a maintainer
explicitly approves a later policy change.
