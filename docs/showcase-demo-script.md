# Showcase Demo Script

Status: demo script draft.

Round: 220.

This script is for a 5-10 minute public showcase. It uses only demo/public
materials.

## Opening

"TuringResearch is a local-first Research OS for keeping evidence, artifacts,
paper work, experiment routes, dashboards, and plugin boundaries reviewable.
It does not automate research judgment; it helps make the state of a project
clear."

## Segment 1 - Main Repo

Open `README.md`.

Point out:

- local-first Research OS;
- fake/demo default;
- live adapters optional;
- human review required;
- quickstart and public demo links.

## Segment 2 - Public Demo

Open `examples/public_demo/SHOWCASE.md`.

Point out:

- demo-only data;
- evidence ledger;
- artifact audit;
- paper/advisor workflow;
- dashboard surfaces.

## Segment 3 - Dashboard

Open `examples/public_demo/demo_dashboard_refined.html` or preview the local
server route.

Point out:

- read-only review surface;
- dashboard data API can export JSON;
- localhost-only server boundary.

## Segment 4 - VGGT Case Study

Open `split_ready/turingresearch-vggt-case/README.md`.

Point out:

- dogfooding case study;
- not a VGGT experiment source repo;
- no raw data;
- no SparseConv3D success claim;
- main repo remains the flagship.

## Segment 5 - Plugin Safety

Open `docs/plugin-sandbox-policy.md`.

Point out:

- third-party plugins disabled by default;
- permissions must be declared;
- no unknown code execution by default;
- human review required.

## Close

"The point of the project is not to replace a researcher. It is to make the
research workflow easier to audit, easier to explain, and harder to overclaim."
