# v1.0.0 Split Execution Checklist

Status: checklist only.

Round: 188.

This checklist is for a future human-approved split. It is not permission to
create repositories during v1.0 launch hardening.

## Required Before Creating Any Spoke Repo

- Maintainer approval recorded.
- Repository name approved.
- License posture reviewed.
- Export manifest frozen.
- README reviewed.
- Privacy gate passes.
- Compliance gate passes.
- Claim-safety gate passes.
- No secrets.
- No raw data.
- No private paths.
- No restricted model payloads.
- No unsupported VGGT or SparseConv3D success claims.
- Main repo README still works without the spoke.
- Main repo quickstart still works without the spoke.
- Child README links to flagship above the fold.
- Child repo says it is optional demo/case material.

## Candidate-specific Checks

### `turingresearch-vggt-case`

- Contains only public-safe case study material.
- Includes redaction and claim-safety reports.
- Does not include experiment source code as a promise.
- Does not claim final research success.
- Does not include raw data or restricted model files.

### `turingresearch-examples`

- Contains only fake/demo examples.
- Includes public demo manifest.
- Includes expected outputs.
- Does not include real private logs.
- Does not include API keys or raw data.

## Post-create Checks

- Flagship README link added only after repo exists.
- Spoke README links back to flagship.
- Spoke issue templates avoid support confusion.
- Main repo release notes describe spokes as optional.
- Install path remains unchanged.

## Explicit Non-actions For v1.0

- Do not create repositories automatically.
- Do not push external child repositories automatically.
- Do not move core runtime.
- Do not change package imports.
- Do not split plugins, paper, artifact, dashboard, or core.
