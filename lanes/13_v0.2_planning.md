- Default tests require network access or real API keys.
- Any live adapter bypasses service protocols.
- Any live source can enter implementation without Source Hygiene.
- Any extracted PDF artifact lacks provenance or serialization support.
- Any graph expansion path lacks budget and depth controls.
- Any survey gap or paper claim lacks EvidenceRef.
- Any paper draft path fabricates experiment results.

## Round 33.6 VGGT Local Path Reader Dry Run

Round 33.6 implements the minimal local VGGT path reader for same-machine
dogfooding. The reader checks local path candidates, scans allow-listed
lightweight files when a private config exists, summarizes `.npz` metadata
without loading arrays, writes outputs only under the TulingResearch Plus
example directory, and remains non-fatal when the private config is missing.

Created or updated:

- `src/tuling_research_plus/dogfooding/`
- `src/tuling_research_plus/dogfooding/__init__.py`
- `src/tuling_research_plus/dogfooding/models.py`
- `src/tuling_research_plus/dogfooding/vggt_local_reader.py`
- `examples/vggt-human-prior-survey/local_scan_summary.md`
- `examples/vggt-human-prior-survey/local_scan_missing_items.md`
- `examples/vggt-human-prior-survey/local_scan_artifact_index.md`
- `tests/workflow/test_vggt_local_reader_dry_run.py`

Rules:

- Private config path is `examples/vggt-human-prior-survey/local_project_links.yaml`.
- Missing private config uses the example config for guidance and does not fail.
- Existing root candidates may be marked `observed`; missing roots are marked `missing`.
- Without a private config, artifact scanning is not started.
- Image files are indexed by path and size only.
- `.npz` files are summary-only: keys, shapes, dtypes, and file size.
- No VGGT code is run, no network is used, and no VGGT directory is written.
- Keyword watchlist covers V770, V129, V260, V900, V930, V999, SparseConv3D, Modal, SMPL-X, triplane, feature adapter, NeuralBody, and HumanRAM.
