## Decision

v0.9 recommendation: `DEFER BROAD SPLIT / PREPARE CASE AND EXAMPLES ONLY`.

Priority candidates:

1. `turingresearch-vggt-case`
2. `turingresearch-examples`

Deferred:

- core;
- paper;
- artifact;
- dashboard;
- plugins.

## Rationale

- Case/examples have lower risk of harming main repo star growth.
- Core split would hollow out the flagship too early.
- Paper and artifact APIs need stabilization.
- Dashboard/export needs a clearer standalone package story.
- Plugins should wait for a real third-party ecosystem and stronger review
  infrastructure.

## Boundaries

- No physical split.
- No repository creation.
- No code movement.
- No package rename.
- No network access.
- No private path read.
- No raw data or model payload.
- No experiment success claim.
