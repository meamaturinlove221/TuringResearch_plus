## Public Release Strategy

Public release should publish only local, demo-safe, review-first materials:
source code, docs, fake demos, static dashboards, manifests, and replay checks.
It must not publish secrets, private paths, raw data, licensed/private model
payloads, unreviewed plugin execution, or final paper claims.

## Boundaries

- No code implementation.
- No publication.
- No tag creation.
- No default network access.
- No private VGGT path read.
- No fake result promoted to observed.
- No final paper claim.
- No SparseConv3D success claim.
