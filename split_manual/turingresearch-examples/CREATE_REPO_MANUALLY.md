# Create Repository Manually

Status: human instructions only.

This file describes the future manual path for creating the
`turingresearch-examples` child repository. It deliberately does not run
commands or create external resources.

## Required Human Decisions

Before any repository exists, a maintainer must approve:

- repository owner;
- repository name;
- visibility;
- final file tree copied from `split_ready/turingresearch-examples/`;
- exact public demo file set;
- README flagship backlink wording;
- privacy review;
- release hygiene review;
- compliance and license posture review.

## Manual Creation Shape

1. Review the source bundle under `split_ready/turingresearch-examples/`.
2. Confirm `SAFETY_CHECKLIST.md` has no blockers.
3. Manually create a public repository named `turingresearch-examples`, if
   approved.
4. Copy only the reviewed source bundle and approved public demo files.
5. Keep the main TuringResearch repository as the flagship install and docs
   entry.
6. Insert a real flagship URL only after the human confirms the canonical URL.
7. Record the action back in the flagship repository ledger after completion.

## Not Allowed In This Pack

- automatic GitHub repository creation;
- automatic external push;
- automatic release creation;
- automatic URL insertion for a repository that does not exist;
- private data upload;
- raw data upload;
- API key or token upload;
- huge artifact upload;
- unsupported research or benchmark claims.
