# Skill Proposals

Use this directory for collaborator-submitted skill proposals that may later become repo-scoped skills under `.agents/skills/`.

Community skill proposals are **design documents**, not implementation code.

## File Location

```text
community/skills/<github-username>/<short-title>.md
```

Use the template:

```text
community/skills/_template.md
```

## Good Skill Proposals Include

- trigger conditions;
- what the skill should do;
- required inputs;
- expected outputs;
- boundaries and refusal conditions;
- source hygiene rules;
- testing requirements;
- how it maps to TuringResearch modules.

Accepted proposals can later be converted by maintainers into `.agents/skills/<skill-name>/SKILL.md`.
