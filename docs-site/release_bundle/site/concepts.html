Concepts

TuringResearch is organized around a few durable ideas:

states stay separate;

keys, private data, or remote commands;

when they fit the project safety model;

review before release or ledger adoption;

review without automatic deployment.

  • **review-first state**: planned, fake/demo, missing, proposed, and reviewed
  • **local-first execution**: default workflows run without cloud services,
  • **parity with boundaries**: original reference repo ideas are replicated only
  • **human confirmation**: generated reports and proposed updates require human
  • **public-safe externalization**: docs and demos are prepared for public
