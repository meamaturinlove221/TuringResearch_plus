# TuringResearch Code Of Conduct

## Expected Behavior

- Be respectful and constructive.
- Keep technical discussion evidence-based.
- Respect source licenses, privacy, and access restrictions.
- Avoid sharing secrets, private papers, restricted datasets, private local
  paths, or unauthorized materials.
- Distinguish planned, fake/demo, live, and observed states clearly.

## Unacceptable Behavior

- Harassment or personal attacks.
- Posting credentials, tokens, private data, or restricted source material.
- Posting raw datasets, private model files, private project paths, or private
  advisor feedback.
- Pressuring maintainers to bypass Source Hygiene rules.
- Presenting unauthorized or private content as public implementation material.
- Presenting fake/demo or planned output as observed evidence.

## Reporting

Report conduct or safety concerns through the repository security or maintainer
contact process. Do not include secrets or private data in public issues.

## Enforcement

Maintainers may edit, hide, or remove content that violates this code of
conduct, the TuringResearch Source Hygiene policy, or the privacy/data boundary.
